Q-102.781
Rechte in der Benutzerkonfiguration von David


Frage
Problem Welche Bedeutungen haben die einzelnen Rechte, die in der Benutzerkonfiguration des David.Adminstrator unter »Konfiguration -> Benutzer -> "Benutzername" -> Rechte« gesetzt werden können?
Produkt David.Administrator
Priorität normal
Effekt normal
Datum 5.11.2009

Antwort
Die folgende Aufstellung zeigt die einzelnen Rechte sowie die entsprechenden Erklärungen dazu:

Hinweis:
Bei Verwendung von David.zehn! oder höher erhalten Sie Informationen zu den Benutzerrechten im KnowledgeBase Artikel Q-109.993.

  • Verteilen von Nachrichten
    Ist dem Anwender diese Berechtigung zugewiesen, so kann er Nachrichten an andere David User im Netz verteilen. Im David.InfoCenter steht ihm hierzu die Funktion Verteilen zur Verfügung.

  • Identifizierung speichern
    Besitzt der Anwender diese Berechtigung, so kann er bei empfangenen Nachrichten die Kennung der Gegenstelle »lernen« lassen. Das bedeutet, die jeweilige Kennung wird in der Identifizierungsliste eingetragen (siehe Menü Datenbanken -> Verteilung -> Identifizierung). Im David.InfoCenter steht ihm hierzu die Funktion Identifizierung zur Verfügung. Soll der Anwender über diese Funktion zusätzlich noch die Verteilkennung ändern können, benötigt er auch die Konsole-Berechtigung.

  • Konsole-Berechtigung
    Besitzt der Anwender diese Berechtigung, so hat er die Möglichkeit, den David.Administrator direkt aus dem David.InfoCenter an seiner Arbeitstation aufzurufen. Voraussetzung hierfür ist, dass diese Arbeitstation unter Windows NT, Windows 2000 oder Windows XP betrieben wird. In diesem Fall steht dann in der Allgemeinen Symbolleiste des David.InfoCenter die Schaltfläche David.Administrator zur Verfügung.

    Hinweis:
    Dieser Schalter gilt auch als Erweiterung zu der Funktion Identifizierung speichern. Der Anwender hat die Möglichkeit, beim Identifizieren von Nachrichten die Verteilkennung zu ersetzen (Funktion Identifizieren im David.InfoCenter).

  • Aufträge aller Benutzer
    Bei Aktivierung des Kontrollkästchens kann der Anwender in seinem Auftragsbuch neben seinen eigenen Versandaufträgen auch diejeneigen aller anderen Anwender sehen. Hierzu muss er im David.InfoCenter den Schalter Ansicht -> Filter -> Aufträge aller Benutzer aktivieren. Ohne das zusätzliche Leserecht auf die Archives der Anwender kann er jedoch die eigentlichen Nachrichten nicht öffnen. Es geht hier nicht um eine globale Ansicht aller versendeten oder empfangenen Nachrichten, sondern nur um die Nachrichten, die sich im Auftragsbuch der Benutzer befinden.

  • Archive Eigenschaften
    Ist dieses Kontrollkästchen aktiviert, so besitzt der Anwender die Berechtigung, auf die Eigenschaften der David Archives ( z.B. Datenbereinigung, Sortierung, etc.) in seinem David.InfoCenter zuzugreifen und diese ggf. zu verändern. In diesem Fall kann er ein Archive im Archive Baum des David.InfoCenter markieren und über das Menü Datei bzw. über die rechte Maustaste die Archive-Eigenschaften aufrufen und hier entsprechende Änderungen vornehmen. Deaktivieren Sie das Kontrollkästchen, wenn Sie dem Benutzer diese Berechtigung entziehen wollen.

  • Versenden von Rundsendungen
    Ist dieses Kontrollkästchen aktiviert, so besitzt der Anwender die Berechtigung, Rundsendungen unter Verwendung des David Befehls @@Rundsendung@@ zu erstellen. Deaktivieren Sie dieses Kontrollkästchen, wenn Sie dem Benutzer diese Berechtigung entziehen wollen. Entdeckt der Service Layer dennoch Aufträge dieses Users, in denen der Befehl @@Rundsendung@@ verwendet wurde, so werden diese nicht versendet, sondern mit der Fehlermeldung »Unerlaubte Rundsendung« in das Sendebuch gestellt.

  • Verschieben und Kopieren von Archives
    Ist dieses Kontrollkästchen aktiviert, so besitzt der Anwender die Berechtigung, im David.InfoCenter per »Drag & Drop« Archives zu verschieben und zu kopieren.

  • Verteilung
    Ist dieses Kontrollkästchen aktiviert, so besitzt der Anwender die Berechtigung, im David.InfoCenter Verteilregeln zu erzeugen. In diesem Fall steht ihm im Archive Kontextmenü bzw. unter dem Hauptmenüpunkt Archive der Eintrag Verteilung zur Verfügung, der es erlaubt, eigene Verteilregeln für beliebige David Archives zu erstellen. Deaktivieren Sie das Kontrollkästchen, wenn Sie dem Benutzer diese Berechtigung entziehen wollen.

  • PGP Verschlüsselung
    Ist dieses Kontrollkästchen aktiviert, so besitzt der Anwender die Berechtigung, PGP-verschlüsselte Nachrichten direkt über das David.InfoCenter zu empfangen und zu versenden. Voraussetzung hierfür ist, dass PGP auf der Arbeitsstation dieses Benutzers installiert und korrekt konfiguriert ist. Trifft dies zu, so stehen im Nachrichten Editor zusätzliche Buttons für die PGP-Ver- und Entschüsselung sowie für die PGP-Signatur zur Verfügung.

    Deaktivieren Sie das Kontrollkästchen, um dem Benutzer diese Berechtigung zu nehmen. In diesem Fall sind die zusätzlichen PGP Buttons im Nachrichten-Editor generell nicht verfügbar, auch wenn PGP auf der Workstation installiert ist.

  • Darf DFÜ-Verbindung aktivieren (Nur Windows BackEnd)
    Falls diese Option aktiviert ist, erscheint in der Benutzer Symbolleiste im David.InfoCenter ein Symbol Senden/Empf.. Mit diesem kann der Benutzer aus seinem InfoCenter die in PostMan oder Grabbing Server eingerichteten Verbindungen aktivieren.

  • Öffentliche Benutzerdaten
    Aktivieren Sie dieses Kontrollkästchen, wenn es auch anderen David Usern möglich sein soll, die Daten dieses Anwenders (z.B. die Unterschrift) zu verwenden. Anwender können dann Sendeaufträgen durch Verwendung des Befehls @@Ownername@@ einen anderen Besitzer zuweisen und dessen Unterschrift über den Befehl @@Unterschrift@@ in die Nachricht einsetzen lassen, sofern für diesen User der Schalter Öffentliche Benutzerdaten gesetzt ist.

  • Kostenschlüssel mitwählen
    Bei Aktivierung des Kontrollkästchens wird der eingetragene Kostenschlüssel bei jedem Wahlversuch dieses Anwenders mitgewählt. Einige Telefonanlagen können diesen Kostenschlüssel z.B. für eine Kostenstellenzuweisung herausfiltern.

    Hinweis:
    Falls die Telefonanlage eine solche Funktion nicht unterstützt, sollten Sie auf keinen Fall diese Option aktivieren.

  • Max. Anzahl Adressaten
    Legen Sie hier die maximale Anzahl von Adressen fest, die der Benutzer für einen Sendeauftrag angeben darf. Versucht dieser Benutzer eine Rundsendung zu versenden, deren Adressliste die hier eingestellt Anzahl von Empfängeradressen überschreitet, so wird eine entsprechende Fehlermeldung in seinem Ausgangs-Archive erstellt. Der Versand findet in diesem Fall nicht statt.

    Tragen Sie den Wert 0 ein, wenn für den Benutzer keine Begrenzung gelten soll.
