Q-100.632
SMTP Fehlermeldungen


Frage
Problem Der eMailversand funktioniert nicht und im Tobit InfoCenter erscheint im Status eine Fehlermeldung.
Produkt David PostMan
Priorität normal
Effekt normal
Datum 16.05.2002

Antwort
Um das Problem zu beheben, müssen Sie die genaue Ursache kennen. Eine Aufstellung der wichtigsten SMTP-Fehlermeldungen mit dem zugehörigen Lösungsvorschlag finden Sie unten.

Zum einen ist die im Tobit InfoCenter unter »Status« erscheinende Fehlermeldung berücksichtigt, zum anderen finden Sie aber auch die Fehlermeldung aus dem Portmonitor des PostMan. Dorthin gelangen Sie im DvISE Administrator bzw. Service Layer über »Monitor -> Ports -> PostMan -> Communication«.

Eine Liste aller SMTP-Fehlercodes finden Sie im
KnowledgeBase Artikel Q-100.344

InfoCenter Portmonitor Code Lösung
Domäne nicht gefunden Resolver could not find "domain" 450 Die Domäne ist existiert nicht oder es besteht keine Verbindung zum Internet. Überprüfen Sie daher die Schreibweise und Ihre Internetverbindung.
Belegt Could not connect to "domain" % Keine Ausgabe eines SMTP-Codes, da das SMTP-Protokoll erst nach Herstellen der Verbindung startet.
Die Leitung ist besetzt oder die Domäne ist ungültig. Überprüfen Sie die Schreibweise oder versuchen Sie es später erneut.
Falscher Absender Wrong sender name 550 Die Absenderadresse ist dem Mailserver nicht bekannt und wird somit von diesem nicht akzeptiert. Überprüfen Sie, ob Sie eine gültige Absenderadresse angegeben haben. Es ist hierbei zu beachten, dass viele öffentliche Mailserver keine "fremden" Absenderadressen akzeptieren.
Falscher Empfänger Wrong receiver name 550 Die Empfängeradresse ist dem Mailserver nicht bekannt und wird somit von diesem nicht akzeptiert. Überprüfen Sie, ob Sie eine gültige Empfängeradresse angegeben haben.
Belegt Time Out % Die Nachricht konnte in diesem Fall nicht übertragen werden. Überprüfen Sie bitte Ihre Verbindung zum Internet. Ursache ist möglicherweise eine Überlastung der Gegenstelle. Dadurch ist die Übertragung zu langsam und Sie bekommen einen "Time Out".
% Transaction failed 554 Die Transaktion ist fehlgeschlagen. Bekommen Sie diese Meldung bereits beim Verbindungsaufbau, hat die Gegenstelle Port 25 nicht freigegeben und es ist somit kein SMTP-Service verfügbar
