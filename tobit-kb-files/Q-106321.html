Service Pack 31-Jul-2003

Code History (Linux)

Die folgenden Neuerungen und Bugfixes sind im Service Pack enthalten.

  2. Neuerungen und Bugfixes sortiert nach Modulen

1. Neuerungen und Bugfixes des aktuellen Service Packs

Diese Übersicht enthält alle Neuerungen und Bugfixes, die im aktuellen Service Pack (31-Jul-2003) vorgenommen wurden.

  • Im DvISE Administrator kann der Login Name eines vorhandenen DvISE Users nun umbenannt werden (z.B. auf Grund einer Namensänderung nach einer Hochzeit).
  • Mögliche fehlerhafte Meldung »Print job ignored. No printer filter defined.« im Archive »Fehlermeldungen«. Diese Meldung trat unter Umständen trotz korrekter Druckereinrichtung in den "Archive-Eigenschaften" auf.
  • Die Eingabelänge der Datenfelder wurde bei den folgenden Datenbanken erweitert: Identifizierung, Verteilregeln, Verteilvorgaben und Robinson-Verzeichnis. Beim ersten Start des neuen DvISE Service Layers werden evtl. vorhandene Daten in das neue Format konvertiert. Die Dateien der bisherigen Datenbanken werden in *.OLD umbenannt.
  • Wurde eine eMail Adresse mit einem Bindestrich (-) in die Robinsonliste eingetragen, so wurde dieser Eintrag nicht gefunden und eine eMail an diese Adresse wurde trotzdem versendet.
  • Unter NT 4 kam es zu einer Schutzverletzung, wenn im DvISE Administrator der Menüpunkt »Tools -> Systeminformationen« aufgerufen wurde.
  • Bei Tracking eMails fehlte die abschließende Boundery. Somit wurden diese unter Umständen von einigen Mail Systemen nicht weiterversendet.
  • Das Tobit InfoCenter Web enthält nun einen HTML Editor.
  • Fehlverhalten, wenn der Betreff einer Nachricht »Navigation« lautete.
  • Audio/Video-Dateien im Archive System können nun auch geöffnet und abgespielt werden.
  • Der Zugriff auf die Homepage funktionierte nach Einspielen des Service Packs 30-Jun-2003 mit MAC Clients nicht mehr.
  • Beim Verschieben/Kopieren von Nachrichten in andere Archives bleibt der ursprüngliche DvISE Benutzer dieser Nachricht nun erhalten (Eintrags-Eigenschaften).
  • Bei den Nachrichtentypen Link, Termin, Aufgabe und Adresse wird nun der wirkliche "Erzeuger" in den Eintrags-Eigenschaften angezeigt (bisher: 00000000).
  • Die Suche in den "Erweiterten Informationen" eines Eintrags funktionierte unter Windows XP/Server 2003 nicht.
  • Fehlverhalten beim Erstellen eines Deckblattes mit dem Tobit InfoCenter: Nach der Erstellung über »Datei -> Deckblatt erstellen« erschien wieder die Aufforderung zur Adresseingabe und die CVP-Datei wurde nicht erstellt.
  • Bei der Installation des Tobit InfoCenter Mobile werden nun auch die Stempel des Tobit InfoCenter Image Editors vom Host auf den Mobile Client kopiert und stehen dort im Tobit InfoCenter Image Editor zur Auswahl.
  • Durch einen automatischen, regelmässigen Refresh wurden unter Umständen bestimmte eingebundene Web-Sites auf der Portal Seite nicht mehr angezeigt.
  • Aktualisierte Druckertreiber (Tobit FaxWare, Tobit Image Messaging) für den Betrieb unter Windows Server 2003.

2. Neuerungen und Bugfixes sortiert nach Modulen

Diese Übersicht enthält alle Neuerungen und Bugfixes, die seit dem ersten Service Pack vorgenommen wurden, unterteilt nach den einzelnen DvISE Modulen. Die neusten Änderungen stehen jeweils am Anfang der Liste.

    • DvISE Service Layer/DvISE Administrator
    • DvISE PostMan
    • DvISE WebBox
    • Tobit InfoCenter Web
    • DvISE TLD für ISDN Karten
    • DvISE Communication Controller
    • Tobit InfoCenter

DvISE Service Layer/DvISE Administrator

  • Im DvISE Administrator kann der Login Name eines vorhandenen DvISE Users nun umbenannt werden (z.B. auf Grund einer Namensänderung nach einer Hochzeit).
  • Mögliche fehlerhafte Meldung "Print job ignored. No printer filter defined." im Archive "Fehlermeldungen". Diese Meldung trat unter Umständen trotz korrekter Druckereinrichtung in den "Archive-Eigenschaften" auf.
  • Die Eingabelänge der Datenfelder wurde bei den folgenden Datenbanken erweitert: Identifizierung, Verteilregeln, Verteilvorgaben und Robinson-Verzeichnis. Beim ersten Start des neuen DvISE Service Layers werden evtl. vorhandene Daten in das neue Format konvertiert. Die Dateien der bisherigen Datenbanken werden in *.OLD umbenannt.
  • Wurde eine eMail Adresse mit einem Bindestrich (-) in die Robinsonliste eingetragen, so wurde dieser Eintrag nicht gefunden und eine eMail an diese Adresse wurde trotzdem versendet.
  • Unter NT 4 kam es zu einer Schutzverletzung, wenn im DvISE Administrator der Menüpunkt »Tools -> Systeminformationen« aufgerufen wurde.
  • Mögliches Fehlverhalten (hohe Systemauslastung) beim Autoreply von HTML Mails behoben.

DvISE PostMan

  • Bei Tracking eMails fehlte die abschließende Boundery. Somit wurden diese unter Umständen von einigen Mail Systemen nicht weiterversendet.

DvISE WebBox

  • Der Zugriff auf die Homepage funktionierte nach Einspielen des Service Packs 30-Jun-2003 mit MAC Clients nicht mehr.

Tobit InfoCenter Web

  • Das Tobit InfoCenter Web enthält nun einen HTML Editor.
  • Fehlverhalten, wenn der Betreff einer Nachricht »Navigation« lautete.
  • Audio/Video-Dateien im Tobit Archive System können nun auch geöffnet und abgespielt werden.
  • Termine im Kalender können nun auch gelöscht werden.

DvISE TLD für ISDN Karten

  • Text-to-Speech und Spracherkennung wird nun auch unterstützt.

DvISE Communication Controller

  • Das Programm DvISE Communication Controller (DvCC.exe) ist nun als 32 Bit Version verfügbar und ersetzt die bisherige 16 Bit Version.

Tobit InfoCenter

  • Beim Verschieben/Kopieren von Nachrichten in andere Archives bleibt der ursprüngliche "Benutzer" dieser Nachricht nun erhalten (Eintrags-Eigenschaften).
  • Bei den Nachrichtentypen Link, Termin, Aufgabe, Adresse wird nun der wirkliche Erzeuger in den Eintrags-Eigenschaften angezeigt (bisher: 00000000).
  • Die Suche in den "Erweiterten Informationen" eines Eintrags funktionierte unter Windows XP/Server 2003 nicht.
  • Fehlverhalten beim Erstellen eines Deckblattes mit dem Tobit InfoCenter: Nach der Erstellung über »Datei -> Deckblatt erstellen« erschien wieder die Aufforderung zur Adresseingabe und die CVP-Datei wurde nicht erstellt.
  • Bei der Installation des Tobit InfoCenter Mobile werden nun auch die Stempel des Tobit InfoCenter Image Editors vom Host auf den Mobile Client kopiert und stehen dort im Tobit InfoCenter Image Editor zur Auswahl.
  • Durch einen automatischen, regelmässigen Refresh wurden unter Umständen bestimmte eingebundene Web-Sites auf der Portal Seite nicht mehr angezeigt.
  • Fehlfunktion bei der Auswahl einer anderen eMail Adresse, als die vorgeschlagene, behoben.
  • Fehlfunktion bei der Neuinstallation des Tobit InfoCenter Mobil nach Installation des Service Pack 1a behoben.
  • Design Änderungen der Symbolleisten unter Windows XP.
  • Beim Löschen externer Nachrichten sind die erweiterten Funktionen (»Absender blockieren«, etc.) nur noch verfügbar, wenn der User das Recht hat Verteilregeln anzulegen (User Konfiguration).
  • Neues User Archive: »Anwendungen«. Beliebige Anwendungen können als Shortcut in dieses Archive gezogen werden und erscheinen dann in einer eigenen Symbolleiste.
  • Fehlfunktion bei der Auswahl des Versandzeitpunktes im Nachrichten Editor behoben.
  • Im Tobit InfoCenter Image Editor kann das aktuell geöffnete Fax nun über den neuen »Löschen« Button direkt gelöscht werden.
  • Bei der Serienfunktion können nun auf Wunsch die nachträglich vorgenommenen, fehlenden Adressdaten direkt in der Original Adresse gespeichert werden.
  • Aktualisierte Druckertreiber (Tobit FaxWare, Tobit Image Messaging) für den Betrieb unter Windows Server 2003.
