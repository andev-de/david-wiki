Q-102.559
Wiederherstellung der DvISE Installation nach Problem mit der Systemintegrität


Frage
Problem Während der Installation des aktuellen Service Packs für die DvISE Installation wurde die Meldung ausgegeben, dass ein Zugriff auf Dateien im Verzeichnis »DAVID\Z$BACKUP« nicht möglich wäre. Mit dem »Ignorieren«-Button wurde die Installation fortgesetzt.
Bei erneuten Versuchen das aktuelle Service Pack zu installieren, wurden weitere Meldungen ausgegeben, z.B. dass kein David 6.6 vorliegt.
Der Versuch, das ursprüngliche Service Pack wieder einzuspielen oder die Uninstall Routine aus dem Verzeichnis Z$BACKUP aufzurufen, scheiterten.

Wie können solche Meldungen zustande kommen und wie kann das DvISE System wiederhergestellt werden, da der DvISE Service Layer nicht mehr gestartet werden kann und Hinweise auf unterschiedliche Versionen der Systemdateien gegeben sind?
Produkt David
Priorität dringend
Effekt kritisch
Datum 24.10.2002

Antwort
Solche Probleme können im Zusammenhang mit der Systemintegrität und/oder der Rechtevergabe stehen. Im weiteren Verlauf der Dokumentation wird hierauf detaillierter eingegangen.

Folgende Vorgehensweise empfiehlt sich, um das DvISE System mit dem aktuellsten Service Pack wieder in den einwandfreien Zustand zu befördern:

  1. Zunächst empfiehlt es sich, dass Sie eine Datensicherung mittels MKINST32.EXE durchführen, um Inhalte des Tobit Archive Systems zu sichern, z.B. die Benutzerarchive, Zentraleingang, usw.
    Nutzen Sie hierzu die Hilfestellungen der folgenden KnowledgeBase Artikel:
    • Q-100.279 (Der Archivepacker MKINST32.EXE)
    • Q-100.075 (nur Punkt 2 und 3 der Anleitung)
  2. Falls noch nicht geschehen, notieren Sie sich alle relevanten Informationen insoweit vorhanden, wie z.B. Vergabe der eMail Adressen, Faxnummerzuweisungen,Einstellungsoptionen der DvISE Dienste, usw.
    Hier empfiehlt es sich generell eine eigene Kurzdokumentation anzulegen, um stets alle relevanten Informationen schnell zur Verfügung zu haben.
  3. Nach manueller Deaktivierung aller DvISE Dienste sichern Sie die komplette David-Verzeichnisstruktur auf einen externen Datenträger, z.B. Bandsicherung, CDR, etc., um eine zusätzliche Datensicherung zur Hand zu haben.
  4. Es empfiehlt sich ebenfalls, vor der einer Deinstallation des DvISE Systems die lokalen Tobit InfoCenter Installationen zu deinstallieren, da für die Deinstallation Abhängigkeiten zum Server bestehen.
  5. Als Letztes deinstallieren Sie nun über die Uninstallroutine das DvISE System.
    Ist dies widererwartend nicht möglich, und auch nur dann, notieren Sie sich alle in der Windows Dienstesteuerung eingetragenen DvISE Dienste und starten Sie auf dem Server den DOS Prompt:
    • Wechseln Sie in das Verzeichnis »DAVID\APPS\...\CODE«, wobei "..." für den jeweiligen Dienst steht und führen Sie dort die jeweilige EXE Datei, z.B. POSTMAN.EXE mit der Option "-u" aus. Beispiele:
      POSTMAN -u
      DSERVER -u
      usw.
    • Im Verzeichnis »DAVID\CODE« führen Sie SL.EXE mit dem Prompt »SL -u« aus, um als letztes den Dienst »DvISE Service Layer« aus der Dienstesteuerung zu entfernen.
    • Nennen Sie das Verzeichnis »DAVID« um, z.B. »DAVID_OLD«.
    • Eine Entfernung von Registry-Einträgen ist nicht erforderlich, da diese bei einer erneuten Installation neu eingetragen werden.
  6. Führen Sie nun einen Serverneustart durch.
  7. Installieren Sie nun von Ihrer neuesten Produkt-CD das DvISE System erneut und führen Sie anschließend alle Schritte zur Grundeinrichtung durch. Hierzu gehören folgende Punkte in genannter Reihenfolge:
    • Anmeldung als Administrator am Server
      Sehen Sie davon ab, spezielle Service-User zu verwenden, um sicherzustellen, insbesondere bei Active Directory Umgebungen, dass der Benutzer definitiv alle Rechte hat.
    • Starten der Installation von der Produkt-CD
    • Sicherstellung der korrekten Rechtevergabe
      Verwenden Sie hierzu die Hinweise im
      KnowledgeBase Article Q-100.212.
    • Grundkonfiguration via DvISE Administrator
      Hierzu eignet sich insbesondere die Verwendung der eigenen Kurzdokumentation, wie bereits vorab geschildert.
    • Grundkonfiguration des Betriebssystems
      Insbesondere in einer Active Directory Umgebung empfiehlt es sich, über ein Script ein einheitliches Laufwerksmapping für das Serververzeichnis »DAVID« auf allen Clients sicherzustellen. Z.B. über eine Grouppolicy können Sie dies verwirklichen, weisen Sie das Script optimaler Weise benutzerspezifisch zu und nicht clientspezifisch.
      In einer nicht Active Directory Umgebung kann solch ein Script auch über Autostart auf den Clients implementiert werden.
      Beispiel für eine Script zum Laufwerksmapping:
      »net use o: \\servername\david«
    • Testen der Funktionalität
      Prüfen Sie die Funktion aller benötigen Dienste mittels einem Client und korrigieren Sie gegebenfalls festgestellte Irritationen, z.B. falsches Passwort für POP3 Postfächer.
  8. Sollten Sie in der Vergangenheit Probleme mit der Integrität von heruntergeladenen gepackten Dateien haben, so prüfen Sie zunächst (nach Download des aktuellsten Service Pack von der Club Tobit.com Site) das aktuellste Service Pack für das DvISE System manuell mittels z.B. dem Packer "WINRAR" oder "WINZIP":
    • Hierzu öffnen Sie die gepackte Datei mittels einem dieser Tools und entpacken manuell den Inhalt in ein temporäres Verzeichnis.
    • Gelingt diese Operation einwandfrei, so ist die Integrität gewährleistet und sie können die Executable normal zum Update der DvISE Installation starten.
    • Gelingt dies nicht, so laden Sie die Datei erneut von einer anderen Workstation runter, wo Sie den aktuellsten Internet Explorer installiert haben.
      Hinweis:
      Teilweise ist die Integrität auch abhängig von dem vorliegendem Dateisystem, der Fragmentierung oder Defekten der Festplatte.
  9. Nach erfolgreicher Installation des aktuellsten Service Packs und Überprüfung der Grundfunktionalität des DvISE Systems mit einem Testclient, führen Sie alle weiteren Schritte zur Installation des Tobit InfoCenters auf anderen Clients durch.
