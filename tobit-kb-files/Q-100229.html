Q-100.229
Konfiguration der Server Locator Services (SLS) bei Nutzung eines DSL-Routers


Frage
Problem Konfiguration der Server Locator Services (SLS) beim Einsatz eines DSL-Routers.
Produkt Server Locator Services
Priorität normal
Effekt normal
Datum 6.03.2007

Antwort
Die Server Locator Services for David (SLS) ermöglichen die ständige Online-Präsenz sowie den direkten Zugriff auf Ihren David Server, auch wenn dieser nicht über eine Standleitung mit dem Internet verbunden ist und somit nicht über eine statische IP-Adresse verfügt. Da Computer im Internet ausschließlich über ihre IP-Adresse identifiziert werden, ist der direkte Zugriff auf die Informationen eines Rechners mit wechselnder IP-Adresse üblicherweise praktisch unmöglich.

Bei Nutzung der Server Locator Services wird jede Änderung der IP-Adresse Ihres David Servers automatisch erkannt und dem DNS-Server bei Tobit Software mitgeteilt. Dieser von Tobit Software zentral betriebene DNS Server verwaltet die DNS Einträge sämtlicher Nutzer der Server Locator Services. Erfolgt nun eine Anfrage an Ihre Domäne, z.B. über einen Web-oder WAP-Browser, wird diesem vom DNS-Server stets die aktuelle IP-Adresse Ihres David Servers übermittelt. Auf diese Weise ist der David Server über das Internet ständig erreichbar und verhält sich praktisch so, als wäre die Internetanbindung über eine Standleitung mit statischer IP-Adresse realisiert.

Grundlegend ist es wichtig das die derzeit, durch die Einwahl bei Ihrem Internetprovider, öffentliche IP-Adresse ermittelt wird. Der Router sollte so konfiguriert werden, dass die Anfragen ( HTTP/SMTP/POP3/ NNTP oder z.B. IMAP4 ) aus dem Internet an die lokale IP des David-Servers verteilt werden. (Portmapping)

Um diese IP zu ermitteln gibt es zwei Möglichkeiten:

  1. Sie tragen in die POSTMAN.INI (»DAVID\APPS\POSTMAN\CODE«) den Eintrag "SLSInterval = 180" ein.
    Nach anschließendem Neustart des Dienstes »David PostMan« wird nun alle 180 Sekunden eine ATRN-Anfrage an den SLS-Server von Tobit Software gestellt. Hierdurch wird die IP des Routers ermittelt. Ist die IP unverändert, wird die Verbindung sofort wieder abgebaut, ansonsten überträgt der PostMan die aktuelle IP.

  2. Die zweite Möglichkeit ist sehr viel effizienter, da eine Verbindung nur bei einer IP-Änderung aufgebaut wird.
    Im Verzeichnis »DAVID\APPS\POSTMAN\CODE« finden Sie das Tool "David IP Check" (IPCHECK.EXE). Es überprüft die jeweils aktuelle IP-Adresse, unter der die Internetverbindung aufgebaut wurde, und veranlasst, sobald eine Änderung festgestellt wird, den PostMan, eine Verbindung zum SLS-Server von Tobit Software aufzubauen.

    Dieses Tool muss nicht am David-Server ausgeführt werden, sondern übernimmt auch auf einer Workstation diese Aufgabe. Sie können unter Options den David-Server angeben. Bitte beachten Sie, dass Sie den David-Server in UNC-Schreibweise angeben:

    Windows NT/2000 Server: \\Servername
    NetWare Server: \\Servername\Volume

    Über die Funktion »Update im Intervall durchführen« können Sie den Verbindungsaufbau zum DNS-Server in bestimmten Intervallen erzwingen. Dies hat den selben Effekt wie der Eintrag in der POSTMAN.INI.

Im Verzeichnis »DAVID\APPS\POSTMAN\CODE« befindet sich das SLS.PDF, wo weitere Informationen zum Thema SLS erläutert werden.
