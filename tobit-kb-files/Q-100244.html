Q-100.244
Grundlagen der Server Locator Services


Frage
Produkt Server Locator Services
Priorität keine
Effekt harmlos
Datum 14.07.2005

Antwort
Die Server Locator Services for DvISE (SLS) ermöglichen die ständige Online-Präsenz, sowie den direkten Zugriff auf Ihren DvISE Server, auch wenn dieser nicht über eine Standleitung mit dem Internet verbunden ist und somit nicht über eine statische IP-Adresse verfügt.

Da Computer im Internet ausschließlich über ihre IP-Adresse identifiziert werden, ist der direkte Zugriff auf die Informationen eines Rechners mit wechselnder IP-Adresse praktisch unmöglich.
Eine Änderung der IP-Adresse wird von den SLS automatisch erkannt und dem DNS-Server bei Tobit Software mitgeteilt. Dieser von Tobit Software zentral betriebene DNS Server verwaltet die DNS Einträge sämtlicher Nutzer der Server Locator Services. Erfolgt eine Anfrage über einen Web-oder WAP-Browser, wird diesem vom DNS-Server stets die aktuelle IP-Adresse Ihres DvISE Servers übermittelt. Auf diese Weise ist der DvISE Server über das Internet ständig erreichbar und verhält sich praktisch so, als wäre die Internetanbindung statisch.

Voraussetzungen
Folgende Voraussetzungen müssen Sie erfüllen, um die Server Locator Services nutzen zu können:

  1. Sie benötigen eins der folgenden DvISE Produkte:
    • David Professional
    • David DSL
    • David
    Wollen Sie bei Einsatz von David auch die Möglichkeiten des Fernzugriffs auf die eigenen Nachrichten nutzen, müssen Sie zusätzlich das Internet Access Module einsetzen, das den Web- und WAP-Server beinhaltet.
  2. Weiterhin müssen Sie eine "Always-Online-Verbindung" zum Internet bereitstellen, z.B. über eine Flatrate (wie DSL Flat).
  3. Für die Nutzung der SLS müssen Sie außerdem den Dienst »DvISE PostMan« auf dem DvISE Server einrichten.
    Aus diesem Grund wird der DvISE PostMan bei der SLS Installation automatisch gestartet, auch wenn dieser Dienst bisher nicht genutzt wurde.
Installation
Die Installation der Server Locator Services for DvISE geschieht in der Regel direkt bei der DvISE-Grundinstallation bzw. bei einer Service Pack Installation ab SP2:
  1. Aktivieren Sie die Option »Server Locator Services for DvISE installieren«.
  2. Im folgenden Dialogfenster werden Sie zur Eingabe eines Kennwortes aufgefordert.
    Tragen Sie hier den dritten Zeichenblock Ihrer Systemlizenz ein. Diese Zeichenkette ist gleichzeitig Ihr Kennwort für das Kundenforum »Club tobit.com«.
    Falls Sie sich bereits im Club tobit.com registriert haben und eventuell auch schon eine Passwortänderung vorgenommen haben, so tragen Sie an dieser Stelle das aktuell gültige Passwort ein. Dieses Passwort dient zur Authentifizierung für die regelmäßige Übermittlung der IP-Adresse an den DNS-Server von Tobit Software.
  3. Ihre Zugangsdaten werden automatisch generiert und im Tobit Archive System auf Ihrem DvISE Server unterhalb des Archives »System -> DvISE -> Domänen -> Authentifizierungs Domänen« gespeichert.
  4. Ihr aktueller Domain Name setzt sich aus Ihrer Site-ID (die ersten 10 Stellen Ihrer Systemlizenz) und der Domain ".tobit.net" zusammen, also z.B. 12345-67890.tobit.net.
  5. Dieser Domain Name wird in der PostMan Konfiguration unter »Datenbanken -> Gültige Domain Namen« eingetragen.
  6. Im Club tobit.com können Sie zusätzlich zu diesem automatisch vergebenen Domain Namen eine weitere freiwählbare Sub-Domain von tobit.net für sich einrichten, also z.B. »mustermann.tobit.net«, sofern diese nicht bereits anderweitig vergeben wurde.
  7. Beachten Sie hierbei, dass dieser Domain Name dann ebenfalls in der DvISE Konfiguration (z.B. bei WebBox und PostMan) eingetragen werden muss.
  8. Die Angaben im DvISE Administrator unterhalb von »DvISE PostMan -> Datenbanken -> AUTH Domain Namen« bleiben hiervon unberührt. Ein Authentifizierung findet ausschließlich durch den einen Eintrag mit der Site-ID.tobit.net statt.
  9. Die Angaben im DvISE Administrator unterhalb von »DvISE PostMan -> Datenbanken -> Gültige Domain Namen« müssen Sie im Falle einer oder mehrerer zusätzlicher Sub-Domains entsprechend ergänzen.
Betrieb
Der Betrieb der Server Locator Services for DvISE geschieht praktisch vollkommen im Hintergrund. Der DvISE PostMan prüft die IP-Adresse des DvISE Servers. Sobald er eine Änderung der IP-Adresse feststellt, übermittelt er die neue Adresse automatisch an den DNS-Server von Tobit Software. Dieser stellt somit die ständige Erreichbarkeit Ihres DvISE Servers über das Internet sicher. Das gilt sowohl für Anfragen von Web- oder WAP-Browsern, als auch für den Austausch von eMails.

Somit kann der eigene DvISE Server als eigenständiger Mail-Server betrieben werden, ohne dass Versand und Empfang von eMails über einen externen Provider abgewickelt werden müssen.

Damit der Abgleich der IP-Adresse mit dem DNS Server sicher ist, verwendet PostMan für den Verbindungsaufbau die SMTP Authentifizierung (AUTH=CRAM-MD5). Die Zugangsdaten hierzu werden aus Ihrer Systemlizenz bestimmt und auf Ihrem DvISE Server in der PostMan Konfiguration unter dem Punkt »AUTH Domain Namen« eingetragen.

Hierfür wird Ihre Site-ID verwendet. Diese entspricht den ersten 10 Stellen Ihrer Systemlizenz. Als Passwort wird Ihr Passwort für den Club tobit.com eingetragen, das normalerweise aus dem dritten Zeichenblock Ihrer Systemlizenz besteht.
Beachten Sie unbedingt, dass Sie jedes mal, wenn Sie unter www.tobit.com das Passwort für den Zugang zum Club tobit.com ändern, dieses neue Passwort auch im SLS Eintrag der PostMan Konfiguration unter »AUTH Domain Namen« eintragen müssen.

Über den Schalter »Erweitert -> Server Locator Services« in der PostMan Systemkonfiguration können die Server Locator Services aktiviert oder deaktiviert werden.

Sonderfälle
In folgenden Fällen kann der DvISE PostMan die IP-Adresse der Servers nicht eindeutig auslesen:

  1. Die Internetanbindung erfolgt über einen DSL Router
    Prüfen Sie hierfür bitte die Punkte im
    KnowledgeBase Article Q-100.229.
  2. Die Internet-Zugangssoftware befindet sich nicht auf dem DvISE Server.
    In diesem Fall ist für die Feststellung der IP-Adresse der Einsatz des zusätzlichen Utility Programms IPCHECK.EXE notwendig. Dieses Programm befindet sich im Verzeichnis »DAVID\APPS\POSTMAN\CODE\« des DvISE Servers.
    Es überprüft die jeweils aktuelle IP-Adresse, unter der die Internetverbindung aufgebaut wurde, und veranlasst, sobald eine Änderung festgestellt wird, den DvISE PostMan eine IP-Überprüfung durchzuführen. Dieser nimmt daraufhin den Abgleich mit dem DNS-Server von Tobit Software vor. Gehen Sie zur Einrichtung des Programms folgendermaßen vor:
    • Rufen Sie die Datei IPCHECK.EXE auf einem Windows Rechner auf. Daraufhin öffnet sich der Konfigurationsdialog und das Programm wird als Symbol in der Windows Systemmenüleiste dargestellt.
    • Auf dem Register »State« können Sie den Betrieb des Programms beobachtet.
    • Wählen Sie die Registerkarte »Options« und tragen Sie in das Eingabefeld »DvISE Server« den Namen Ihres DvISE Servers in UNC Schreibweise ein.
      Handelt es sich dabei um einen NetWare Server, so ist zusätzlich das Volume, auf dem sich Ihre DvISE Installation befindet, anzugeben. Beispiele:
      • Windows NT/2000 Server: \\Servername
      • NetWare Server: \\Servername\Volume
    • Über die Funktion »Update im Intervall durchführen« können Sie den Verbindungsaufbau zum DNS-Server in bestimmten Intervallen erzwingen.
      In diesem Fall wird die Verbindung auch dann hergestellt, wenn keine Änderung der IP-Adresse festgestellt wurde. Tragen Sie die Länge des gewünschten Intervalls in Sekunden ein. Im Normalfall ist hier jedoch kein Eintrag erforderlich.
    • Klicken Sie auf »Speichern«, um die aktuellen Einstellungen zu sichern.
    • Wählen Sie die Option »Verbergen«, wenn Sie wünschen,dass das Programm im Hintergrund arbeitet und das Dialogfenster ausgeblendet wird. In diesem Fall kann das Dialogfenster jederzeit über einen Klick auf das Symbol in der Systemmenüleiste wieder aktiviert werden.
