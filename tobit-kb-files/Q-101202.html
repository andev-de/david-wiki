Q-101.202
DavidMX - Errorcode 80004005 & 80071392 in der SETUP.LOG


Frage
Problem Wenn aus Outlook ein Fax versendet werden soll, klappt dies insoweit, dass der Adress Space »[DVFAX:1234567]« zwar ausgewertet wird, aber die Meldung erscheint, dass die eMailadresse nicht gefunden werden konnte.

In der SETUP.LOG (»\Program Files\David MX\Setup«) stehen folgende Einträge:

Dv4ECleanADs: HrDeleteGatewayObject failed with errocode 80004005
und
CreateGatewayClass: CreateDSObject failed with errorcode 80071392
InstallGateway: CreateGatewayClass failed
Priorität normal
Effekt kritisch
Datum 26.06.2002

Antwort
Diese Solution geht davon aus, dass Sie David MX und Exchange 2000 auf einem Rechner installiert haben.

Voraussetzung für eine Installation des DvISE Connectors for Microsoft Exchange 2000 auf einem Exchange-Server ist folgender Softwarestand:

  • Exchange 2000 SP2, David MX und DvISE Connector auf einem Rechner
  • DvISE Connector NUR nachträglich von der CD installieren, da es sich hierbei um eine angepasste Connectorvariante handelt.
    Sie dürfen auf keinem Fall den DvISE Connector for MS Exchange 2000 (SP2) von unserer Downloadsite nutzen, wenn Sie David MX einsetzen!
Voraussetzung des installierenden Benutzers:
  • Es sollte der Administrator der Domäne sein, der den Connector installiert!
  • Er muss Miglied folgender Gruppen sein:
    • Administratoren
    • Domänen-Admins
    • Domänen-Benutzer
    • Organisations-Admins
    • Richtlinien-Ersteller-Besitzer
    • Schema-Admins
Voraussetzung in der ADS - Schemamaster:
  • Die Installation des Connectors sollte optimaler Weise zusammen mit Exchange auf dem ersten Domaincontroller (DC) sich befinden und dessen Schemamasterfunktionalität (FSMO) sollten allein auf Ihn definiert sein.
  • Mehr hierzu finden Sie im KnowledgeBase Artikel Q-100.584
Voraussetzung in der ADS - Sicherheitsrichtlinien für Domänen:
  • Im Hive »Sicherheitseinstellungen -> Lokale Richtlinien -> Zuweisen von Benutzerrechten« sollte im Eintrag »Als Dienst anmelden« die Benutzer »Administrator« und »DVU-Servername« eingetragen sein.
  • Im Hive »Sicherheitseinstellungen -> Kontorichtlinien« sollten alle Einträge undefiniert sein.
    Sind denoch Definitionen vorhanden, müssen diese insoweit kontrolliert werden, dass die Sicherheitsaspekte die Kennwortkomplexität nicht zu stark einschnürren.
  • Mehr hierzu finden Sie im KnowledgeBase Article Q-100.506
Erneute Vorgehensweise nach absoluter Sicherstellung der genannten Punkte:
  1. Wenn Sie all diese Punkte sichergestellt haben, müssen Sie den DvISE Connector über »Start -> Programme -> DvISE Connector for Microsoft Exchange -> Uninstall DvISE Connector« zunächst deinstallieren.
  2. Jetzt starten Sie das Setup für den DvISE Connector via der CD nochmals und bei den Eintragungen berücksichtigen Sie folgende Aspekte:
    • Exchange Server & ADS Server = »Servername«
      (sollte erkannt werden vom Setup)
    • Default Vorgabe für Speichergruppe & Postfachspeicher beibehalten!
    • Empänger unverteilter Nachrichten: »Administator@IhreDomäne«
      (sollte automatisch vorgegeben sein, ansonsten hat der Admin kein Exchange Postfach oder der Connect zum Exchange Server ist nicht gegeben!)
    • Dienstanmeldender Benutzer: »NETBIOSNAME\Administrator«
  3. Nach der Installation beenden Sie zunächst alle DvISE Dienste und alle Exchange Dienste.
  4. Starten Sie jetzt als Erstes die Exchange Dienste, gefolgt von den DvISE Diensten (PostMan, SL, TLD001, usw. die gewünscht sind) und zuletzt den DvISE Connector Dienst.
Die Meldung erscheint immer noch...
Wenn Sie diese Punkte alle befolgt haben, sollte Sie nach der erneuten Installation innerhalb der SETUP.LOG den Eintrag »InstallGateway: successful« (letzte Zeile) finden, der Ihnen die erfolgreiche Installation des Gateways bescheinigt.

Sollten Sie danach aber noch die Meldung »Ein Programm versucht auf Ihre Outlook gespeicherten E-Mail-Adressen...«, dann liegt dies daran, dass Sie entweder Outlook 2000 (SR1) oder Outlook XP einsetzen. In diesen Versionen prüft Outlook, ob andere Applikationen über die CDO Schnittstelle zugreifen und warnt hiervor.
Um diese Meldung zu deaktivieren, prüfen Sie bitte die Punkte im KnowledgeBase Article Q-100.038.
