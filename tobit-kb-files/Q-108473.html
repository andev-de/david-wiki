Q-108.473
Installation und Betrieb eines EIB TLDs


Frage
Problem Ein Extended TLD mit dem PlugIn »EIB« soll installiert und eingerichtet werden.
Produkt TLD (Gebäudesteuerung)
Priorität keine
Effekt harmlos
Datum 3.07.2006

Antwort
Dieser Knowledge Base Artikel erläutert die Einrichtung von David für den Betrieb innerhalb von EIB-Systemen (Europäischer Installationsbus). Hierzu muss zunächst ein geeigneter Port eingerichtet und anschließend für seinen Bestimmungszweck konfiguriert werden. Führen Sie dazu die beschriebenen Schritte in der hier angegebenen Reihenfolge durch.

Einrichten eines EIB Ports
Für die Einbindung in das EIB-System sollte ein separater Port verwendet werden. Gehen Sie zur Einrichtung dieses Ports folgendermaßen vor:

  • Starten des Port Setups
    Öffnen Sie im David Administrator das Menü »Konfiguration -> Ports« und klicken Sie dann in der Symbolleiste auf das Icon »Neu«. Daraufhin startet das Setup-Programm. Klicken Sie auf »Hinzufügen«, um einen neuen Port einzurichten.

  • Auswahl des Port-Typs
    Wählen Sie aus der dann angezeigten Liste den Typ »Extended (z.B. Automation)« und bestätigen Sie Ihre Auswahl mit »Weiter«.

  • Auswahl des PlugIns
    Im nächsten Dialogfenster ist das für den neuen Port zu nutzende PlugIn anzugeben. Da ein EIB Port erstellt werden soll, wählen Sie an dieserStelle den Eintrag »Automation - EIB« aus. Klicken Sie anschließend auf »Weiter«, um Ihre Auswahl zu bestätigen und mit dem Setup fortzufahren.

  • Benennen des Ports
    Vergeben Sie eine eindeutige Bezeichnung für den neuen Port (z.B. »Automation - EIB«) und klicken Sie wiederum auf »Weiter«.

  • Bestimmen der Schnittstelle und Abschluss
    Wählen Sie schließlich die Schnittstelle aus, über die der DvISE Server mit dem EIB-Busankoppler verbunden ist. Im Normalfall erfolgt die Verbindung über eine der COM-Schnittstellen per seriellem Schnittstellenkabel (RS232). Bei Bedarf kann diese Einstellung auch nachträglich noch geändert werden. Klicken Sie auf »Fertig stellen«, um Ihre Auswahl zu bestätigen und das Setup abzuschließen. Damit ist die Port-Installation beendet und das Setup-Programm schließt sich eigenständig.

Das Starten eines EIB-Ports ist nur möglich, wenn eine Verbindung zum Busankoppler und somit zum Elektrobussystem besteht.

Anpassen der Port-Konfiguration
Nach der Installation des neuen Ports muss dessen Konfiguration angepasst werden. Wählen Sie dazu im David Administrator den Menüpunkt »Konfiguration -> Ports« und öffnen Sie den Konfigurationsdialog des soeben erstellten Ports durch einen Doppelklick auf den zugehörigen Listeneintrag. Nehmen Sie auf der Registerkarte »Allgemein« folgende Einstellungen vor:

  • TLDSubType
    Bestimmen Sie hier eine Bezeichnung, unter der dieser Port direkt adressierbar sein soll. Standardmäßig wird die Bezeichnung »EIB« genutzt. Die Verwendung dieses TLDSubType ist sowohl im Adressfeld des Nachrichten Editors als auch in Verbindung mit DvISE Befehlen (beispielsweise @@NUMMER@@) möglich. Geben Sie als Empfängeradresse zunächst diese Bezeichnung an, direkt gefolgt voneinem Doppelpunkt und der eigentlichen Empfängeradresse.

  • Erweiterung der Sendeservices
    Aktivieren Sie die Funktion »Diesen Port mit der folgenden Bezeichnung im Nachrichten Editor des InfoCenters unter »Sendeservices« zur Auswahl anbieten«, wenn Sie wünschen, dass dieser Port über einen eigenen Eintrag in der Liste »Sendeservices« im Nachrichten Editor des David InfoCenters direkt ansprechbar sein soll. Vergeben Sie dazu in dem darunter stehenden Feld»Bezeichnung« einen Namen, der den verfügbaren Sendeservices hinzugefügt werden soll. Bei der Erstellung von Aufträgen, die über diesen Port abgewickelt werden sollen, ist dann als Sendeservice dieser Eintrag zu wählen.

Erweiterte Einstellungen
Um über den soeben erstellten Port mit dem EIB-System kommunizieren zu können, sind einige zusätzliche Einstellungen vorzunehmen. Klicken Sie dazu auf die Schaltfläche »Extended TLD...«. Daraufhin öffnet sich ein weitererKonfigurationsdialog.

Hier können Sie die Schnittstelleneinstellung prüfen und bei Bedarf ändern. Wenn Sie wünschen, dass bei jedem Starten des Ports die Werte sämtlicher Aktoren im State Archive aktualisiert werden sollen, aktivieren Sie das zugehörige Kontrollkästchen. In diesem Fall wird beim Starten des Ports eine Abfrage an jeden Aktor gesendet und der daraufhin erhaltene Rückgabewert in das State Archive übernommen. Auf diese Weise liefert das Automation Portal nach einem Neustart des Ports immer die aktuell gültigen Werte.

Nutzen Sie die Option »Beim Starten dieWerte aller Aktoren im State Archive aktualisieren« nur, wenn sichergestellt ist, dass bei der Parametrierung über die ETS (EIB Tool Software) in den Objekt-Eigenschaften der Aktoren das Flag »Lesen« gesetzt ist. Andernfalls können durch fehlende Rückmeldungen von Aktoren, für die dieses Flag nicht gesetzt ist, spürbare Zeitverzögerungen auftreten.

In der darunter stehenden Liste können Sie beliebig viele Gruppen Adressen eintragen, deren Werte beim Start des Ports abgefragt werden sollen. Klicken Sie hier auf »Hinzufügen«, um einen neuen Eintrag zu erstellen. Tragen Sie in das Feld »Name« eine beliebige Bezeichnung ein. Mit den Feldern »Anfangsadresse« und »Letzte Adresse« können Sie den abzufragenden Bereich der EIB-Gruppen-Adresse definieren.

Betrieb des EIB Ports
Nach dem Abschluss der Konfiguration ist der Port betriebsbereit. Aufträge für diesen Port können über die API- oder über die FileScan-Services (z.B. per Text-Datei im ASCII-Format) erzeugt werden, wobei in der Adressierung der in der Port-Konfiguration definierte TLDSubType zu verwenden ist.

Eine gültige Zieladresse setzt sich folgendermaßen zusammen:

    TLDSubType:Gruppen Adresse

Beispiel:

    EIB:3/4/66

Die Erstellung von Aufträgen ist natürlich auch über den Nachrichten Editor des David InfoCenters möglich. Hier kann die Adressierung auch komfortabel über die Auswahl des betreffenden Sendeservices geschehen, wenn dieser zuvor in der Port-Konfiguration definiert wurde.

In dem System-Archive »Archive/System/DvISE/ Automation/State« werden alle Meldungen, die der DvISE Server über den Beckhoff Port vom Bus empfängt, verwaltet. Hier existiert für jede der bisher angesprochenen Bus-Adressen ein Eintrag, der die jeweils zugehörige Adresse als Bezeichnung trägt. Zur leichteren Unterscheidung oder zum schnellen Auffinden bestimmter Adressen kann in das Feld »Bezeichnung« auch ein beliebiger Text geschrieben werden. Die Spalte »Adresse« beinhaltet in jedem Fall die korrekte Adresse. Vergeben Sie für wichtige Adressen, auf die ein schneller Zugriff möglich sein soll, manuell die Priorität »hoch«. Diese Adressen werden dann in einem gesondertenBereich des Automation Portals von David aufgelistet.

Sobald der Port Aufträge bearbeitet, kann die System-Kommunikation im Port Monitor überwacht werden. Wählen Sie dazu im David Administrator den Menüpunkt »Monitor -> Ports« und öffnen Sie den Monitor des gewünschten Ports per Doppelklick auf den zugehörigen Listeneintrag. Wechseln Sie hier auf das Register »Communication« um den Verlauf der Kommunikation zu verfolgen.
