Service Pack 30-Jun-2003

Code History (NetWare)

Die folgenden Neuerungen und Bugfixes sind im Service Pack enthalten:

  1. DvISE Service Layer
  2. DvISE PostMan
  3. DvISE Grabbing Server
  4. DvISE WebBox
  5. Tobit InfoCenter Web
  6. DvISE Replication Services
  7. DvISE Discussion Server
  8. DvISE TLD für ISDN Karten
  9. DvISE Mail Access Server
  10. Tobit InfoCenter
  11. DvISE Communication Controller
  12. Aktuelle Dateiversionen

1. DvISE Service Layer

  • Der Wert der Variable @ @GET SYS_LAUFENDENUMMER entspricht nun der Variable "Laufende Nummer", die im Nachrichten Editor ausgewählt werden kann. Bisher waren diese unterschiedlich.
  • ECT funktioniert nun auch ohne Erweiterungslizenz ("Extended Telephony Module for DvISE").
  • Möglicher DvISE Service Layer Absturz beim Virus Scan bestimmter eMails wurde behoben.
  • Anpassung des VCF-Formates (Adresskarte beim eMail Versand/Empfang) aus Kompatibilitätsgründen.
  • Optimierung der Datenübertragung zum Tobit InfoCenter (ca. 10 fache Steigerung).
  • Die abonnierten News Groups Archives der DvISE Benutzer werden nun unterhalb von »Archive -> Resourcen -> News« im Tobit InfoCenter angezeigt.
  • Optimierung bei der Abarbeitung von Faxaufträgen bis zur Konvertierung.
  • Fehlerhafte Daten zur vorhandenen Speicherkapazität in der "Finish Purging Archive" Nachricht (Archive "Fehlermeldungen").
  • In der Meldung »Virus Engine started« (Archive »Fehlermeldungen«) wurde die Anzahl der Viren, die mit den aktuellen Signaturen gefunden werden, nicht korrekt angezeigt.
  • Der DvISE Service Layer lief nicht weiter, wenn nach zu wenig freiem Speicherplatz wieder genügend Speicherplatz zur Verfügung stand.
2. DvISE PostMan
  • Erweiterter SPAM Filter (»Datenbanken -> Nicht erlaubte Absenderadressen«). Es können nun Einstellungen für das Ablehnen/Löschen von SPAM Mails vorgenommen werden. Wahlweise kann nach Absender, Empfänger und/oder Textinhalt gefiltert werden. Die entsprechenden eMails können direkt gelöscht oder in ein anzugebendes Archive verschoben werden. Im Error Archive erfolgt ein entsprechender Eintrag (»Unallowed message«).
  • Bei der Konvertierung von Winmail Dateien (Dateiendung *.TNF) kann es nicht mehr zum Stillstand von DvISE PostMan kommen.
  • Beim Entladen von PostMan werden nun alle Resourcen freigegeben (auch bei der Einstellung »nur empfangen« oder »inaktiv«).
  • Kein ABEND mehr beim Löschen von Einträgen in den Datenbanken.
  • Beim Weiterleiten von eMails existiert die Nachricht im Falle weiterer Sendeversuche nur noch einmal auf dem Server (nur, wenn bei PostMan der Schalter »eMail Rundsendung« aktiviert ist).
  • Möglicher Fehler bei der Konvertierung bestimmter empfangener eMails.
3. DvISE Grabbing Server
  • Erweiterter SPAM Filter (siehe Hinweis zu "DvISE PostMan").
  • Empfangene Nachrichten werden nun generell als "extern" in der Eintragsliste markiert.
  • Bei der Konvertierung von Winmail Dateien (Dateiendung *.TNF) kann es nicht mehr zum Stillstand des Grabbing Servers kommen.
  • Möglicher Fehler bei der Konvertierung bestimmter empfangener eMails.
4. DvISE WebBox
  • Kein ABEND mehr möglich beim Browser Zugriff auf Kalender Archives mit alten Einträgen (Dokumenttyp noch nicht "Kalender").
  • Zusätzlicher Wert beim WebBox Parameter ##Archive /FSTR=##:
  • %dl oder %DL gibt nun das Jahr 4-Stellig aus (z.B. 2003, statt 03).
5. Tobit InfoCenter Web
  • Verbesserte Unterstützung von Macintosh Clients.
  • Überarbeitung der Monatsansicht, keine fehlerhafte Darstellung mehr möglich.
  • Kein falsches Eintragen von Terminen mehr möglich.
  • Einführung einer Versionsinfo ("Über..." Button).
  • Beim Öffnen von Einschreiben erscheint ein Hinweis, dass der Absender über das Öffnen informiert wird.
  • Der »Öffnen« Button ist nun auch bei eMails vorhanden.
  • Termine im Kalender können nun auch gelöscht werden.
6. DvISE Replication Services
  • Im "Replication Monitor" wurden übertragene Datenmengen >2 GB falsch angezeigt.
  • Einige Statusmeldungen der Replica (»Replica started«, »Replica ended«) wurden unter Umständen im Archive »Unverteilt« erstellt, statt im Archive »Fehlermeldungen«.
  • Anmerkungen auf Fax-Nachrichten ("Overlays") werden nun bei der Replizierung berücksichtigt.
  • Beim Löschen von Einträgen wird die "LastChangedTime" gesetzt.
  • Wenn eine versendete Nachricht "erneut eingetragen" wurde, konnte es bei anschließenden Replizierungen dazu kommen, dass die Nachricht auf beiden Seiten vervielfältigt wurde.
7. DvISE Discussion Server
  • Möglichen Absturz behoben, wenn Gruppenname geändert wurde und externe Newsclients an den alten Gruppennamen gesendet haben.
8. DvISE TLD für ISDN Karten
  • Für die Nutzung der DBSS (Diphone Based Synthetic Speech) wird nur noch die "Extended Telephony Module for DvISE" Erweiterungslizenz benötigt.

9. DvISE Mail Access Server

  • Optimierung des Logins.
  • Der DvISE Mail Access Server lässt sich jetzt schneller beenden.
  • Wenn bei einer Nachricht das Archive Flag (Optionen -> "Keine Bereinigung") gesetzt ist, löscht der Mail Access Server diese Nachricht nicht. Es kann also jetzt das Löschen einzelner Nachrichten verhindert werden.
10. Tobit InfoCenter
  • Fehlfunktion bei der Auswahl einer anderen eMail Adresse, als die vorgeschlagene, behoben.
  • Fehlfunktion bei der Neuinstallation des Tobit InfoCenter Mobil nach Installation des Service Pack 1a behoben.
  • Design Änderungen der Symbolleisten unter Windows XP.
  • Beim Löschen externer Nachrichten sind die "erweiterten" Funktionen (»Absender blockieren«, etc.) nur noch verfügbar, wenn der User das Recht hat Verteilregeln anzulegen (User Konfiguration).
  • Neues User Archive: »Anwendungen«. Beliebige Anwendungen können als Shortcut in dieses Archive gezogen werden und erscheinen dann in einer eigenen Symbolleiste.
  • Fehlfunktion bei der Auswahl des Versandzeitpunktes im Nachrichten Editor behoben.
    Im Tobit InfoCenter Image Editor kann das aktuell geöffnete Fax nun über den neuen »Löschen«-Button direkt gelöscht werden.
  • Bei der Serienfunktion können nun auf Wunsch die nachträglich vorgenommenen, fehlenden Adressdaten direkt in der Original Adresse gespeichert werden.
  • Im Fenster »Kontakte« und in der Adressauswahl werden die Namen der angemeldeten DvISE Benutzer (Tobit InfoCenter gestartet) farblich markiert (helles/dunkles Icon) bzw. mit dem Vermerk »angemeldet« gekennzeichnet.
  • Bei Adressen werden nun alle Datenfelder in der Eintragsliste angezeigt.
  • Keine Schutzverletzung mehr beim Ändern der Symbolleisten-Konfiguration.
  • Mögliche Darstellungsfehler im Kalender (Tagesansicht) behoben, wenn zur gleichen Zeit mehrere Termine eingetragen sind.
  • Beim Zwischenspeichern einer Faxvorlage im Entwurfs-Archive kann die Vorlage beim anschließenden Öffnen weiter bearbeitet werden.
  • Optimierung der Aufgabenverwaltung.
  • Einstellungen für das Aufgaben-Archive können unter »Einstellungen -> Diverses« vorgenommen werden.
  • Systemsicherheit: Wenn HTML Mails @@ Befehle enthalten, erfolgt beim Weiterleiten/Antworten dieser eMails ein entsprechender Sicherheitshinweis. Es wird dann in den "Reinen-Text" Modus gewechselt und die Befehle werden entfernt.
  • Archives können nun nicht mehr ohne entsprechende Rechte des Users verschoben werden.
  • Anpassung des VCF-Formates (Adresskarte beim eMail Versand/Empfang) aus Kompatibilitätsgründen.
  • Kein Fehlerverhalten (z.B. Schutzverletzung) mehr möglich beim Weiterleiten von Faxen aus dem Tobit InfoCenter Image Editor.
  • Die eingestellte Schriftfarbe wird nun in allen Editoren übernommen (Nachrichten Editor, Aufgaben Editor, etc.).
  • Verbesserung des Ausdrucks der Monatsansicht, die Tagesbezeichnungen (Montag, Dienstag, etc.) konnten u.U. abgeschnitten werden.
  • Aufgaben, die über das Feld »Hier klicken, um Aufgabe zu erstellen« erstellt wurden, werden nun im HTML Format erstellt. Hier kann nun auch per TAB in die Spalten gewechselt werden.
  • Bei manuell eingetragenen Fortschritt wurde anschließend der Wert über die Cursor-Tasten nicht richtig ausgelesen (76% -> 50%).
  • Beim Erstellen von Serien-eMails an CC, Bcc Adressen konnte es zu Fehlern kommen (z.B. doppelte Nachrichten).
  • Beim Erstellen einer neuen Aufgabe per Drag´nDrop wird die Original-Nachricht nicht mehr gelöscht, wenn während des Drag´nDrop "abgebrochen" wird.
  • Das Wechseln zwischen den Feldern AN, CC, BCC, BETREFF und Text ist nun auch mit der ENTER Taste möglich, bisher nur mit der TAB Taste.
  • Beim Löschen einer Client-Verteilung werden nun alle Einstellungen gelöscht. Bisher konnte es in einigen Fällen beim erneuten Erstellen zu Fehlern kommen.
  • Über die Systemsteuerung können eigene Sounds für den Tobit InfoCenter Notifier eingestellt werden.
  • Für Nachrichten, die über den Sendeservice »Drucker« erstellt werden, wird nun ein Eintrag im Ausgangs-Archive erstellt. Das AN Feld enthält dabei den Namen des gewählten Druckers.
  • Bei der Einstellung »Nachrichtenoptionen vor dem Versenden anzeigen« wird dieser Dialog nun beim Sendeservice »Drucker« nicht mehr angezeigt.
  • Bei einem Gruppenkalender wird in der Vorschauleiste von Terminen der Archivename des jeweiligen Kalenders angezeigt.
  • Kein Fehlerverhalten mehr beim Suchen in "Reinen Text" eMails. Unter Umständen wurde der Suchtext nicht gefunden.
  • Adressen können nun auch über den Client in HTML-Formatierung gedruckt werden: rechte Maustaste auf einem Adresseintrag, »lokaler Drucker« wählen. Hierzu liegt ein entsprechendes Template (ADDRESS.PRN) im Verzeichnis »DAVID\CLIENTS\WINDOWS«.
  • Beim Erstellen eines Termins wird nun der eingestellte Standardzeichensatz (Nachrichten Editor) berücksichtigt.
  • Die Geschwindigkeit beim Einlesen der Eintragslisten wurde verbessert.
  • Beim Import von Outlook-Adressen (Menü »Werkzeuge -> Outlook Import«) kann nun über den Button »Optionen« die Reihenfolge der Felder Vorname und Nachname bestimmt werden.
  • Die eingestellte Ansicht für den rechten Rand wird nun gespeichert und gilt für alle Editoren.
  • Die eingestellten Initialen (Nachrichten Editor) werden nun auch in der Windows Registry benutzerbezogen gespeichert.
  • Beim Löschen externer Nachrichten erscheint ein neuer Dialog, mit dem die entsprechende Nachricht entweder wie bisher in den Papierkorb verschoben werden, oder automatisch eine entsprechende Verteilregel angelegt werden kann. Im letzten Fall können weitere Nachrichten von dem angegebenen Absender bzw. an den angegebenen Empfänger wahlweise direkt gelöscht oder in den Papierkorb verschoben werden.
  • Neue Option bei der Client-Verteilung: "Sonstige (Löschen oder kennzeichnen)". Bewirkt, dass verteilte Einträge automatisch farbig markiert werden, eine Markierung erhalten und/oder in den Papierkorb verschoben werden.
  • Unter "Optionen/Einstellungen/Diverses" kann eingestellt werden, dass die Einträge in den Eintragslisten in abwechselnder Farbe angezeigt werden.
  • Kalender: Anzeige der Jahreszahl auch in der Wochen-/Monatsansicht.
  • Korrektur der angezeigten Kalenderwoche (KW) im Dialog "Kalender".
  • Bei der Auswahl einer Vorlage konnte es zu Darstellungsfehlern kommen.
  • Kalender: Bei der Auswahl des Zeitraums wurde u.U. die Dauer falsch angezeigt. Ausserdem kam es zu Anzeigefehlern bei der Auswahl der Stoppzeit, wenn diese auf einen Folgetag eingestellt wurde.
  • Eigene Spalten/Spaltenanordnungen für jede Art der Eintragslisten (z.B. Adressen, Eingang, Ausgang).
  • Design Änderungen.
  • Dialog der Client-Verteilung ist nun nicht mehr modal und kann mehrfach aufgerufen werden.
  • Optimierung des Bildschirmschoners.
  • Der Schalter "IHS Replizierung unterbinden" befindet sich nun auf der Seite "Dienste" der Archive-Eigenschaften (bisher "Optionen").
  • Einstellung "Symbolleisten fixieren" galt nach Neustart nicht für den Editor.
  • Anpassung der Hotkeys für die Attribute "Fett/Kursiv" im Nachrichten Editor (SHIFT+STRG F/K).
  • "Nachricht versenden" Dialog: Formularlayout per Tastenkombi erreichbar (ALT+Y)
  • TIFF Dateien (Dateiendung *.TIF) werden nun auch beim Öffnen über den Dateianhang Dialog mit dem Tobit InfoCenter Image Editor geöffnet.
  • Namentliche Bezeichnung des Ports, der versendet/empfangen hat.
  • Bei der Schnellsuche (ALT-Q) kann nun ein vorhandener Eintrag ausgewählt werden, ohne nochmal die ENTER-Taste drücken zu müssen.
  • Die Taste F10 funktioniert nun auch beim Erstellen einer neuen Aufgabe.
  • Beim Schließen einer Aufgabe wird nun generell nach dem Speichern gefragt, wenn Änderungen vorgenommen wurden.
  • Der neue Button »Gruppiert« ordnet die Einträge der aktuellen Liste nach auswählbaren Kriterien an, z.B. nach Betreff, Datum, etc. Steht man im Eingangsbuch, können optional die Einträge des Ausgangsbuches mit in die Gruppierung aufgenommen werden.
  • Beim Posten mit dem News Client wurde die eMail Adresse des Absenders (FROM) nicht gesetzt.
  • Bei der Adressauswahl wird nun die "2. eMail" Adresse vorgeschlagen, wenn diese als "bevorzugte" eMail Adresse markiert ist.
  • Bei der Adressauswahl im Tobit InfoCenter wurden unter Umständen nicht alle vorhandenen Adressen angezeigt.
  • Die Adressansicht war fehlerhaft (Text überschrieben) wenn das Feld »Titel« oder »Firma« mehrzeilige Einträge enthielt.
  • In der Kalender Ansicht wird nun ein evtl. eingetragener Ort in der Vorschauleiste angezeigt.
  • Beim Drag & Drop einer Datei aus dem Dateianhang in das Textfeld kam es zu einer Schutzverletzung.

  • Beim "erneuten Eintragen" einer eMail über das Tobit InfoCenter konnte es zu der Fehlermeldung »TXFile nicht zu öffnen« kommen (wenn im DvISE PostMan der Schalter »eMail Rundsendung« gesetzt ist).
  • Erstellen von Verteilregeln über den »Löschen«-Dialog: Eine Verteilregel wird jetzt nicht nochmal zusätzlich angelegt, wenn bereits eine identische vorhanden ist. In der Liste der Regeln können nun mehrere Einträge selektiert und gelöscht werden.
  • Tagesübergreifende Serientermine wurde unter Umständen falsch angezeigt: ein Serientermin von 13:00 bis zum nächsten Tag um 14:30 wurde in der Tagesansicht des 2. Tages falsch angezeigt.
  • Bei den eingerichteten News Servern wird nun eine Portal Seite mit den abonnierten News Groups angezeigt.
  • Wird eine Aufgabe per Drag´nDrop in den Kalender gezogen, so gingen bisher die Informationen der Aufgabe (Status, etc.) verloren. Nun werden diese Informationen als "Status-Report Header" in der neuen Nachricht eingefügt.
  • Im Kalender Editor wird nun bei der Termindauer auch die wirkliche Anzahl der Minuten angezeigt. Bisher stand z.B. bei einem Termin von 08:00 bis 09:50: 1,5 Stunden. Nun wird dort in diesem Fall 1:50 angezeigt.
  • Eine in das TAS importierte Datei kann nun auch per Drag´nDrop als Anhang an eine neu erstellte eMail angehängt werden. Bisher wurde sie in diesem Fall als »*.EML«-Datei angehängt.
  • Beim Scrollen mit der Maus in einem leeren Aufgaben Archive kam es zu einer Schutzverletzung.
  • Neue Portal Seite wird geöffnet, wenn im aktuellen Tobit InfoCenter der Button/Menüpunkt "DvISE Administrator" gewählt wird. Durch den Klick auf diverse Links der Portal Seite wird der DvISE Administrator mit entsprechenden Parametern aufgerufen. Der Button/Menüpunkt ist nur vorhanden (wie auch bisher), wenn der User Rechte im \david\code Verzeichnis besitzt.
  • HTML-eMails können nun formatiert aus der Eintragsliste ausgedruckt werden(Vorlage: \david\clients\windows\EMail.prn).
  • Wird eine neue Verteilregel durch den "Löschen"-Dialog (externe Nachrichten) erstellt, so wird der Zeitpunkt der Erstellung bei der Verteilregel ("Letzter Zugriff") eingetragen.

    Die Anzeige der Sortierungszeichen von Archives (z.B. #01#Archivename) wird nun in der Kontakteleiste, beim Verteilen und in der Adressauswahl unterbunden.
  • Löschen externer Nachrichten (z.B. SPAM): Im »Löschen«-Dialog stehen die Punkte »Absender dauerhaft blockieren« bzw. »Empfänger dauerhaft blockieren« nicht zur Auswahl, wenn die zugehörigen Felder in der Nachricht leer sind. Ist die Empfängeradresse gleich der eigenen eMail-Adresse, so ist ebenfalls das Blockieren des Empfängers nicht möglich. Zusätzlich erscheint bei der Auswahl einer dieser Optionen eine zusätzliche Sicherheistabfrage, bevor die Verteilregel angelegt wird.
  • Über das Menü »Ansicht -> Vorschau« kann das Vorschau-Fenster auch unterhalb (Vorgabe) oder rechts neben der Eintragsliste angeordnet werden.
  • Kopiert man eine eMail per Drag´nDrop in das Aufgaben-Archive, so wird bei der neuen Aufgabe die ursprüngliche Absenderinformation der eMail (FROM) übernommen.
  • Der neue Menüpunkt »Öffnen« im Kontextmenü öffnet das aktuelle Archive in einem eigenen (neuen) Fenster.
  • Die Tastaturkürzel »Strg+F6« bzw. »Strg+F8« öffnen das Eingangs- bzw. Ausgangsbuch in einem eigenen Fenster.
  • Beim Drag & Drop einer Nachricht in eine Plain-Text-eMail wird die Nachricht nun als ».EML«-Datei angehängt und nicht in den Text der eMail eingefügt.
  • Über den Editor können Aufgaben nun HTML-formatiert gedruckt werden. Dazu wird die neue HTML-Vorlage TASK.PRN im Verzeichnis »DAVID\CLIENTS\WINDOWS« verwendet.
  • Die Suche im Tobit InfoCenter ist nun für alle Archives so eingestellt, dass ein eigener Ergebnisdialog mit einer Liste der gefundenen Einträge angezeigt wird.
  • Kalendereinträge können nun auch per Drag´nDrop in andere Kalender-Archives oder zum Beispiel auf den Desktop kopiert werden.
  • In der Monats-/Wochenansicht des Kalenders wird nun eine Quickinfo mit den wichtigsten Informationen angezeigt, wenn man den Mauszeiger über einem Eintrag ´schweben´ lässt: Ort, Gruppenkalender, Teilnehmer.
  • Beim Drag & Drop einer Nachricht in das Aufgaben-Archive bleibt die ursprüngliche Formatierung dieser Nachricht erhalten und wird nicht mehr durch eine evtl. eingestellte Vorlage "überschrieben" (Nachrichten Editor: "Vorlage für neuen Nachrichten").
  • Nach dem Import eines ganztägigen Termins aus Outlook wird dieser nun auch in der Wochen- und Monatsansicht des Tobit InfoCenters korrekt eintägig dargestellt.
  • Dialog "Dokumentenvorlage": Die angezeigte Reihenfolge der Sub Archives unter »Archive -> Resourcen -> Formulare« kann mit einem Sortierungszeichen im Archivenamen festgelegt werden, z.B. #01#Erstes Archive.
  • Über das Menü »Datei -> Drucken« können die Eintragslisten (Eingang, Ausgang, etc.) gedruckt werden. Der Ausdruck entspricht der aktuell eingestellten Ansicht im Tobit InfoCenter (Spaltenanordnung/Datum, etc.).
  • Über das Menü »Datei -> Drucken« kann die Aufgabenliste gedruckt werden. Der Ausdruck entspricht der aktuell eingestellten Ansicht im Tobit InfoCenter (Spaltenanordnung/Datum, etc.).
  • Einträge im Kalender-Archive können nun als eMail-Anhang weitergeleitet werden ("Weiterleiten" im Kontextmenü bzw. als Menüpunkt).
  • Im Kalender kann mit dem Pfeil im "Heute"-Button ein Kalenderdialog geöffnet werden, um zu einem bestimmten Tag zu wechseln.
  • Beim Erstellen einer Nachricht mit Dateianhang aus dem "combit address manager" wird nun zuverlässig die Nachricht inkl. Anhang erstellt.
  • Bei den Eintrags-Eigenschaften kann nun auch die Farbe für den aktuellen Eintrag eingestellt werden.
  • Die Suche im Kalender-Archive wurde erweitert. In einem neuen Dialog werden nun die Suchergebnisse aus dem gesamten Kalender-Archive angezeigt (bisher nur aus der aktuellen Ansicht). Die Suche kann auch auf gemeinsam genutzte Kalender, die im aktuellen Kalender-Archive angezeigt werden, erweitert werden.
  • Bei der Erstellung von Briefen im Tobit InfoCenter werden nun auch die Benutzer-Variablen »Benutzer Fax«, »Benutzer Tel.« und »Benutzer SMS« ausgewertet.
    Bei einem empfangenen und ungeöffneten Einschreiben ist nun der Button »Löschen« nicht mehr auswählbar.
  • Das "Weiterleiten" einer Nachricht wurde trotz deaktiviertem Tracking an den Absender übermittelt.
  • Beim Erstellen einer Brief-Rundsendung kann nun gewählt werden, ob für jeden Ausdruck ein Protokolleintrag im Ausgangsbuch eingetragen werden soll.
  • Im englischen Tobit InfoCenter werden die Zeiten zwischen 12:00 und 13:00 Uhr nun korrekt mit »PM« statt »AM« angezeigt.
  • Beim Speichern mehrerer Dateianhänge erfolgte unter Umständen nicht die Abfrage nach dem Überschreiben bereits vorhandener Dateien.
    Aktiviert man unter »Optionen -> Ansicht -> Nachricht bei Wechsel als gelesen markieren«, so wurde damit die Funktion "Nachricht als ungelesen markieren" außer Kraft gesetzt.
  • Wird bei einer neuen Nachricht die eMail-Adresse mit dem Befehl @@AN im Textfeld eingetragen und das »An...«-Feld freigelassen, wurde die Adresse beim anschließenden Öffnen der versendeten Nachricht nicht angezeigt.
