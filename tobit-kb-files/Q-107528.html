Q-107.528
Installation und Einrichtung des David.InfoCenter Mobile


Frage
Problem Was ist bei der Installation und Einrichtung des David.InfoCenter Mobile zu beachten?
Produkt David.InfoCenter Mobile
Priorität normal
Effekt normal
Datum 2.12.2008

Antwort
Das David.InfoCenter Mobile ist eine spezielle Version des David.InfoCenters, die auch auf mobilen Computern ohne dauerhafte Netzwerkverbindung genutzt werden kann. Damit können z.B. Außendienstmitarbeiter, die mit ihrem Notebook unterwegs sind, auch ohne Verbindung zum David Server und somit unabhängig von Ort und Zeit mit dem David.InfoCenter arbeiten. Zu versendende Nachrichten werden im Offline-Modus – also wenn keine Verbindung zum David Server besteht – als Aufträge im Versandbuch abgelegt und erst dann verschickt, wenn wieder eine Verbindung besteht.
Auch die Inhalte der persönlichen Archives können offline wie gewohnt bearbeitet werden. Nach der Wiederherstellung der Server-Verbindung werden die Inhalte mit den zugehörigen Archives auf dem David Server abgeglichen und so auf den gleichen Stand gebracht.

Voraussetzungen
Die zu verwendenden mobilen Computer müssen unter Windows 2000 oder Windows XP betrieben werden. Auf ihnen darf sich keine eigene David Installation befinden. Eventuell vorhandene David Module müssen vor der Installation David.InfoCenter Mobile von diesen Computern entfernt werden.

    Der komplette Funktionsumfang steht nur dann zur Verfügung, wenn das David.InfoCenter Mobile auf einer NTFS-formatierten Festplattenpartition installiert wird. Stellen Sie daher unbedingt sicher, dass alle Computer, auf denen das David.InfoCenter Mobile genutzt werden soll, über eine NTFS-Partition verfügen.
Einstellungen während der David Installation
Bei der David Installation auf den Server können Sie entscheiden, ob die Nutzung des David.InfoCenter Mobile für die David Benutzer generell möglich sein soll oder nicht. Wenn Sie die Nutzung des David.InfoCenter Mobile zunächst unterbunden haben, jedoch zu einem späteren Zeitpunkt dennoch ermöglichen wollen, führen Sie einfach das David Installationsprogramm erneut aus und aktivieren Sie die entsprechende Option.

Falls bisher noch kein System-Passwort für die David Replication Services vergeben wurde, werden Sie während der David.InfoCenter Mobile Installation zur Eingabe eines Passworts aufgefordert. Das nachfolgende Fenster bietet die Möglichkeit, dieses Passwort für die anschließende Installation des David.InfoCenter Mobile auf den Arbeitsstationen zu speichern. In diesem Fall wird das Passwort dann bei der Einrichtung der Clients automatisch in die Konfiguration des David InfoCenter (»Optionen -> Einstellungen -> Mobile«) übernommen und muss später nicht manuell eingetragen werden.

Beachten Sie jedoch, dass bei einer Speicherung des Passworts jeder Benutzer, der auf seiner Arbeitsstation über Administratorrechte verfügt, das David.InfoCenter Mobile an dieser Arbeitsstation installieren und nutzen kann, da ja ein Passwort bei der Installation nicht mehr abgefragt wird. Legen Sie anschließend fest, welche globalen Archives in die Replizierung einbezogen werden sollen und geben Sie schließlich den Domain-Namen bzw. die IP-Adresse des David Servers an.

Arbeitsweise
Die Dienste des David.InfoCenter Mobile werden unter Verwendung der David Replica Services bereitgestellt. Damit ein Betrieb des David.InfoCenter Mobile auch im Offline-Modus – also ohne direkte Verbindung zum David Server – möglich ist, werden die David-Systemdienste »Service Layer« und »Replica« auch auf den Arbeitstationen installiert und gestartet. Aus diesem Grund ist die Nutzung des David.InfoCenter Mobile ausschließlich auf Arbeitsstationen möglich, die unter Windows 2000 oder Windows XP Professional betrieben werden.

    Das David.InfoCenter Mobile kann ausschließlich mit genau einem David Server als Host betrieben werden.
Eine abwechselnde Replizierung mit unterschiedlichen David Servern ist nicht möglich. Auf den mobilen Computern wird bei der Installation des David.InfoCenter Mobile unterhalb des Programmordners des David InfoCenters eine lokale DAVID-Verzeichnisstruktur angelegt. Diese enthält Kopien aller Objekte des David Servers, die für den Offline-Betrieb benötigt werden. Der David Administrator ist auf den mobilen Arbeitsstationen jedoch nicht verfügbar.

Für eine Verbindung zum David Server und die Archive-Replizierung ist keine lokale Anmeldung am Server notwendig, es genügt eine IP-Connection. Die Legitimation findet über ein sicheres Authentifizierungsprotokoll statt, bei dem keine Anmeldedaten übertragen werden.

Replizierung der persönlichen Archives
Bei der Archive-Replizierung werden generell die persönlichen Archives des auf dem mobilen Computer angemeldeten David-Benutzers mit den zugehörigen Datenbeständen auf dem David Server abgeglichen. Dieser Abgleich erfolgt als so genannte »Zwei-Weg-Replizierung«, was bedeutet, dass der David Server und die Arbeitsstation gleichberechtigt sind. Einträge oder Unterordner, die in den persönlichen Archives auf dem mobilen Computer neu erstellt wurden, werden auch in die entsprechenden Archives des David Servers übernommen und umgekehrt. Ebenso werden Einträge, Archives oder Dokumente, die auf einem der Computer gelöscht oder geändert wurden, auch auf dem anderen entfernt oder geändert.

Falls bestimmte Dokumente zwischen zwei Abgleichintervallen unabhängig voneinander sowohl auf dem David Server als auch auf der Arbeitsstation bearbeitet wurden, so sind nach dem Abgleich auf jedem der Computer beide Versionen des Dokumentes vorhanden. Das Dokument des jeweils anderen Rechners wird durch den Zusatz »REP:« in der Betreffzeile gekennzeichnet.

Replizierung globaler Archives
Außer den persönlichen Benutzer-Archives können auch bestimmte globale Archives in die Replizierung einbezogen werden. Hierbei sind ausschließlich folgende Archives wählbar:

  • Archives/Ressource/Feiertage
  • Archives/Ressource/Formulare
  • Archives/Ressource/Textbausteine
  • Archives/Ressource/Vorlagen
  • Archives/Adressen (globales Adress-Archive)
Welche dieser Archives in die Replizierung einbezogen werden sollen, wird direkt während der Installation der Server-Komponenten festgelegt. Diese Auswahl gilt daraufhin für den Abgleich mit allen mobilen Clients. Der Abgleich dieser Archives erfolgt grundsätzlich als Master-Slave-Replizierung, wobei in jedem Fall der David Server der Master ist. Die Arbeitsstationen fungieren dann zwangsläufig als Slaves. Bei dieser Art der Replizierung verläuft der Abgleich ausschließlich in eine Richtung, nämlich vom Master auf den Slave. Nach Abschluss des Abgleichs verfügt der Slave also über ein Abbild der Datenstruktur des Masters. Befinden sich auf dem Slave Änderungen oder neu erstellte Archives oder Dokumente, so werden diese nicht zum Master übertragen, sondern stattdessen auf dem Slave gelöscht.

Einrichten der Arbeitsstationen
Um die Installation des David.InfoCenter Mobile auf den mobilen Arbeitsstationen vornehmen zu können, müssen diese müssen korrekt im Netzwerk angemeldet sein und Zugriff auf den David Server besitzen.Weiterhin muss der Benutzer, der die Installation durchführt, über Administratorrechte auf der jeweiligen Arbeitsstation verfügen. Standardmäßig können bis zu 25 mobile Arbeitsstationen gleichzeitig ihre Datenmit demselben Host Server replizieren. Dieser Maximalwert kann bei Bedarf in der Systemkonfiguration der David Replication Services am David Server verändert werden (»Max. Verbindungen«).

    Das Zielverzeichnis sollte sich unbedingt auf einer NTFS-formatierten Festplattenpartition befinden.
Wechseln Sie von der Arbeitsstation aus über das Netzwerk in das \DAVID\CLIENTS Verzeichnis auf dem David Server und führen Sie dort die Datei SETUP.EXE aus, um die Installation zu starten. Wählen Sie die gewünschte Dialogsprache und Installationsart (Standard oder Benutzerdefiniert). Aktivieren Sie anschließend im folgenden Dialogfenster die Option »David.InfoCenter Mobile installieren«.

Wurde das System-Passwort für die Replizierung während der Server-Installation nicht gespeichert, erfolgt nun eine Aufforderung zur Eingabe dieses Passwortes. Ohne die Eingabe des korrekten System-Passworts wird die Installation zwar zu Ende geführt, eine Replizierung mit dem Host Server führt jedoch zu einer Fehlermeldung (Replikationsmonitor). Erst wenn im David InfoCenter unter »Optionen -> Einstellungen« auf dem Register »Mobile« das korrekte System-Passwort eingetragen wurde, erfolgt eine fehlerfreie Replizierung.

Zum Abschluss wird abgefragt, ob beim Hoch- bzw. Herunterfahren des Betriebssystems an die Durchführung eine Replizierung erinnert werden soll. Zu diesem Zweck wird auf der mobilen Arbeitsstation ein Dienst installiert und gestartet. Setzen Sie den Schalter »Erinnerung einschalten«, wenn Sie an die Replizierung erinnert werden möchten. Sofern für den Remote Access auf die Archives eines Benutzers noch keine Zugriffsparameter vergeben wurden, erfolgt beim ersten Start des David InfoCenters die Abfrage dieser Parameter, die für die Durchführung der Replizierung benötigt werden.

Betrieb des David.InfoCenter Mobile
Das David.InfoCenter Mobile verfügt im Menü »Netzwerk« über die Option »Offline«, mit welcher der Offline-Modus aktiviert bzw. deaktiviert werden kann. Des Weiteren finden Sie dort als Server-Eintrag außer dem Namen des David Servers auch den der lokalen Arbeitsstation. Über diese beiden Einträge kann ohne Replizierung zwischen David Server und Arbeitsstation umgeschaltet werden. Der Wechsel zur Arbeitsstation entspricht dann dem Wechsel in den Offline-Modus.

Unter dem Hauptmenüpunkt »Optionen - > Einstellungen -> Mobile« können Sie Einstellungen für den Abgleich mit dem David Server konfigurieren. Hier stehen die folgenden Konfigurationspunkte zur Verfügung:

  • Host IP/Domäne
    Tragen Sie hier die IP-Nummer des David Servers oder dessen Domain Namen ein.
  • System Passwort
    Das hier eingetragene System Passwort gilt für die Replizierung Ihrer Daten mit dem David Server. Das Passwort darf nur verändert werden, wenn auch der gleichnamige Eintrag auf dem Server (Menüpunkt »David Replica«) angepasst wird. Im Zweifelsfall belassen Sie den Eintrag unverändert, um Fehler bei der Replizierung zu vermeiden.
  • Beim Starten/Beenden des PCs replizieren
    Aktivieren Sie den Schalter, wenn beim Herauf- bzw. Herunterfahren des Betriebssystems eine Abfrage erfolgen soll, die Arbeitsstation mit dem Host Server zu replizieren.
  • Remote Access Parameter verwenden
    Setzen Sie diesen Schalter, wenn Sie wünschen, dass für die Archive-Replizierung dieselben Authentifizierungsparameter verwendet werden, wie für den Remote Access. Wenn Sie die Remote-Access-Parameter nicht verwenden möchten, tragen Sie Benutzername und Passwort in die zugehörigen Felder ein.
  • Intervall
    Wählen Sie aus der Liste ein Intervall, wonach automatisch eine Replizierung mit dem David Server vorgenommen werden soll. Nach dem hier konfigurierten Zeitabstand wird generell versucht, einen Abgleich durchzuführen. Wählen Sie z.B. ein Intervall von 30 Minuten um zu erreichen, dass spätestens eine halbe Stunde, nachdem der Rechner wieder mit dem Netzwerk verbunden wurde, automatisch ein Abgleich stattfindet. Ist nach Ablauf des Intervalls keine Verbindung zum David Server möglich, wird eine entsprechende Meldung im Archive »System - David - Ereignisse« erzeugt.
Offline-Modus aktivieren bzw. deaktivieren
Solange sich die mobile Arbeitsstation im lokalen Netzwerk befindet, also direkte Verbindung zum David Server besitzt, greift das David InfoCenter direkt auf diesen Server zu. Das bedeutet, dass sämtliche Aktionen im David InfoCenter sofort über den David Server abgewickelt werden. Änderungen an Inhalten im Tobit Archive System werden ebenfalls sofort auf dem Server vorgenommen.

Wenn der mobile Computer vom Netzwerk getrennt wird, ist der direkte Zugriff auf den Server natürlich nicht mehr möglich. Daher muss vor dem Trennen der Netzwerkverbindung der Offline-Modus aktiviert werden.

Klicken Sie dazu auf »Netzwerk -> Offline« in der Menüleiste des David InfoCenters. Dieses wird nun intern auf die lokale Kopie des Tobit Archive Systems umgeschaltet. Im Normalfall sollte anschließend eine Archive-Replizierung durchgeführt werden, um Ihre Arbeitsstation und den David Server auf den gleichen Stand zu bringen. Es erscheint ein Dialogfenster, das zur Durchführung der Replizierung auffordert.

Im Offline-Modus wird ausschließlich mit der lokalen Kopie des Tobit Archive Systems gearbeitet. Die während der Offline-Zeit durchgeführten Änderungen werden auf den David Server übertragen, sobald die Arbeitsstation wieder mit dem Netzwerk verbunden und das David InfoCenter wieder in den Online-Modus geschaltet wird. Das hat zur Folge, dass z.B. ein Versandauftrag, der im Offline-Modus erstellt wurde, erst bei der nächsten Replizierung an den Host Server übertragen und von diesem anschießend versendet wird. Arbeitet man mit der Arbeitsstation zwischenzeitlich wieder im Offline-Modus, so wird dieser Versandauftrag mit dem entsprechenden Status-Eintrag wiederum erst mit der nächsten Replizierung im Ausgangsbuch der Arbeitsstation sichtbar.

Klicken Sie dazu wiederum den Menüpunkt »Netzwerk -> Offline« an. Das Häkchen vor »Offline« verschwindet und das David InfoCenter wird nun wieder für den direkten Zugriff auf den David Server umgeschaltet. Auch hier erscheint das Dialogfenster, das zur Durchführung der Archive-Replizierung auffordert, um wiederum die Datenbestände beider Computer miteinander abzugleichen.

Weiterführende Informationen
Ausführliche Hilfetexte finden Sie zum einen im David-Systemhandbuch »David Information Server Engine Systemhandbuch und technische Referenz« zum anderen dem Handbuch »David InfoCenter Einführung und Anwendung«. Außerdem steht für jedes Dialogfenster per [F1]-Taste eine Online-Hilfe zur Verfügung.

Hinweise:
Bei der Nutzung des David.InfoCenter Mobile unter Windows XP ist es u.U. erforderlich, dass Sie die »Einfache Dateifreigabe« deaktivieren. Weitere Informationen entnehmen Sie dem KnowledgeBase Artikel Q-100.903.

Die Nutzung des David.InfoCenter Mobile auf einem Computer ist nur von einem Benutzer bzw. mit einem Benutzerkonto möglich.
