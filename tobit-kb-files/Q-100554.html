Q-100.554
Anmeldefehler beim Starten des Connectordienstes


Frage
Problem Beim Start des Connector for Microsoft Exchange 2000 wird unter den Windowsdiensten der Fehlercode 7023 »Der Dienst konnte aufgrund einer fehlerhaften Anmeldung nicht gestartet werden« ausgegeben.
Produkt Exchange Connector for David
Priorität normal
Effekt kritisch
Datum 26.07.2005

Antwort
Prüfen Sie bitte zunächst die folgende Einstellungen:
  1. Der Benutzer, der den Dienst »Connector for Microsoft Exchange« startet, benötigt auf dem Exchangesystem und auf dem DvISE System administrative Rechte. Weiterhin stellen Sie sicher, dass das David-Verzeichniss freigegeben und für diesen Benutzer erreichbar ist.

  2. Weiterhin müssen Sie gewährleisten, dass der Server bei der Installation auf die erweitert Mapi von Microsoft zugreift und nicht die Mapi von DvISE oder die Simple Mapi von Outlook Express:
    • Tauschen Sie die MAPI32.DLL im Verzeichnis »WINNT\SYSTEM32« gegen die MAPI32.DLL aus dem BIN-Verzeichnis von MS Exchange aus.

  3. Nach dem Austausch der DLL sollten Sie den DvISE Connector deinstallieren, den Dienst »IIS-Admin« mit allem zusammenhängenden Diensten neu starten und den DvISE Connector erneut installieren.

  4. Nun prüfen Sie zusätzlich die Datei DAVIDLIB.INI, die sich nach der Installation im Verzeichniss »WINNT\SYSTEM32« befindet.
    Hier muss, falls vorhanden, der Servername des DvISE-Servers hinterlegt sein. Der Eintrag in der DAVIDLIB.INI muss wie folgt definiert werden:

    [INIT]
    Server=DvISE-Server

  5. Haben Sie den DvISE Server nicht auf dem Exchange-Server installiert, ist einer der häufigsten Ursachen für einen Anmeldefehler der Eintrag »DvISE als Remote Server« in der Systemkonsole von Exchange (bei Exchange 2000 unterhalb von »Connectors/Connector to Microsoft Exchange for DvISE/Gerneral«).

    Bitte prüfen Sie den hier definierten Benutzer, er benötigt minimal Vollzugriff auf die folgenden Verzeichnisse mitsamt Unterverzeichnisse:

    • »DAVID\CODE«
    • »DAVID\IMPORT«
    • »DAVID\APPS\FAXWARE«
    Es empfiehlt sich allerdings, dass Sie diesem Benutzer Vollzgriff auf die gesamte David Verzeichnisstruktur gewähren.
    Folgenden Sie hierzu bitte den Hilfestellungen im
    KnowledgeBase Artikel Q-100.212

  6. Sollten Sie Änderungen an der Rechtevergabe getätigt haben, so vergessen Sie bitte nicht, die DvISE Dienste inkl. dem Dienst »DvISE Connector for Microsoft Exchange« neu zu starten, damit die Änderungen wirksam werden.
