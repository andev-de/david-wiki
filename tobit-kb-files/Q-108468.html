Q-108.468
Hinweise zum David Dashboard


Frage
Problem Anwendern von David.zehn! steht im Verzeichnis »David\Util\Windows« das Programm »David Dashboard.exe« zur Verfügung. Wie wird dieses Programm genutzt und welche Funktionen sind enthalten?
Produkt Automation
Priorität normal
Effekt normal
Datum 8.05.2008

Antwort
David verfügt über spezielle Softwareschnittstellen zur Automation, die z.B. eine Anbindung an Hausbussysteme ermöglichen. Die Funktionen der Gebäudesteuerung werden vom »Automation Module for David« übernommen. Dieses Modul ist in der Lage, Informationen aus Bussystemen zu empfangen und an die am Bus angeschlossenen Module zu versenden. Für die Nutzung der Funktionen zur Automation muss auf dem David Server ein geeigneter Port installiert sowie eine physikalische Verbindung zum Bus geschaffen werden.Weitere Informationen zu dem Thema »Grundlegende Informationen zur »Automation mit David« und der Einrichtung eines Automation-Ports« erhalten Sie im KnowledgeBase Artikel 108.470.

Zur Nutzung der Automation Services befindet sich auf dem David Server unterhalb von »Archive - System - David« ein spezielles Archive mit dem Namen »Automation«. Ebenso wie bei allen anderen Archives unterhalb von »System« besitzt standardmäßig ausschließlich der Administrator Zugriffsrechte auf dieses Archive. Die Schalt- und Kontrollvorgänge können z.B. durch die Ausführung von JavaScripts erreicht werden, die unterhalb des Automation Archives abzulegen sind. Hierzu sind Kenntnisse im Umgang mit und in der Erstellung von JavaScripts hilfreich.

Das David Dashboard bietet Ihnen stattdessen die Möglichkeit, in einer grafischen Oberfläche Objekte auszuwählen und automatisch mit den Adressen des Bussystems zu verknüpfen. Die Konfiguration des Bussystems (Visualisierung zur Zustandsanzeige) bzw. die Steuerung von Geräten zur Gebäude-Automation wird somit erleichtert.

Zum Start des David Dashboard wechseln Sie auf dem David Server in das Verzeichnis »David\Util\Windows« und doppelklicken die Datei »David Dashboard.exe«. Das David Dashboard wird dann im sogenannten Design-Modus geöffnet. Hier können Sie neue Automations-Projekte erstellen, oder schon bestehende Projekte aufrufen. Die Steuerung des erstellen Projektes erfolgt im Runtime-Modus. Dieser Modus ermöglicht die Steuerung und zeigt die aktuellen Einstellungen und Werte. Das Umschalten zwischen Design- und Runtime-Modus erfolgt mit der Taste [F6].

Ein neues Projekt bzw. eine neue Automation erstellen Sie mit einem Klick auf »Datei -> Neu« oder mit der Taste [F9]. In dem neuen Dialogfenster tragen Sie den Namen des Servers, auf dem die Automation ausgeführt wird, sowie optional einen Namen für das Projekt (z.B. Erdgeschoss) ein. Nach einem Klick auf »Weiter« binden Sie eine Hintergrundgrafik ein (z.B. den Grundriss des Erdgeschosses) und wählen die anzuzeigenden Symbole aus. Achten Sie bei der Auswahl einer Hintergrundgrafik auf das richtige Dateiformat. Es werden unter anderem »JPG«, »GIF« und »BMP« unterstützt. Legen Sie anschließend das Bussystem und weitere Eigenschaften fest. Ein Klick auf »Fertig stellen« schließt die Konfiguration ab.

Nach Abschluss der Konfiguration wird die Verbindung zum angegebenen Server geprüft. Kann die Verbindung erfolgreich hergestellt werden, stehen auf der rechten Seite des Programmfensters die Module des Bussystems (aus dem Archive »State«) zu Verfügung. Sie haben nun die die Möglichkeit, die Steuerung Ihres Bussystems grafikbasiert zu gestalten bzw. zu programmieren. Aus der Symbolleiste auf der linken Seite ziehen Sie z.B. das Symbol »Lampe« auf Ihre Hintergrundgrafik. Automatisch öffnet sich ein Dialogfenster, auf dem Sie die Objekteigenschaften festlegen. Geben Sie auf der Registerkarte »Allgemein« unter anderem »Name«, »Adresse« und »Objekttyp« an. Auf der Registerkarte »Erweitert« legen Sie die Kommandos fest bzw. definieren spezielle Regeln.

Haben Sie ein neues Projekt erstellt und wollen nun vom Design- in den Runtime-Modus wechseln, müssen Sie das Projekt zunächst speichern. Gespeicherte Projekte haben die Dateiendung »*.DDB« und Dateien mit dieser Endung werden nach dem ersten Aufruf dem David Dashboard zugeordnet. Das Umschalten zwischen den Modi ist anschließend mit der Taste [F6] möglich. Beachten Sie, dass das erstellte Projekt geöffnet sein muss, damit die dort eingerichteten Steuerungen ausgeführt werden. Wird das David Dashboard beendet, können die Aktionen, Regeln oder Steuerungen, die Sie in Ihrem Projekt definiert haben, unter Umständen nicht mehr durchgeführt werden.

Hinweise:
Die Nutzung dieser Dienste ist nur möglich, wenn sich die David Installation auf einem Computer befindet, der unter Microsoft Windows betrieben wird. Beim Betrieb unter Novell NetWare oder Linux stehen dieAutomation-Dienste nicht zur Verfügung.

Grundlagen zu den Standard-Bussystemen entnehmen Sie dem KnowledgeBase Artikel 107.971.

Eine Übersicht der zurzeit unterstützten Bussysteme entnehmen Sie dem KnowledgeBase Artikel 103.699.
