Q-101.581
Die Datei NTVDM.EXE hat einen Fehler verursacht


Frage
Problem Die Datei NTVDM.EXE hat einen Fehler verursacht und das Tobit InfoCenter wird beendet.
Produkt David.InfoCenter
Priorität normal
Effekt normal
Datum 2.08.2002

Antwort
Wichtig: Dieser Artikel enthält Informationen zum Bearbeiten der Registrierung. Bevor Sie die Registrierung bearbeiten, vergewissern Sie sich, dass Sie die Registrierung wiederherstellen können, wenn ein Problem auftritt. Informationen hierzu erhalten Sie, indem Sie in der Datei "REGEDIT.EXE" unter dem Hilfethema "Wiederherstellen der Registrierung" oder in der Datei "REGEDT32.EXE" unter dem Hilfethema "Wiederherstellen eines Registrierungsschlüssels" nachsehen.

Zusammenfassung:
Im folgenden wird Schritt für Schritt beschrieben, wie Sie Fehler beim NTVDM- oder WOW-Start beheben. Beachten Sie bitte, dass SYSEDIT.EXE ein 16-Bit-Programm ist, dass in Windows NT für die Fehlerbehandlung bei solchen Problemen zur Verfügung steht. Wenn dieses Programm gestartet wird, sind die WOW- und NTVDM-DLLs und die ausführbaren Dateien in Ordnung, und Sie müssen sich dann auf die unten beschriebenen Schritte konzentrieren, die sich auf die Umgebung beziehen. Wenn SYSEDIT nicht gestartet wird, versuchen Sie, nur die MS-DOS-Shell "COMMAND.COM" aus der Eingabeaufforderung zu starten.

Bitte beachten Sie, dass alle folgenden Schritte auszuführen sind, wenn SYSEDIT nicht läuft.

Weitere Informationen:

  1. Überprüfen Sie AUTOEXEC.NT und CONFIG.NT auf irgendetwas Ungewöhnliches oder alles, was auskommentiert werden kann.
  2. Benennen Sie die Datei AUTOEXEC.BAT um.
  3. Überprüfen Sie WIN.INI oder SYSTEM.INI auf Änderungen oder ersetzen Sie sie, indem Sie die Originaldateien vom Installationsmedium expandieren.
  4. Benennen Sie möglichst alle anderen *.INI-Dateien um.
  5. In der Datei CONFIG.NT sind nur folgende Angaben zulässig:

    dos=high, umb
    device=%SystemRoot%\system32\himem.sys
    files=60
    shell=%SystemRoot%\system32\command.com /e:4096

Hinweis:
Mit den Standardeinträgen vom Installationsdatenträger kann SYSEDIT laufen; es können jedoch weitere Einträge wie oben beschrieben erforderlich sein, damit andere Anwendungen laufen können. Gültiger Bereich für Dateien: files = 5 bis 254.

6. Überprüfen Sie die Umgebungsvariablen wie folgt: Klicken Sie auf START, zeigen Sie auf EINSTELLUNGEN, klicken Sie auf SYSTEMSTEUERUNG, doppelklicken Sie auf SYSTEM und klicken Sie dann auf die Registerkarte UMGEBUNG. In Windows 2000 klicken Sie auf EINSTELLUNGEN, dann auf SYSTEMSTEUERUNG, doppelklicken Sie auf SYSTEM, klicken Sie auf die Registerkarte ERWEITERT und dann auf UMGEBUNGSVARIABLEN.

    -oder-
Führen Sie den Befehl SET aus der Eingabeaufforderung aus und überprüfen Sie alle Umgebungsvariablen auf Nullen, Leerzeichen, doppelte Gleichheitszeichen (==) oder sonstige unbekannte Einträge.

7. Vergewissern Sie sich, dass der Pfad mit dem Standardpfad der CD übereinstimmt, d.h., dass dem Pfad keine Einträge vorangestellt sind. Falls es vom Standard abweichende Einträge gibt, entfernen Sie sie. 8. Überprüfen Sie die Pfadlängen auf Längen, die nach der Expandierung nahezu 200 Zeichen betragen. Der Pfad setzt sich zusammen aus dem Pfad in Autoexec.nt und in der Registerkarte UMGEBUNG im Programm SYSTEM in der Systemsteuerung. Wenn der Pfad mehr als 100 Zeichen hat, ändern Sie ihn zu Testzwecken.

9. Überprüfen Sie die folgenden Registrierungseinträge. Die einfachste Möglichkeit dafür ist, zunächst festzustellen, ob sie gefüllt sind. (In manchen Fällen verschwinden Schlüssel oder Teilschlüssel einfach). Wenn die Schlüssel komplett gefüllt sind, versuchen Sie nicht, die richtigen Werte zu ermitteln; laden Sie sie einfach herunter und laden Sie sie in ein Testsystem, um herauszufinden, ob das Problem reproduzierbar ist.

WARNUNG:
Die unkorrekte Verwendung des Registrierungseditors kann schwerwiegende Probleme verursachen, die eine Neuinstallierung von Windows erforderlich machen. Microsoft kann nicht dafür garantieren, dass Probleme, die von einer falschen Verwendung des Registrierungseditors herrühren, behoben werden können. Benutzen Sie den Registrierungseditor auf eigene Verantwortung.

Informationen zum Bearbeiten der Registrierung finden Sie unter dem Online-Hilfethema "Ändern von Schlüsseln und Werten" im Registrierungseditor (Regedit.exe). Beachten Sie bitte, dass Sie eine Sicherungskopie der Registrierungsdateien (System.dat und User.dat) erstellen sollten, bevor Sie die Registrierung bearbeiten. Wenn Sie mit Windows NT arbeiten, sollten Sie auch Ihre Notfalldiskette (ERD = Emergency Repair Disk) sichern.

HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\WOW: Parameters für WOW-Start

HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\WOW: System.ini-Einstellungen

HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\IniFile\Mapping: Zuordnungen für Win.ini, System.ini, Winfile.ini,Progman.ini und Control.ini

HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\Session Manager\Environment: Umgebungseinstellungen Sonstige INI-Einstellungen, die in der Registrierung zu finden sind: s. Seiten 905 bis 908 des Windows NT Workstation 4.0 Resource Kit.
