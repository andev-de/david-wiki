Q-108.288
Anwendungsbeispiel des JavaScript TLDs: Conrad C-Control


Frage
Problem Die Conrad C-Control-Station soll unter Verwendung des JavaScript TLDs an David angebunden werden.
Produkt Automation
Priorität normal
Effekt harmlos
Datum 16.02.2006

Antwort
Dieser Knowledge Base Artikel erläutert die Einbindung der C-Control-Station von Conrad Electronic. Hierbei handelt es sich um eine programmierbare Steuer-Einheit, die sich per seriellem Anschluss (COM-Port) mit dem David Server verbinden lässt, und so für die individuelle Steuerung verschiedenerSysteme sorgt.

Voraussetzung für den Betrieb mit der C-Control-Station ist der Einsatz von David ab der Version V8+. Der David Server muss darüber hinaus über mindestens eine freie serielle Schnittstelle für den Anschluss der C-Control-Station verfügen.

Installation und Einrichtung
Führen Sie die nachfolgend aufgeführten Schritte durch, um die Integration einer C-Control-Sation in Ihre David Installation vorzunehmen:

Installation eines JavaScript TLDs
Die Kommunikation zwischen David und der C-Control-Hardware findet über einen eigenen DvISE Port statt. Hierbei handelt es sich um einen JavaScript TLD. Also ist es zunächst erforderlich, einen solchen Port zu installieren. Befolgen Sie dazu die Anweisungen, wie im KnowledgeBase Artikel Q-108.144 beschrieben.

Einbinden des JavaScript Codes
Laden Sie sich die Datei ccontrol.zip herunter und entpacken die beiden Beispiel-Dateien »tldx.js« und »ccontrol.bas« in ein beliebiges Verzeichnis. Speichern Sie die Datei »tldx.js« in das Port Verzeichnis des zuvor erstellten Extended Port ab (z.B. DAVID\TLD\PORT\003, für den Fall, dass es sich hierbei um den Port mit der Nummer 003 handelt). Stellen Sie anschließend sicher, dass die in dieser Datei die serielle Schnittstelle eingestellt ist, mit der die C-Control-Station tatsächlich verbunden ist.

Öffnen Sie dazu die Datei mit einem Editor und suchen Sie den Eintrag »var ComPort =« . Standardmäßig ist diesem Eintrag der Wert »1« zugewiesen, was für COM-Port 1 steht. Sollte die C-Control-Sation jedoch an einem anderen COM-Port angeschlossen sein, ändern Sie den Eintrag so ab, dass der zugewiesene Wert mit der Nummer des betreffenden COM-Port übereinstimmt. Speichern Sie anschließend die Änderung und schließen Sie die Datei wieder.

C-CONTROL I
Installieren Sie auf dem David Server das Programm mit dem Namen »C-CONTROL I«, das sich im Lieferumfang der C-Control-Hardware befindet. Diese Software ermöglicht es, Programmcodes auf die C-Control-Station aufzuspielen. Nähere Informationen hierzu entnehmen Sie dem C-Control Handbuch. Bevor Sie das Programm starten, sollten Sie unbedingt den JavaScript TLD entladen. Eingemeinsamer Betrieb beider Module ist zwar generell möglich, das Übertragen von Programmcode darf jedoch nur bei entladenem TLD geschehen.Wählen Sie »Start -> Programme -> Conrad Electronic -> C-CONTROL I -> C-CONTROL I«, um das soeben installierte Programm zu starten. Öffnen Sie dann im Programmfenster über »Datei -> Öffnen« die Datei »ccontrol.bas«. Klicken Sie als Nächstes auf »Entwicklung -> In C-Control Unit übertragen«, um den Programmcode auf die C-Control-Station zu kopieren.

Stellen Sie unbedingt sicher, dass zu diesem Zeitpunkt der JavaScript TLD nicht geladen ist.

Starten Sie nach der erfolgreichen Übertragung den JavaScript-TLD. Damit ist das System betriebsbereit. Die folgenden Funktionen stehen zur Verfügung:

  • Einlesen aller aktuellen Status nach Starten des JavaScript TLDs in das DvISE Automation Archive »State«
  • Einlesen aller aktuellen Status nach einem Reset der C-Control-Station in das DvISE Automation Archive »State«
  • Schalten aller Ausgänge der C-Control (K1, K2, LED1-LED4, P1-P6)
  • Empfang aller Werte aus der C-Control (A1-A4, F1-F4, P1-P6, T1, T2, U1, U2)
  • Automatische Abfrage folgender Werte aus der C-Control-Station: A1-A4, T1, T2, U1, U2
  • Senden eines Befehls über den Nachrichten Editor (siehe Abbildung)
