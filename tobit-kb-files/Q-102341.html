Q-102.341
dboard2(AVM B1) Installation und Einrichtung unter Novell NetWare


Frage
Problem Was ist bei der Installation und Einrichtung unter Novell NetWare zu beachten?
Produkt Hardware
Priorität normal
Effekt normal
Datum 2.10.2002

Antwort
Installation und Einrichtung Novell NetWare
Nachdem Sie das dBoard2 in Ihren Rechner eingebaut haben, müssen Sie nun die Treibersoftware installieren. Für die Installation unter Novell NetWare 4.x und 5.x werden zwei Treiberversionen geliefert:
»Stand Alone-CAPI« und »CAPI-Manager«. Wenn Sie den CAPI-Manager verwendenmöchten, achten Sie darauf, dass dieser vor der Installation des dBoard2-Controllers bereits eingerichtet ist.

Der Novell CAPI-Manager ist Bestandteil des Betriebssystems Novell NetWare 4.xund 5.x. Der Ablauf von Installation und Deinstallation ist – unabhängig von derTreiberart – für alle Novell NetWare-Versionen identisch. Die Konfiguration und Inbetriebnahme des dBoard2-Controllers hingegen ist für Stand Alone-CAPI undCAPI-Manager unterschiedlich.

Wenn Sie das dBoard2 unter Novell Netware 5.x mit dem Novell CAPI-Managerinstallieren wollen, müssen Sie diesen zuvor aktualisieren. Kopieren Sie dazu die Datei CAPIMGR.NLM von der Installations-CD aus dem Unterverzeichnis»DBOARD2\NETWARE\CAPIMGR\NOVELL« in das »SYSTEM-Verzeichnis« Ihrer Net-Ware 5.x-Installation und starten Sie anschließend den Server neu.

Die folgenden Installationsbeschreibungen gelten für Novell Netware 5.x. Für Novell NetWare 4.x können sich geringfügige Abweichungen ergeben. Gehen Sie folgendermaßen vor:

Richten Sie vor Beginn der Treiber-Installation für die Betriebssysteme Novell Netware 4.x das bei Novell erhältliche Patch »LANDR9« oder höher ein.

Legen Sie die Installations-CD ein und geben Sie an der Konsolefolgenden Befehl ein:

LOAD NWCONFIG

Damit wird das Installationsmenü des Novell NetWare-Servers geladen. Das Menü»Installation Options« des Installationsprogramms erscheint.

Wählen Sie den Menüpunkt »Product options« und drücken Sie die Eingabe-Taste.Es erscheint ein Menü mit Namen »Other Installation Items/Products«. Wählen Siehier den Menüpunkt »Install a product not listed« und drücken Sie die Eingabe-Taste.

Standardmäßig wird das Diskettenlaufwerk A:\ vorgeschlagen. Um einen anderenPfad anzugeben, drücken Sie [F3].

Sollte auf Ihrem NetWare-Server das CD-ROM-Laufwerk nicht gemountet sein, sokopieren Sie das Verzeichnis »DBOARD2\NETWARE\CAPI20« bzw. »DBOARD2\NETWARE\CAPIMGR« der Installations-CD von einer Workstation aus, auf den Novell NetWare-Server und fahren mit dem nächsten Installationsschritt fort.

Geben Sie den Pfad zu den Installationsdateien an.

Wählen Sie im nächsten Schritt »Install on this server« und drücken Sie die Eingabe-Taste. Die Dateien für den dBoard2-Controller werden nun in das Verzeichnis »SYS:SYSTEM« kopiert.

Drücken Sie [Esc], um die Anzeige zu verlassen. Drücken Sie so oft [Esc], bis Sie wieder im Menü »Configuration Options« sind und wählen Sie dort »Exit«, um das Installationsprogramm zu beenden.

dBoard2 konfigurieren mit Stand Alone-CAPI

Um das dBoard2 mit Stand Alone-CAPI zu konfigurieren, geben Sie an der Serverkonsole Folgendes ein:

LOAD B1SETUP

Eine Übersicht der verfügbaren ISDN-Controller mit Angaben zu I/O-Adresse, Interrupt und D-Kanal-Protokoll erscheint.

Um einen Controller aus dieser Liste zu konfigurieren, bewegen Sie die Markierung mit Hilfe der Cursortasten auf den entsprechenden Eintrag unddrücken die Eingabe-Taste. Das Konfigurationsfenster für den Controller erscheint. Hier nehmen Sie Einstellungen zu D-Kanal-Protokoll, Anschlussart(Anlagenanschluss/Point-to-Point bzw. Mehgeräteanschluss/Point-to-Multipoint) und zur automatischen Datenkompression vor.

Bewegen Sie die Markierung zum jeweiligen Eintrag und drücken Sie die Eingabe-Taste.

Wählen Sie die gewünschte Einstellung aus der eingeblendeten Übersicht aus odertragen Sie den neuen Wert in das aktivierte Feld ein.

Verlassen Sie die Konfiguration mit [Esc] und beantworten Sie die Frage, ob die Einstellungen gespeichert werden sollen, mit »Yes«. Auf diese Weise können Sie die Einstellungen des dBoard2 jederzeit ändern.

Laden der CAPI-Treiber

Um den Controller zu laden, geben Sie an der Serverkonsole folgenden Befehl ein:

LOAD CAPI20

Es erscheint die Meldung, dass der ISDN-Controller geladen wurde und die CAPI zur Verfügung steht.

Um den Controller zu entladen, geben Sie ein:

LOAD CAPI20 -D

Das System gibt eine entsprechende Meldung aus.

dBoard2 konfigurieren mit dem CAPI-Manager

Um das dBoard2 mit CAPI-Manager zu konfigurieren, geben Sie an der Serverkonsolefolgenden Befehl ein:

LOAD INETCFG

Das Dialogfenster »Internetworking Configuration« wird geöffnet.

Wählen Sie den Eintrag »Boards«. Eine Übersicht der bereits konfiguriertenISDN-Controller erscheint.

Binden Sie das dBoard2 ein, indem Sie [Einfg] auf Ihrer Tastatur drücken.

Markieren Sie im Dialogfenster »Select A Driver« den Listeneintrag »WHSMCAPI«und drücken Sie die Eingabe-Taste.

Vergeben Sie im nächsten Fenster einen Namen für den dBoard2-Controller, z.B.»dBoard2«.

Im Dialogfenster »WHSMCAPI Board Configuration« wird das dBoard2 konfiguriert.Markieren Sie den Eintrag »Select To View« und drücken Sie die Eingabe-Taste.

Beantworten Sie die Frage nach dem automatischen Laden mit »Yes«.

Wählen Sie im folgenden Dialog den ISDN-Controller »AVM B1PCI« aus und drückenSie die Eingabe-Taste.

Nehmen Sie dann Ihre Einstellungen vor: PCI-Slot, Maxports (feste Größe 2),Switch Type (D-Kanal-Protokoll), Line Config (MTP = Mehrgeräteanschluss oder PPT=Anlagenanschluss), Automatic Data Compression.

Verlassen Sie den Konfigurationsdialog, indem Sie zweimal [Esc] drücken und speichern Sie auf Aufforderung Ihre Angaben mit »Yes«.

Im letzten Schritt der Konfiguration soll nun das System aktualisiert und derISDN-Controller geladen werden.

Wählen Sie dazu im Fenster »Internetworking Configuration« den Befehl »Reinitialize System« und bestätigen Sie mit »Yes«.

Verlassen Sie die Konfiguration mit »Exit« und beantworten Sie die Frage, ob dieEinstellungen gespeichert werden sollen, mit »Yes«.

Damit sind Konfiguration und Laden des dBoard2-Controllers mit dem CAPI-Managerabgeschlossen. Auf diese Weise können Sie die Einstellungen jederzeit ändern.

Zusätzliche dBoard2-Controller installieren

Möchten Sie weitere dBoard2-Controller installieren, geben Sie an der Serverkonsole den folgenden Befehl ein:

LOAD B1SETUP

In der Übersicht verfügbarer ISDN-Controller drücken Sie die [Einfg]-Taste. Es wird jeweils ein weiterer ISDN-Controller angezeigt einschließlich verfügbarer I/O-Adresse und Interrupt.

dBoard2-Controller deinstallieren

Zur Deinstallation des dBoard2-Controllers unter Novell Net-Ware 4.x und 5.x gehen Sie folgendermaßen vor:

Geben Sie an der Serverkonsole den folgenden Befehl ein:

LOAD NWCONFIG

Das Menü »Installation Options« des Installationsprogramms erscheint.

Wählen Sie den Menüpunkt »Product options« und drücken Sie die Eingabe-Taste.Es erscheint ein Menü mit dem Namen »Other Installation Items/Products«.

Wählen Sie hier den Menüpunkt »View/Configure/Remove installed products«und drücken Sie die Eingabe-Taste. Eine Liste der bereits installierten Programme wird angezeigt.

Bewegen Sie die Markierung auf den Eintrag »AVM ISDN-Controller B1« und drückenSie die [Entf]-Taste. Bestätigen Sie mit »Yes«.

Drücken Sie so oft »Esc«, bis Sie wieder im Menü »Installation Options« sind undwählen Sie anschließend »Exit«, um den Dialog zu verlassen.

Die Deinstallation des dBoard2 unter Novell NetWare ist damit abgeschlossen.
