Q-100.193
Parameter der TLD.INI


Frage
Problem Welche Funktionen haben die Parameter in der TLD.INI ?
Produkt TLD (allgemein)
Priorität normal
Effekt normal
Datum 17.02.2004

Antwort
In der folgenden Auflistung erhalten Sie eine Erläuterung der einzelnen Parameter. Nach dem Eintragen eines Parameters und Speichern der TLD.INI müssen Sie den entsprechenden Port neu starten.

Name =
Dieser Punkt beschreibt den Namen der ISDN-Karte, entsprechend Ihrer Angabe während des Setups.

CAPIVersion = 0
Dieser Eintrag registriert die eingesetzte Version des CAPI-Treibers, welche automatisch erkannt wird.
0 = Unbekannte CAPI-Version
1 = CAPI 1.1
2 = CAPI 2.0

Controller = 0
Wenn Sie mehrere ISDN-Karten (aber keine S2M) einsetzen, müssen Sie die Ports den verschiedenen Karten zuweisen. Die Nummerierung der Karten ist von der eingesetzten CAPI-Version abhängig.
Falls ein CAPI-1.1-Treiber läuft, wird die erste Karte mit dem Wert "0" bezeichnet, die zweite mit "1", usw. Bei CAPI 2.0 wird die erste Karte mit dem Wert "1" bezeichnet, die zweite mit "2", usw.Diesen Parameter können Sie auch in der Port-Konfiguration (DvISE Administrator bzw. Service Layer) setzen.

Routing = 0
Dieser Eintrag bestimmt die Verteilmethode des jeweiligen Ports.
Die "0" steht dabei für CSID (Sendekennung der Gegenstelle), die "1" für DDI (Punkt-zu-Punkt-Anschlüsse), die "2" für MSN (Punkt-zu-Mehrpunkt-Anschluss).
Sie müssen die ISDN-Karte entsprechend des eingesetzten Anlagenanschlusseskonfiguriert haben, da ansonsten keine korrekte Funktion der TLDs gewährleistet ist.

DDIPrefix =
Dieser Parameter ist nur gültig in Verbindung mit »Routing Method=DDI«.
Er definiert eine dreistellige Zahl, die den ausgewerteten DDI-Ziffernvorangestellt wird. Bei der Konfiguration der Verteilmethode nach DDI im DvISE Administrator (bzw. Service Layer) müssen Sie die dreistellige Zahl des »DDIPrefix« vor jeder Durchwahlziffer mit angegeben.
»DDIPrefix« sollten Sie nur verwenden, wenn mehrere NT1- (TK-Anlagen-) Anschlüsse verwendet werden, welche sich in ihren Nummernbereichen überschneiden.

DDILenOnly = False
Dieser Parameter ist nur wirksam in Verbindung mit »Routing Method=DDI«. Hiermit bestimmen Sie, ob die Entgegennahme von DDI-Informationen direkt nach Empfang der unter »DDILength« definierten Anzahl von Ziffern abgebrochen werden soll.
Durch aktivieren des Schalters bewirken Sie diesen Abbruch und dies hat somit ggf. einen schnelleren Verbindungsaufbau zur Folge.

DDILength = 3
Dieser Parameter ist nur gültig in Verbindung mit »Routing Method=DDI«. Er bestimmt die Länge der DDI-Durchwahlen, die am entsprechenden Anschluss zur Verfügung stehen. Gültige Werte sind 1 – 5.

DDIOffset = -1
Dieser Parameter ist nur wirksam in Verbindung mit »Routing Method=DDI«. Hiermit bestimmen Sie, von welcher Position (die Nummerierung beginnt bei "0") im gesamten übergebenen DDI-String ausgehend, die mit dem Parameter »DDILength« bestimmte Anzahl von Ziffern als DDI gewertet werden.
Die gültigen Werte liegen zwischen -1 und 10, wobei "-1" bedeutet, dass unabhängig von der Länge des gesamten DDI-Strings immer die letzten Ziffern gewertet werden.

DDIRange = 0 - 0
Dieser Parameter ist nur wirksam in Verbindung mit »Routing Method=DDI«. Hiermitbestimmen Sie den Rufnummernbereich, in welchem der Port den Ruf annehmen soll.
Beispiel: 100 – 199

DDIOut =
Dieser Parameter ist nur wirksam in Verbindung mit »Routing Method=DDI«.
Hier tragen Sie eine ausgehende Kennung für die TK-Anlage ein.
Hinweis:
Dieser Eintrag ist nicht identisch mit der Vorwahl für Amtsholung oder einem Kostenschlüssel. Einige TK-Anlagen benötigen eine Kennnummer des Anschlusses, welcher hier eingesetzt wird. Nähere Informationen zu dieser Rufnummer erhalten Sie beim entsprechenden TK-Anlagen-Hersteller.

TxMSN =
Hier tragen Sie ein ausgehende Kennung für die TK-Anlage ein.
Hinweis:
Dieser Eintrag ist nicht identisch mit der Vorwahl für Amtsholung oder einem Kostenschlüssel. Einige TK-Anlagen benötigen eine Kennnummer des Anschlusses, welcher hier eingesetzt wird. Nähere Informationen zu dieser Rufnummer erhalten Sie beim entsprechenden TK-Anlagen-Hersteller.

RxMSN =
Hiermit bestimmen Sie den Rufnummernbereich, in welchem der Port den Ruf annehmen soll.
Beispiel: 1, 2, 5, 7, 8

TimeOut = 60
Hier definieren Sie die Einstellung des automatischen TimeOuts, wenn z.B. keine Datenübertragung mehr stattfindet. Die Einheit ist Sekunden.

DebugMode = 3
Hier können Sie die Einstellungen der Monitor-Informationen vornehmen, wobei "0"bedeutet, dass keine Anzeige erfolgt, und "5" meint, dass alle Angaben der Informationen angezeigt werden.
Um eine Fehlersuche und die Behebung eventueller Probleme zu erleichtern, sollte der Wert auf "5" gesetzt werden.

IsULaw = FALSE
Hier definieren Sie die Datenformatierung bei eingehenden Srachnachrichten. Im Allgemeinen kann dieser auf »FALSE« gesetzt bleiben; dies entspricht der "u-Law"-Einstellung.

AnswerDataCalls = TRUE
Hier können Sie bestimmen, ob Datenanrufe wie z.B. FaxPlus auf dem TLD abgelehnt werden sollen. Diese Einstellung kann vorgenommen werden, wenn Sie z.B. einen MPR auf der ISDN-Karte zusätzlich gebunden haben oder ein Programm auf der gleichen Karte aktiv ist.

ISDN_Service_FaxG3 = 1
Über diesen Parameter änderen Sie die Einstellung, über die der Versand eines Faxes (G3) erfolgen soll, wenn eine ISDN-Karte mit Faxaufsatz eingesetzt wird.
Eine Umstellung ist im Allgemeinen nicht notwendig; ggf. kann bei Übertragungs- oder Handshake-Problemen eine Umstellung erfolgen. Der Eintrag "0" steht für a/b Services, "1" steht für analoge Telefone.

DynamicCSID = True
Mit diesem Wert geben Sie an, ob die Durchwahlinformation mit der Sendekennung der Gegenstelle überschrieben werden soll oder die entsprechende eigene Durchwahl angezeigt werden soll.
Hinweis:
Dieser Wert gilt nur in Verbindung mit DDI Routing.

HeaderLineFeed = 0
Hiermit können Sie die entsprechenden Zeilen eines Faxes eingegeben, die alsLeerzeilen übertragen werden, um evtl. Verluste von Informationen des Fax-Headers zu vermeiden.

DefaultFaxPlus = False
Mit dieser Einstellung können Sie definieren, ob der TLD zuerst versuchen soll, anstelle einer Fax-G3-Verbindung eine FaxPlus-Verbindung aufzubauen.

BusyCharge =
Hiermit initiieren Sie eine Angabe der angezeigten Gebühreneinheit, wenn z.B. die Gegenstelle nicht erreichbar ist. Der Parameter ist zur Zeit nur sinnvoll in Ländern einzusetzen, wo die D-Kanal-Verbindung nicht kostenlos ist, wie z.B. in der Schweiz.

HandleUnloadEvent = True
Hiermit aktivieren bzw. deaktivieren Sie die Abfrage bei Entladung des TLDs (nur NetWare).

MaxTimeActive = 600
Mit diesem Paramter definieren Sie die maximale Dauer einer Voice-Verbindung in Sekunden. Der Port trennt die Verbindung dann automatisch.

MaxPageSize = 5.242.880
Hiermit bestimmen Sie die maximale Größe einer zu empfangenen Seite. Ist diese Größe erreicht,wird der Empfang abgebrochen.

Optimized = 1
DvISE erzeugt bei abzuspielenden Wave-Dateien in einem Script eine Kopie imA-LAW-Format. Mit der Einstellung "1" verbessern Sie die System-Performance.
Folgende Einstellungen sind möglich:
0 = Keine Erstellung von A-LAW-Dateien.
1 = Erstellung von A-LAW-Dateien nur aus Wave-Dateien, die im Verzeichnis»DAVID\TLD\RESOURCE« liegen.
2 = Erstellung von A-LAW-Dateien aus allen Wave-Dateien, die im Script mitkomplettem Dateinamen inklusive Extension aufgerufen werden.

ProtocolFromEntry = False
Hierdurch können Sie festlegen, ob CLI der TK-Anlage anstelle der Absenderkennung (CSID) eingetragen wird, wenn der Wert »True« eingestellt ist.

BeepOnRx = False
Nur gültig bei NetWare:
Hiermit können Sie bei jedem eingehenden Fax ein Server-Beep ausgegeben lassen.

FaxTxScript = False
Mit diesem Parameter bewirken Sie, dass beim Start eines Scripts ein Faxauftrag generiert wird. Diesen sollten Sie nur dann eiesetzen, wenn eine solche Funktion im Script erforderlich ist (nur bei ausgehenden Nachrichten).

Script=\\SERVERNAME\DAVID\TLD\COMMON\TLD.DCC
Hiermit bestimmen Sie den Pfad eines individuellen Scripts, welches nur auf diesem TLD ausgeführt werden soll, anstelle des globalen Scripts (TLD.DCC).

AllowCapiMgr = True
Damit sorgen Sie dafür, dass auch mit einem CAPI Manager die ECT-Funktionen ausgeführt werden.

DisableFaxPolling = False
Einige ISDN-Karten warten bei Faxempfang zunächst eine Zeit von mehreren Sekunden, ob die Gegenstelle ein Fax-Polling Signal sendet, anstatt direkt ein Fax zu senden.
Wenn Sie den Wert auf »True« setzen, bewirken Sie, dass Fax-Polling Calls direkt abgewiesen werden und somit diese Wartezeit entfällt. Die Annahme eingehender Faxen geschieht also wesentlich schneller.
Beachten Sie jedoch, dass ein Bereitstellen von Faxdokumenten für Fax-Polling in diesem Fall generell nicht möglich ist.

ValidNumberSigns = *
Hiermit können Sie Sonderzeichen (# oder *) an die TK Anlage übergeben, wenn Sie diese zu Statistikzwecken vor jeder Faxnummer benötigen.
Seit SP3 für DvISE 6.6 gibt es dafür diesen Eintrag.
Für andere Zeichenfolgen ersetzen Sie einfach das »*« gegen ein anderes gewünschtes Zeichen bzw. Zeichenkette.
