Q-109.993
Rechte in der Benutzerkonfiguration bei David.zehn! und David.fx


Frage
Problem Welche Bedeutungen haben die einzelnen Rechte, die in der Benutzerkonfiguration im David.Administrator unter »System > Benutzer« auf der Registerkarte »Rechte« gesetzt werden können?
Produkt David.Administrator
Priorität normal
Effekt normal
Datum 5.11.2009

Antwort
Die Benutzerkonfiguration im David.Administrator erlaubt auf der Registerkarte »Rechte« die Aktivierung von Rechten in den folgenden Bereichen:
  • Allgemein
  • Erweitert
  • Nachrichtenempfang
  • Nachrichtenversand
  • Voreinstellungen
  • StrongBox (Datensicherung)

Allgemein

  • Remote Access nutzen
    Diese Berechtigung muss jedem Benutzer zugewiesen werden, der in der Lage sein soll, Fernzugriff auf den David Server auszuüben. In diesem Fall kann der Anwender z.B. den WebAcces ausüben oder mit einem mobilen Gerät (z.B. Notebook oder iPhone) per Internet die David Dienste nutzen sowie auf den Inhalt seiner Ordner zugreifen.

    Hinweis:
    Um tatsächlich Fernzugriff ausüben zu können, müssen der David Server und das David Ordnersystem entsprechend freigegeben bzw. eingerichtet sein.

  • Ordner-Eigenschaften bearbeiten
    Ist dieses Kontrollkästchen aktiviert, so besitzt der Anwender die Berechtigung, auf die Eigenschaften der David Ordner (z.B. Datenbereinigung, Sortierung, etc.) in seinem David Client zuzugreifen und diese ggf. zu verändern. In diesem Fall kann er einen Ordner im Ordnersystem des David Client markieren und über das Menü Datei bzw. über die rechte Maustaste die Ordner-Eigenschaften aufrufen und hier entsprechende Änderungen vornehmen. Deaktivieren Sie das Kontrollkästchen, wenn Sie dem Benutzer diese Berechtigung entziehen wollen.
  • PGP-Signatur/-Verschlüsselung nutzen
    Ist dieses Kontrollkästchen aktiviert, besitzt der Anwender die Berechtigung zur Nutzung des clientbasierten Verschlüsselungsverfahrens PGP. Somit ist es ihm möglich, PGP-signierte bzw. PGP-verschlüsselte Nachrichten direkt über den David Client zu empfangen und zu versenden. Voraussetzung hierfür ist, dass PGP auf der Arbeitsstation dieses Benutzers installiert und korrekt konfiguriert ist. Trifft dies zu, so stehen im Nachrichten-Editor zusätzliche Schaltflächen für die PGP-Ver- und Entschlüsselung sowie für die PGP-Signatur zur Verfügung.
    Deaktivieren Sie das Kontrollkästchen, um dem Benutzer diese Berechtigung zu nehmen. In diesem Fall sind die zusätzlichen Schaltflächen im Nachrichten-Editor generell nicht verfügbar, auch wenn PGP auf der Workstation installiert ist.
  • S/MIME-Verschlüsselung nutzen
    Ist dieses Kontrollkästchen aktiviert, besitzt der Anwender die Berechtigung zur Nutzung des clientbasierten Verschlüsselungsverfahrens S/MIME. Somit ist es ihm möglich, S/MIME-verschlüsselte Nachrichten direkt über David zu empfangen und zu versenden. Voraussetzung hierfür ist, dass der Benutzer über ein persönliches S/MIME-Zertifikat inklusive der dazugehörigen Schlüssel verfügt (z.B. ein David Signaturzertifikat) und dieses in das Betriebssystem seiner Arbeitsstation korrekt eingebunden ist. Trifft dies zu, so steht im Nachrichten-Editor eine zusätzliche Schaltfläche für die Nutzung der S/MIME-Verschlüsselung zur Verfügung.

    Deaktivieren Sie das Kontrollkästchen, um dem Benutzer diese Berechtigung zu nehmen. In diesem Fall sind die zusätzlichen Schaltflächen im Nachrichten-Editor generell nicht verfügbar, auch wenn S/MIME-Zertifikate auf der Workstation vorhanden sind.

    Hinweis:
    Beachten Sie hierzu auch die Einstellung auf dem Register »Sicherheit« in der Systemkonfiguration. Ist diese Option deaktiviert, steht dem Benutzer die Nutzung der S/MIME-Verschlüsselung generell nicht zur Verfügung – auch dann nicht, wenn ihm in der Benutzerkonfiguration diese Berechtigung explizit erteilt wird.

  • Zertifikate auch beim Remote Access verwenden
    Aktivieren Sie diese Option, wenn die über David bezogenen Signaturzertifikate des Benutzers zentral auf dem David Server installiert werden sollen. Bei dieser Einstellung steht dem Benutzer die digitale Signatur bei der Arbeit mit David generell zur Verfügung, unabhängig davon, welche Arbeitsstation er nutzt.

    Ist dieses Kontrollkästchen deaktiviert, erfolgt keine Speicherung der Signaturzertifikate dieses Benutzers auf dem David Server. Stattdessen wird das Zertifikat ausschließlich auf der Arbeitsstation eingebunden, an der die Installation durchgeführt wird.

    Erfolgt eine nachträgliche Deaktivierung dieses Kontrollkästchens, nachdem bereits mindestens ein Zertifikat dieses Benutzers auf dem David Server gespeichert wurde, erhält der Benutzer beim nächsten Start des David Client eine entsprechende Meldung angezeigt. Er hat dann die Möglichkeit, sein(e) Zertifikat(e) vom David Server auf die Arbeitsstation, die er momentan verwendet, übertragen zu lassen. Erfolgreich übertragene Zertifikate werden anschließend automatisch vom David Server entfernt.

    Hinweis:
    Bei zentraler Speicherung auf dem David Server werden die Zertifikate im persönlichen Root-Verzeichnis de jeweiligen Benutzers unterhalb von »ARCHIVE\USER« abgelegt. Der Dateiname besteht dabei jeweils aus der eMail-Adresse, für die das Zertifikat ausgestellt wurde, und der Extension ».P12«, also z.B.:
    »DAVID\ARCHIVE\USER\10004000\LEO.BENZ@EXAMPLE.NET.P12«

Erweitert

  • David Administrator
    Besitzt der Anwender diese Berechtigung, so hat er die Möglichkeit, den David.Administrator direkt aus dem David Client an seiner Arbeitstation aufzurufen. Voraussetzung hierfür ist, dass diese Arbeitstation unter Windows XP oder höher betrieben wird, und dass der betreffende Anwender über Zugriffsrechte auf das Verzeichnis »\DAVID\CODE« des David Servers verfügt. In diesem Fall enthält dann die »Allgemeine Symbolleiste« im David Client die Schaltfläche »David.Administrator«.

    Hinweis:
    Der Anwender hat durch dieses Recht die Möglichkeit, beim Identifizieren von Nachrichten die Verteilkennung zu ersetzen (Funktion »Identifizieren« im David Client).

    Außerdem haben David Benutzer, die über dieses Recht verfügen, die Möglichkeit, Ordner direkt aus dem David Client heraus mit dem Windows Explorer zu öffnen. Dazu steht ihnen im Ordner-Kontextmenü (Rechtsklick) die zusätzliche Option »Im Explorer öffnen« zur Verfügung.

  • Versandaufträge aller Benutzer anzeigen
    Bei Aktivierung dieser Option kann der Anwender in seinem Auftragsbuch neben seinen eigenen auch die Versandaufträge aller anderen Anwender sehen. Hierzu muss er im David Client den Schalter »Aufträge aller Benutzer« aktivieren (unter »Ansicht > Filter > Optionen«). Ohne das zusätzliche Leserecht auf die Ordner der Anwender kann er jedoch die eigentlichen Nachrichten nicht öffnen. Es geht hier nicht um eine globale Ansicht aller versendeten oder empfangenen Nachrichten, sondern nur um die Nachrichten, die sich im Auftragsbuch der Benutzer befinden.
  • DFÜ-Verbindung aktivieren
    Falls diese Option aktiviert ist, erscheint in der Benutzer-Symbolleiste im David Client ein Symbol »Senden/Empf.«. Mit diesem kann der Benutzer von seiner Arbeitsstation aus die im PostMan oder Grabbing Server eingerichteten Verbindungen aktivieren, um eMails zu versenden oder zu empfangen.
  • Message Identification Services (MIS) anpassen
    Ist diese Option aktiviert, besitzt der Benutzer die Berechtigung, eigene Einstellungen für die Identifizierung seiner empfangenen Nachrichten zu definieren. Zu diesem Zweck steht ihm im David Client eine zusätzliche Registerkarte im Dialog »Ordner-Eigenschaften« seines Eingangsordners zur Verfügung. Hier hat er die Möglichkeit, individuelle Anpassungen an den Einstellungen zur Nachrichtenidentifizierung vorzunehmen. Diese Einstellungen gelten jedoch ausschließlich für den eigenen Eingangsordner des Benutzers.

    Hinweis:
    Diese Funktionen können nur bei Einsatz des kostenpflichtigen Backline Features »Message Identification Services« genutzt werden. Hierzu muss im David.Administrator der Punkt »Identifizierung für eingehende eMails« aktiviert sein. Sie finden diesen Punkt in der Konfiguration der »Backline Features for David« auf der »Registerkarte Identifizierung«.

  • Ordner-Zugriffsrechte anpassen
    Aktivieren Sie diese Option, wenn Sie dem Benutzer erlauben möchten, im David Client Verzeichnisrechte auf Ordner zu vergeben. In diesem Fall kann die Vergabe der Verzeichnisrechte also direkt im David Client erfolgen, ohne dazu auf Betriebssystemebene wechseln zu müssen. Zu diesem Zweck steht in den Ordner-Eigenschaften auf dem Register »Zugang« die Schaltfläche »Berechtigungen« zur Verfügung, über die der zugehörige Betriebssystemdialog aufgerufen werden kann.

    Hinweis:
    Diese Berechtigung sollte ausschließlich an erfahrene Anwender vergeben werden, da sie u.U. schwerwiegende Eingriffe in das Gesamtsystem ermöglicht.

  • Ordner verschieben und kopieren
    Ist die Option »Ordner verschieben und kopieren« aktiviert, so besitzt der Anwender die Berechtigung, im David Client per »Drag & Drop« Ordner zu verschieben und zu kopieren.
  • Feeds nutzen
    Ist diese Option aktiviert, stehen dem Anwender die »RSS-Feeds« Funktionen im David Client zur Verfügung, die es ihm erlauben, verschiedene so genannte RSS-Feeds (z.B. News-Feeds oder Video/Audio Casts) zu nutzen. Deaktivieren Sie das Kontrollkästchen, um dem Benutzer diese Berechtigung zu nehmen. In diesem Fall stellt der David Client dem Anwender keine Funktionen zur Nutzung und zur Konfiguration von Feeds zur Verfügung.
  • Live Forum nutzen
    Ist diese Option aktiviert, steht dem Anwender die Nutzung des »Live Forums« zur Verfügung. In diesem Fall enthält das persönliche Menü, das im Hauptmenü des David Client aufrufbar ist, einen zusätzlichen Eintrag, über welchen der Benutzer das Live Forum direkt aufrufen kann. Deaktivieren Sie diese Option, um die Nutzung des Live Forums zu unterbinden.

Nachrichtenempfang

  • Nachrichten manuell verteilen
    Ist dem Anwender diese Berechtigung zugewiesen, so kann er Nachrichten an andere David User im Netz verteilen. Im David Client steht ihm hierzu die Funktion »Verteilen...« zur Verfügung.
  • Absender identifizieren
    Besitzt der Anwender diese Berechtigung, so kann er bei empfangenen Nachrichten die Kennung der Gegenstelle »lernen« lassen. Das bedeutet, die jeweilige Kennung wird in der Identifizierungsliste eingetragen (siehe Menü »System > Verteilung > Identifizierung«). Im David Client steht ihm hierzu die Funktion »Identifizierung« zur Verfügung. Soll der Anwender über diese Funktion zusätzlich noch die Verteilkennung ändern können, benötigt er auch das Benutzerrecht »David Administrator«.
  • Verteilregeln erstellen/bearbeiten
    Ist dieses Kontrollkästchen aktiviert, so besitzt der Anwender die Berechtigung, im David Client Verteilregeln zu erzeugen. In diesem Fall steht ihm unter dem Hauptmenüpunkt »Aktionen« sowie im Ordner-Kontextmenü der Eintrag »Regeln...« zur Verfügung, der es erlaubt, eigene Verteilregeln für beliebige David Ordner zu erstellen. Deaktivieren Sie das Kontrollkästchen, wenn Sie dem Benutzer diese Berechtigung entziehen wollen.

Nachrichtenversand

  • Rundsendungen versenden
    Ist dieses Kontrollkästchen aktiviert, so besitzt der Anwender die Berechtigung, Rundsendungen unter Verwendung des David Befehls »@@RUNDSENDUNG« zu erstellen. Deaktivieren Sie dieses Kontrollkästchen, wenn Sie dem Benutzer diese Berechtigung entziehen wollen. Entdeckt der Service Layer dennoch Aufträge dieses Users, in denen der Befehl »@@RUNDSENDUNG« enthalten ist, so werden diese nicht versendet, sondern mit der Fehlermeldung »Unerlaubte Rundsendung« in das Sendebuch gestellt.
  • Externe Faxe versenden
    Aktivieren Sie diese Option, wenn dem Benutzer der Versand von Faxen sowohl intern im lokalen Netzwerk als auch extern über das Telefonnetz möglich sein soll. Entfernen Sie den Haken aus dem Kontrollkästchen, wenn der Benutzer nicht die Möglichkeit besitzen soll, externe Faxe zu versenden. Der interne Faxversand ist auch in dieser Einstellung weiterhin möglich.
  • Externe eMails versenden
    Aktivieren Sie diese Option, wenn dem Benutzer der Versand von eMails sowohl intern im lokalen Netzwerk als auch extern über das Internet möglich sein soll. Entfernen Sie den Haken aus dem Kontrollkästchen, wenn der Benutzer nicht die Möglichkeit besitzen soll, externe eMails zu versenden. Der interne Versand von eMails ist auch in dieser Einstellung weiterhin möglich.
  • Externe SMS Nachrichten versenden
    Aktivieren Sie diese Option, wenn dem Benutzer der Versand von SMS Nachrichten sowohl intern im lokalen Netzwerk als auch extern über das Mobilfunk-Netz möglich sein soll. Entfernen Sie den Haken aus dem Kontrollkästchen, wenn der Benutzer nicht die Möglichkeit besitzen soll, externe SMS Nachrichten zu versenden. Der interne SMS-Versand ist auch in dieser Einstellung weiterhin möglich.
  • Externe TMails versenden
    Aktivieren Sie diese Option, wenn dem Benutzer der Versand von TMails sowohl intern als auch extern möglich sein soll. Entfernen Sie den Haken aus dem Kontrollkästchen, wenn der Benutzer nicht die Möglichkeit besitzen soll, externe TMails zu versenden. Der interne Versand von TMails ist auch in dieser Einstellung weiterhin möglich.

Voreinstellungen

  • Öffentliche Benutzerdaten
    Aktivieren Sie dieses Kontrollkästchen, wenn es auch anderen David Usern möglich sein soll, die Daten dieses Anwenders (z.B. die Unterschrift) zu verwenden. Anwender können dann Sendeaufträgen durch Verwendung des Befehls »@@OWNERNAME« einen anderen Besitzer zuweisen und dessen Unterschrift über den Befehl »@@UNTERSCHRIFT« in die Nachricht einsetzen lassen, sofern für diesen User der Schalter »Öffentliche Benutzerdaten« gesetzt ist.
  • Kostenschlüssel mitwählen
    Bei Aktivierung des Kontrollkästchens wird der eingetragene Kostenschlüssel bei jedem Wahlversuch dieses Anwenders mitgewählt. Einige Telefonanlagen können diesen Kostenschlüssel z.B. für eine Kostenstellenzuweisung herausfiltern.

    Hinweis:Falls die Telefonanlage eine solche Funktion nicht unterstützt, sollten Sie auf keinen Fall diese Option aktivieren.

StrongBox (Datensicherung)

  • StrongBox Ordner anzeigen
    Wählen Sie diese Option, wenn der Benutzer das Recht haben soll, StrongBox Backups zu nutzen. Nur wenn der Benutzer über diese Berechtigung verfügt, wird ihm im David Client der Ordner »StrongBox Images« angezeigt, in dem sich die zur Verfügung stehenden Backups befinden.
  • Mounten lokaler StrongBox Images
    Ist diese Option markiert, hat der Benutzer die Berechtigung, lokal in seinem David Client StrongBox Images zu mounten, um auf den darin enthaltenen Inhalt zuzugreifen. Im David Client steht in diesem Fall eine entsprechende Funktion unterhalb des Hauptmenüpunktes »Optionen« zur Verfügung.
  • Wiederherstellen (Restore) von StrongBox-Inhalten
    Aktivieren Sie diese Option, wenn der Benutzer das Recht haben soll, Inhalte aus StrongBox Backups in das David Ordnersystem auf dem David Server zurückzusichern. Im Kontextmenü von Ordnern, die sich innerhalb eines Backups befinden, steht dann die Funktion »Restore« zur Verfügung.
  • Drag & Drop aus StrongBox Ordnern
    Besitzt der Benutzer dieses Recht, hat er die Möglichkeit, nicht nur einzelne Dokumente sondern auch ganze Ordner per Drag & Drop aus StrongBox Backups zu kopieren. Der Inhalt der Backups selbst kann dadurch nicht verändert werden.

Zurück zum Artikelanfang

Außerdem können Sie im Bereich »Einschränkungen« die »max. Anzahl von Adressaten« festlegen, die der Benutzer für einen Sendeauftrag angeben darf. Versucht dieser Benutzer eine Rundsendung zu versenden, deren Adressliste die hier eingestellt Anzahl von Empfängeradressen überschreitet, so wird eine entsprechende Fehlermeldung in seinem Ausgangsordner erstellt. Der Versand findet in diesem Fall nicht statt.
