ServicePack 1b - Code History für DvISE 6.5 (Windows NT/2000)


Frage
Produkt David
Priorität keine
Effekt harmlos
Datum 22.01.2010

Antwort
Service Pack 1b

Code History (WIndows NT/2000)

Die folgenden Neuerungen und Bugfixes sind im Service Pack enthalten:

  1. DvISE Service Layer/DvISE Administrator
  2. Tobit InfoCenter
  3. Tobit Image Editor
  4. DvISE TLD für ISDN Karten
  5. DvISE TLD für SMS Modems
  6. DvISE TLD für Brooktrout Karten
  7. DvISE PostMan
  8. DvISE Grabbing Server
  9. DvISE Mail Access Server
  10. DvISE Discussion Server
  11. DvISE Replica
  12. Aktuelle Dateiversionen

1. DvISE Service Layer/DvISE Administrator

  • Mögliche Probleme bei Nutzung der Sammelliste
  • Evtl. neue User Archives, die nach einem Update einer früheren David Version auf David 6.5 fehlten, werden beim ersten Start des Service Layers erstellt
  • Möglicher Serverabsturz während der Datenbereinigung bei falschen Servernamen in der Archivestruktur (Archive.dir)
  • In der Konfiguration des DvISE Host war die Controller Auswahl fehlerhaft
  • In dem Feld eMail Adresse der Benutzerkonfiguration konnten mehr Zeichen eingetragen werden, als gespeichert wurden
  • Mögliche C/S Probleme beim Betrieb auf einem Terminal Server (nur bei der CD vom 18.09.2000)
  • Mögliche Probleme beim Reparieren von Attachments, die Viren enthielten
  • Möglicher SL Absturz bei Verwendung des RAND Befehls mit einer ungültigen Parameterlänge
  • Wenn das Papierkorb Archive nicht mehr vorhanden war, wurden manuell gelöschte Nachrichten in das Archive "Unverteilt" gelegt
  • Fehlender Versandstatus (OK/NOK) auf dem Ausdruck von versendeten HTML Mails
  • Absturz des SL, wenn während der Datenbereinigung neue Virensignaturen eingespielt wurden und gleichzeitig ein Virencheck lief
  • Der Befehl GET Job_Seiten zählte bei "Notiz auf dem Deckblatt" eine Seite zu wenig
  • Möglicher Fehler bei der Index Suche in Archives, wenn für parallel liegende Archives die Index Suche ausgeschlossen wurde
  • Neuer Schalter in Systemkonfiguration "Wartezustand für externe Nachrichten"

2. Tobit InfoCenter

  • Probleme beim nachträglichen Einfügen von Attachments in eMails
  • Übernahme einer eMail Adresse in den Editor beim Anklicken einer eMail Adresse in der Adressvorschau
  • Probleme beim abarbeiten des FileScan Verzeichnisses im Zusammenhang mit Tracking Mails
  • Probleme beim Betrieb unter Windows Me
  • Änderungen an der Symbolleiste »Konfiguration« des Nachrichteneditors
  • Fokus im »An« Feld des »Nachricht versenden« Dialogs beim Drucken über den FaxWare Druckertreiber
  • Mögliche Probleme beim Anhängen von schreibgeschützten Dateien über den »Senden an« Dialog
  • Änderungen bei der Handhabung von Dateianhängen
  • Funktion und Schalter für "automatische Aktualisierung der Auftragsliste" entfernt
  • Im Nachrichten Editor ausgewählte Rundsendedateien, Deckblätter etc. wurden nicht durch eine entsprechenden Befehl dargestellt
  • Bei der Auswahl von Rundsendedateien wurden nur die Dateien aufgelistet, die eine Kommentarzeile enthielten
  • Probleme bei der Client/Server Verbindung bei Usern, die einen Umlaut im Namen tragen
  • Probleme bei der Client/Server Verbindung bei der Verwendung von mehreren Netzwerkkarten im Server
  • Schalter »Rückmeldung per Broadcast« entfernt
  • Layout-Verbesserungen der HTML Adressvorschau
  • Falsche Zeitangabe bei Auswahl von Versandzeitpunkt und Ablaufzeit im Nachrichten Editor
  • Zusätzliche Verteilmöglichkeit nach Wildcards mit Fragezeichen (?)
  • Mögliche Sonderzeichen in eMails beim Erstellen und/oder Versenden von Dokumenten
  • Mögliche Fehler beim Drag & Drop von Grafiken aus dem Interner Explorer in den Nachrichten Editor
  • Anzeigefehler in der HTML Vorschau bei ausgeschalteter Archive-Leiste
  • Fehlfunktionen im Kontextmenü der Archive-Leiste (nur bei den Shortcuts)
  • Mögliche Fehlfunktionen beim Öffnen von HTML eMails mit Sound
  • Beim Weiterleiten von HTML eMails wird die Quotinglinie nicht mehr eingefügt
  • Fehler in der Vorschau beim Wechsel auf leere eMails, die nur einen Zeilenumbruch enthielten
  • Beim Versand an mehrere Adressen inkl. Bcc Adressen wurden die BCC Adressen bei der Info über die Anzahl der Empfänger nicht mitgezählt
  • Die Verteilerliste wurde beim Versand nicht in die eMail eingefügt
  • Die Suche in der Kommentardatei funktionierte nicht
  • Fehler beim Kopieren eines Dateianhangs über das Kontextmenü aus dem Nachrichten Editor (nur Windows 95/98/Me)
  • Einschreiben und Tracking waren im Nachrichten Editor auch dann auswählbar, wenn es am Service Layer deaktiviert war
  • Mögliche Schutzverletzung bei der Auswahl vieler Adressen
  • Mögliche Schutzverletzung beim Schließen der Archive Root
  • Beim erneuten Versand einer eMail, die eine Adresskarte enthielt, konnte die Adressekarte erneut angehängt werden
  • Falsche Formatierung beim Einfügen von Textbausteinen in HTML eMails (bei Auswahl über den »Neu« Button)
  • Neues Modul (DVPREV.EXE) ersetzt die bisherige Vorschau von Textnachrichten
  • Möglicher Absturz der NTVDM.EXE unter Windows NT/2000 in Verbindung mit der HTML Vorschau
  • Neuer Schalter »Im Bearbeiten-Modus öffnen« ermöglicht das Bearbeiten empfangener eMails
  • Falsche Darstellung der Hauptsymbolleiste beim Ändern der Button Größe
  • Wiederaufnahme der Auswahlbox "Zustellung" im "Nachricht versenden" Dialog (Drucken über FaxWare Druckertreiber)
  • Neuer Schalter in "Archive-Eigenschaften" (Einträge bei der Verteilung verschieben) legt die Vorgabe für das manuelle Verteilen fest
  • Mögliche Schutzverletzung beim Drucken neu erstellter Plain Text Nachrichten
  • Im "Nur lesen" Modus konnten bei empfangenen Nachrichten angehängte VCF-Dateien nicht geöffnet werden
  • Neuer Schalter im Kontextmenü "Als eMail versenden" erstellt eine neue Nachricht und hängt alle ausgewählten Einträge als Dateianhang an (*.EML)
  • Fehlende Meldung beim Versuch in einem Archive ohne ausreichende Rechte zu speichern
  • Optimierung der Funktion "Suchen und Ersetzen" im Nachrichten Editor
  • Umlaute in Formularbezeichnungen wurden im Editor falsch angezeigt
  • Beim Reply wurden u.U. angegebene Cc Empfänger nicht berücksichtigt
  • Die Kommentardatei wurde beim Wechseln zwischen verschiedenen Einträgen falsch gespeichert
  • Mögliche Schutzverletzung im Nachrichten Editor wenn während der PGP Verschlüsselung der Editor im Vordergrund lag
  • eMails (i*.001) werden nun für den Nachrichten Editor registriert (bisher nur für den Image Editor)
  • Schalter "Allen antworten" wurde wieder aufgenommen
  • Im 32 Bit Editor können Anhänge wieder verschlüsselt werden ("Nachrichten Optionen/Sicherheit")
  • Neues Tool DvExtInf.exe wird in das Tobit InfoCenter Verzeichnis kopiert. Durch das Anklicken wird es im Kontextmenü der Eintragsliste eingetragen und ermöglicht das Auslesen der Kommentardatei (z.B. Header Informationen)
  • Fehlendes Tracking Reply beim Weiterleiten von Nachrichten
  • Falsche TO und FROM Informationen beim Öffnen von empfangenen Nachrichten im Bearbeiten Modus
  • Beim Ausdruck von versendeten Mails über den Editor fehlte die TO Information
  • Mögliche Schutzverletzung beim anhängen von Dateien und anschließendem Versand

3. Tobit Image Editor

  • Faxe, die nur als Shortcut im aktuellen Archive lagen, wurden nicht angezeigt

4. DvISE TLD für ISDN Karten

  • Neuer Schalter »SMS über ISDN« ermöglicht den wahlweisen SMS Versand über ISDN oder IHS
  • Unterstützung von bis zu 10-stelligen DDIs
  • DvISE TLD wurde nicht korrekt gestartet, wenn kein Script vorhanden war
  • Fehler beim Versand von HTML eMails über IHS
  • Provider Fax über IHS funktionierte nicht korrekt

5. DvISE TLD für SMS Modems

  • Diverse Detailverbesserungen und unterstützung von weiteren Handys
  • Der neue Befehl @@MSGFlash bewirkt die direke Anzeige einer SMS Nachricht

6. DvISE TLD für Brooktrout Karten

  • Mögliche Schutzverletzung beim Faxempfang

7. DvISE PostMan

  • Mögliche Abstürze bei hohem Mailempfang
  • Die gleichzeitige Einstellung »Mails sofort senden« bei DvISE PostMan und »PostMan Einstellungen verwenden« bei DvISE Grabbing Server wird ignoriert
  • Beim eMailversand zwischen zwei David Installationen wird eine gewählte Ablaufzeit mit übertragen

8. DvISE Grabbing Server

  • Manuelle Leerzeilen und Einrückungen in HTML eMails wurden nicht übertragen
  • Mehrfaches Eintragen von abgeholten Rundsendungmails war möglich
  • Das Datum von abgeholten Faxen, Voice und SMS Nachrichten entsprach der Abholzeit, nicht der Empfangszeit
  • Mögliches Fehlverhalten bei der Angabe von Wildcards in der Zieladresse

9. DvISE Mail Access Server

  • Leerzeilen zwischen Verteilerliste beim Abholen von eMails

10. DvISE Discussion Server

  • Unterstützung von HTML eMails

11. DvISE Replica

  • Fehler beim Abgleich gelöschter Einträge (nur bei der CD vom 18.09.2000)
