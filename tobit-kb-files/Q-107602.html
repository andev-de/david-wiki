Q-107.602
Meldung »Authentification failed« beim Update der IP-Adresse per SLS


Frage
Problem Im David.Administrator (Win32 und Linux) bzw. im Service Layer (Novell) wurde die Option zur Nutzung der Server Locator Services for David (SLS) aktiviert. Die aktuelle WAN IP-Adresse wird aber nicht an Tobit.Software übermittelt. Im Monitor des David PostMan wird eine Meldung mit folgendem Muster angezeigt:

IP transmission failure!
An authentication error occurred.
Possible reason: Password mismatch.
Please ensure that the password defined under
>>club tobit.com<< matches the password entered
in the PostMan ->Database -> AUTH Domain Names
configruration for your domain.

Welche Einstellungen müssen durchgeführt werden, damit die aktuelle WAN IP-Adresse korrekt übergeben wird?
Produkt Server Locator Services
Priorität normal
Effekt normal
Datum 12.01.2009

Antwort
Die Server Locator Services for David (SLS) ermöglichen die ständige Online-Präsenz sowie den direkten Zugriff auf Ihren David Server, auch wenn dieser nicht über eine Standleitung mit dem Internet verbunden ist und somit nicht über eine statische IP-Adresse verfügt (z.B. bei Nutzung einer DSL-Flatrate).

Da Computer im Internet ausschließlich über ihre IP-Adresse identifiziert werden, ist der direkte Zugriff auf die Informationen eines Rechners mit wechselnder IP-Adresse üblicherweise praktisch unmöglich. Bei Einsatz der Server Locator Services wird jede Änderung der IP-Adresse Ihres David Servers automatisch erkannt und dem DNS-Server bei Tobit.Software mitgeteilt. Dieser von Tobit.Software zentral betriebene DNS Server verwaltet die DNS Einträge sämtlicher Nutzer der Server Locator Services. Erfolgt nun eine Anfrage an Ihre Domäne, z.B. über einen Web-oder WAP-Browser, wird diesem vom DNS-Server stets die aktuelle IP-Adresse Ihres David Servers übermittelt. Auf diese Weise ist der David Server über das Internet ständig erreichbar und verhält sich praktisch so, als wäre die Internetanbindung über eine Standleitung mit statischer IP-Adresse realisiert.

Der Betrieb der Server Locator Services geschieht praktisch vollkommen im Hintergrund. Der David PostMan prüft die IP-Adresse des David Servers. Sobald er eine Änderung der IP Adresse feststellt, übermittelt er die neue Adresse automatisch an den DNS-Server von Tobit.Software. Dieser stellt somit die ständige Erreichbarkeit Ihres David Servers über das Internet sicher.

Damit die aktuelle WAN IP-Adresse fehlerfrei an den DNS-Server bei Tobit.Software übertragen werden kann, müssen folgende Einstellungen vorgenommen werden:

  • Die »Server Locator Services« müssen in den Einstellungen des David PostMan aktiviert sein.
  • Unter »PostMan -> Datenbanken -> Gültige Domain Namen« muss ein Eintrag für Ihre SLS-Domain ( Side-ID.tobit.net ) vorhanden sein.
  • Ihr aktueller Domain Name setzt sich aus Ihrer Site-ID (die ersten 10 Stellen Ihrer Systemlizenz inkl. Bindestrich) und der Domain »tobit.net« zusammen, also z.B.: 12345-67890.tobit.net
  • Unter »PostMan -> Datenbanken -> AUTH Domain Namen« muss Ihre SLS-Domain mit den Authentifizierungsdaten hinterlegt sein. Geben Sie den Benutzernamen und das Passwort für die Anmeldung an. Die Zugangsdaten hierzu werden aus Ihrer Systemlizenz bestimmt und auf Ihrem David Server in der PostMan Konfiguration unter dem Punkt AUTH Domain Namen eingetragen. Hierbei wird standardmäßig als Benutzername Ihre Site-ID verwendet. Diese entspricht den ersten 10 Stellen Ihrer Systemlizenz. Als Passwort wird Ihr Passwort für den Club tobit.com eingetragen, das normalerweise aus dem dritten Zeichenblock Ihrer Systemlizenz besteht. Damit der Abgleich der IP-Adresse mit dem DNS Server sicher ist, verwendet PostMan für den Verbindungsaufbau die SMTP Authentifizierung (AUTH=CRAM-MD5).

    Hinweise:
    Beachten Sie unbedingt, dass Sie jedes Mal, wenn Sie unter www.tobit.com das Passwort für den Zugang zum Club.tobit.com ändern, dieses neue Passwort auch im SLS Eintrag der PostMan Konfiguration unter AUTH Domain Namen eintragen müssen.

  • Beachten Sie, dass sich keine weiteren Einträge unterhalb von »PostMan -> Datenbanken -> ATHN Domain Name« und »TURN/ETRN/ATRN Domain Name« befinden dürfen.
Sollen auch die Möglichkeiten des Fernzugriffs auf die eigenen Nachrichten genutzt werden, muss eine entsprechende Lizenz (Remote CAL) vorhanden und in der Lizenzdatenbank am David Server (System - Lizenzen) eingetragen sein.
