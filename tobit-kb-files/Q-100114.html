Q-100.114
Konfiguration des Replication Services for DvISE


Frage
Problem Was ist bei der Konfiguration der Replication Services for DvISE zu beachten?
Produkt Replication Services for David
Priorität keine
Effekt harmlos
Datum 7.06.2005

Antwort
Im David InfoCenter können Sie prinzipiell jedes Archive unterhalb von »Archive« bestimmen, die auf beiden Servern miteinander abgeglichen werden sollen und auf welche Weise der Abgleich erfolgen soll.

Nähere Informationen bezüglich Einstellungen im David InfoCenter finden Sie unter dem KnowledgeBase Article Q-100.113.

Allgemeine Einstellungen im »DvISE Administrator -> DvISE Replica«

Menü »Archive -> Allgemein«
Hier können Sie generelle Einstellungen treffen:

  1. Vergeben Sie zunächst im Feld »Bezeichnung« einen Namen für den aktuellen Replica-Eintrag.
    Diese Bezeichnung muss nicht mit dem tatsächlichen Archive-Namen übereinstimmen, sondern kann frei gewählt werden. Sie dient hauptsächlich zur Unterscheidung der einzelnen Einträge in der Liste der zu replizierenden Archives und wird hier in der entsprechenden Spalte dargestellt.
  2. Legen Sie unter dem Punkt »Master« fest, in welcher Weise die Replizierung durchzuführen ist.
    Wählen Sie die Option »Zwei-Weg-Replizierung«, wenn Sie wünschen, dass Host und Client bei der Replizierung gleichrangig behandelt werden. Wünschen Sie stattdessen, dass die Datenbestände des Clients Vorrang vor dem Host haben sollen, wählen Sie die Option »Client«, um den Client als Master zu definieren. Soll der Host als Master dienen, wählen Sie die Option »Host«.
  3. Über die Option »Aktiv« können Sie die Replizierung für das aktuelle Archive aktivieren bzw. deaktivieren.
    Entfernen Sie die Markierung aus dem Kontrollkästchen, um die Replizierung vorübergehend zu deaktivieren. In diesem Fall müssen Sie nicht den kompletten Replica-Eintrag löschen. Um die Replizierung wieder zuzulassen, müssen Sie lediglich diese Option wieder aktivieren.
  4. Im Normalfall erfolgt die Replizierung in den Intervallen, die in der Replica System-konfiguration definiert sind. Wenn Sie stattdessen ein individuelles Intervall einstellen möchten, deaktivieren Sie die Option »Systemintervall benutzen« und tragen Sie die gewünschten Werte in die darunter stehenden Eingabefelder ein.
Menü »Archive -> Host«
Hier können Sie Einstellungen für den Host treffen:
  1. Tragen Sie im Feld »Host IP« die IP-Adresse oder den Domain Namen des Host Servers ein, mit dem die Archive-Replizierung durchgeführt werden soll.
  2. Im Feld »Host Archive« bestimmen Sie das Archive des Host Servers, mit dem der Abgleich erfolgen soll. Geben Sie hier den kompletten Pfad zu diesem Archive (inklusive Servername) in UNC-Schreibweise ein.
    Handelt es sich bei dem Host um einen NetWare Server, so ist unbedingt auch das Volume, auf dem sich die DvISE-Installation befindet, anzugeben.

    Beispiele:
    \\SERVERNAME\DAVID\ARCHIVE\8\15 (bei Windows NT/2000 Servern)\\SERVERNAME\VOLUME\DAVID\ARCHIVE\8\15 (bei NetWare Servern)

  3. Tragen Sie anschließend unter »Host Passwort« das System Passwort ein, das für den Host Server in der DvISE-Replica-Konfiguration unterhalb des Menüpunktes »System« vergeben wurde.
    Ist diese Angabe nicht vorhanden oder falsch, so kann kein Archive-Abgleich durchgeführt werden.
Menü »Archive -> Client«
Hier können Sie Einstellungen für den Client treffen:
  1. Geben Sie im Feld »Lokales Archive« das Archive des Clients an, das Sie mit dem Host abgleichen wollen.
    Tragen Sie den kompletten Pfad zu diesem Archive (inklusive Servername) in UNC-Schreibweise ein, oder bestimmen Sie das Archive über den Archive-Auswahl-Dialog durch Drücken des Buttons rechts neben dem Datenfeld bzw. durch Drücken der [EINFG]-Taste.

    Beispiel:
    \\SERVERNAME\DAVID\ARCHIVE\9

  2. In die Eingabefelder »Benutzername« und »Passwort« tragen Sie die Daten ein, die für die Parameter »Benutzername« und »Passwort« in den Archive-Eigenschaften der entsprechenden Archives festgelegt wurden.
    Beachten Sie, dass diese Parameter beim Host Archive und beim lokalen Archive auf dem Client identisch sein müssen.
