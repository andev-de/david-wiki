Q-102.181
Bedeutung der Verzeichnisse unterhalb der Verzeichnisstruktur »\DAVID\APPS\FAXWARE\RESOURCE\«


Frage
Problem Welche Bedeutung bzw. Aufgaben haben die Verzeichnisse unterhalb der Verzeichnisstruktur »\DAVID\APPS\FAXWARE\RESOURCE\« auf dem David Server?
Produkt David
Priorität keine
Effekt harmlos
Datum 5.11.2009

Antwort
Wenn Sie auf dem David Server im Windows Explorer die Verzeichnisstruktur »\DAVID\APPS\FAXWARE\RESOURCE\« öffnen, sehen Sie die hauptsächlich für den Faxversand benötigten Verzeichnisse. Nachfolgend erhalten Sie eine Übersicht zur Bedeutung der verschiedenen Verzeichnisse:

VerzeichnisErläuterung
APPEND Speicherot für Grafikdateien, die häufig als Anhänge zu einem Fax versendet werden. Die Dateien müssen im Format »PCX« oder »BMP« vorliegen und in der ASCII-Datei »DAVID.APD« mit ihrem Dateinamen eingetragen werden. Die Datei »DAVID.APD« kann mit einem Editor geöffnet und bearbeitet werden. Beachten Sie dazu die Hinweise, die nach dem Öffnen der Datei angezeigt werden.

Vorhandene Dateien:
»DAVID.APD«: ASCII-Datei mit Verweisen auf die Fax-Anhänge.
»STRESS.PCX«: Beispiel-Grafikdatei als Fax-Anhang.

COVERSpeicherort für Fax-Deckblätter und darin integrierte Grafiken. Die Deckblätter müssen als ASCII-Datei mit der Dateiendung »CVP« vorliegen.

»BEISPIEL.CVP«: Deckblatt-Datei
»BEISPIEL.PCX«: Grafikdatei zur Integration in einem Deckblatt

DATASpeicherort für Steuerungsdateien zum Versand von Rundsendungen. Werden im Nachrichten-Editor des David Client mehrere eMail-Adressen oder Rufnummern im »An«-, »Bcc«- oder »Cc«-Feld eingetragen und die Nachricht als Rundsendung verschickt, so wird temporär in diesem Verzeichnis eine Rundsendedatei mit der Dateiendung »RND« erzeugt, die sämtliche Adressaten beinhaltet.

Auch Rundsendedateien mit einem festen Adressensatz können hier dauerhaft abgelegt und im David Nachrichten-Editor abgerufen werden. Nähere Informationen zum Erstellen von Rundsendedateien erhalten Sie im KnowledgeBase Artikel Q-101.245.

Vorhandene Dateien:
»BCAST.RND«: Beispiel für eine Rundsendedatei

FONTSSpeicherort für Standardschriftarten mit der Dateiendung »FNT«. Diese können bei der Faxerstellung mit dem David Client verwendet werden.

Vorhandene Dateien:
»*.FNT«: Spezielle Schriftarten
»DVFONT.EXE«: Anwendung zum Bearbeiten von Schriftarten

FORM Speicherort für Grafikdateien im »PCX«- oder »BMP«-Format, die als Formular in einem Fax-Dokument hinterlegt werden können.
Formulare können Sie mit dem David Befehl »@@FORMULAR« auswählen, z.B. »@@FORMULAR FORM.PCX«
GRAPHStandardspeicherort für 1-bit-Grafikdateien im »BMP« oder »PCX« Format, die im Nachrichten-Editor des David Client mit dem David Befehl »@@EINFÜGEN« an definierter Position in Fax-Nachrichten eingefügt werden können, z.B. »@@EINFÜGEN LOGO.PCX«.

Vorhandene Dateien:
»GRID_10.PCX«, »GRID_12.PCX«, »LOGO.PCX«, »STRESS.PCX«: Beispielgrafiken zur Einbindung in ein Fax

INCLUDE Speicherort für »Include«-Dateien mit der Dateiendung »INC«, die in der Benutzerkonfiguration im David.Administrator auf der Registerkarte »Versand« ausgewählt werden können. Darüber hinaus ist mit dem David Befehl »@@INCLUDE« das manuelle Einbinden von Include-Dateien in Sendeaufträge möglich.

Include-Dateien sind reine Textdateien im ASCII-Format, deren Inhalt direkt in den Nachrichtentext eingefügt wird. Die Verwendung von David Befehlen innerhalb von Include-Dateien ist möglich.

SIGNSpeicherort für eingescannte Unterschriften der David Benutzer, die im »PCX«-Format vorliegen müssen.

Jedem David Benutzer kann in der Benutzerkonfiguration im David.Administrator auf der Registerkarte »Versand« eine eigene Unterschrift-Grafik zugewiesen werden. Dadurch kann der Benutzer z.B. im Nachrichten-Editor des David Client mit dem David Befehl »@@UNTERSCHRIFT« seine Unterschrift in Faxe einfügen.

Vorhandene Datei:
»TT_SIGN.PCX«: Beispieldatei einer Unterschrift

Im KnowledgeBase Artikel Q-102.288 finden Sie eine Übersicht aller Artikel zu Dateien, Dateitypen sowie Verzeichnissen des David Ordnersystems.
