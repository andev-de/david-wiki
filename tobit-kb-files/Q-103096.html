Q-103.096
UpDate auf David XL mit den SLS Zugangsdaten und Domänen


Frage
Problem Wie werden die Zugangsdaten von SLS bei einem Update auf David XL auf den neuen Club Account übertragen und was ist zu beachten?
Produkt Server Locator Services
Priorität dringend
Effekt kritisch
Datum 15.11.2005

Antwort
Nach der Installation von David XL steht ihnen ein Umzugsservice zur Verfügung der die Service Locator Services mit dem neuen Club Account verbindet.

Während des Starts des Tobit InfoCenters wird die Echtheitsbestätigung abgefragt. Auf der Registerkarte "Echtheitsbestätigung für Site-ID 12345-67890" haben Sie die Möglichkeit über "Anfordern" den Umzugsservice zu aktivieren. Bei der Ümbernahme der Unternehmens- und Lizenzdaten werden zusätzlich alle SLS-Domänen auf den neuen Kundenzugang umgeschrieben. Weiter werden die SLS-Zugangsdaten, die an ihrem David-Server gespeichert sind auf die neue Site-ID umgeschrieben. Nach dieser Prozedur ist ihr Server direkt, ohne Neustart über die neuen Zugangsdaten erreichbar.

Zur Kontrolle sollten Sie allerdings die Zugangsdaten erneut überprüfen:

Konfiguration überprüfen
Überprüfen Sie bitte die Konfiguration des DvISE PostMan.

  • Nach der Installation werden sind folgende Einträge zu prüfen:In der DvISE PostMan-Konfiguration wird unterhalb von »System -> Erweitert« ist der Punkt "Server Locator Services" aktiviert.
  • Innerhalb von »Datenbanken -> Gültige Domain Namen« muss sich ein Eintrag für Ihre SLS-Domain ( SiteID.tobit.net ) befinden.
  • Innerhalb von »Datenbanken -> AUTH Domain Namen« steht Ihre SLS-Domain mit den Authentifizierungsdaten. Diese können, falls nicht eingetragen, auch manuell eingegeben werden:
    1. Der Name der Domain ist SiteID.tobit.net
    2. Der Benutzername ist Ihre SiteID
    3. Das Passwort ist das club.tobit.com Passwort in Großbuchstaben
    4. Als Verschlüsselung verwenden Sie die verschlüsselte Anmeldung (CRAM-MD)
  • Bitte beachten Sie, dass sich keine weiteren Einträge mit *.tobit.net im DvISE PostMan unterhalb von »Datenbanken -> AUTH Domain Name« und »TURN/ETRN/ATRN Domain Name« befinden dürfen.
  • Die aktuelle IP wird mit dem DvISE PostMan über den Port 25 per ATRN Protokoll übertragen. Falls der Internet-Zugang über den DvISE-Server realisiert wird, sind dies alle Punkte, die zu beachten sind.
SLS mit DSL-Router
Wenn Sie einen Router für die Internet-Einwahl verwenden, haben Sie bei David XL die Möglichkeit automatisch eine neue IP auf dem Hardware Router auszulesen. In der PostMan-Konfiguration wird unterhalb von »System -> Erweitert« ist der Punkt "Benutzung eines Routers" zu aktivieren. Über "Routerkonfiguration -> Router-Model" wählen Sie aus der Liste das Router-Modell aus, das in Ihrem Netzwerk eingesetzt wird. Sollte Ihr Router nicht in der Liste aufgeführt sein, wählen Sie den Eintrag Userdefined Router füllen Sie die weiteren Eingabefelder aus und klicken Sie anschließend auf Konfigurieren..., um manuelle Anpassungen vorzunehmen. Halten Sie in diesem Fall die Beschreibung Ihres Routers bereit, um die erforderlichen Informationen nachschlagen zu können.
  • Bei "IP-Adresse" tragen Sie die interne IP-Adresse ein, unter der ihr Router im lokalen Netzwerk erreichbar ist.
  • Als "TCP Port" tragen Sie den TCP Port ein, den Ihr Router verwendet.
  • Unterhalb von Login-Informationen hinterlegen Sie den Benutzername und das Passwort, unter dessen Verwendung auf den Router zugegriffen werden kann, ein.
  • Klicken Sie auf die Schaltfläche "Testen", um zu prüfen, ob unter Verwendung der eingetragenen Daten erfolgreich auf den Router zugegriffen werden kann. Nach Abschluss des Tests wird eine entsprechende Meldung angezeigt.
