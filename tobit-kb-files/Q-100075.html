Q-100.075
Datenübernahme auf einen neuen David Server


Frage
Problem Datenübernahme am Beispiel: Novell NetWare auf Windows NT
Produkt Utilities
Priorität normal
Effekt normal
Datum 3.07.2014

Antwort
Option 1: Server Migration Services for DvISE
Tobit bietet mit den "Server Migration Services for DvISE" ein leistungsfähiges Werkzeug, mit dem DvISE-Daten bei einem Serverwechsel per Mausklick migriert werden können. Dabei werden neben allen im Tobit Archive System abgelegten Dateien auch wichtige Systemdaten übernommen. Die Server Migration Services for DvISE können für Migrationen von Netware auf Win32, von Netware auf Linux, von Win32 auf Linux sowie von Win32 auf Win32 eingesetzt werden.

Die Server Migration Services for DvISE können vom Fachhändler über das Tobit Partner Network bezogen werden. Voraussetzung für den Einsatz ist DvISE ab Version 6.6. Zudem ist zu beachten, dass für den neu einzurichtenden Server der Erwerb eines neuen DvISE Vollproduktes erforderlich ist.

Option 2: Manuelle Datenübernahme
Bitte beachten Sie, dass eine Datenübernahme eigentlich nicht vorgesehen ist und es sich bei folgendem Text um einen Workaround handelt!

Führen Sie zunächst eine entsprechende Datensicherung durch!

Sofern Sie unter Windows NT/2000 arbeiten (NT -> NT) und der Servername »gleich« bleibt können Sie im Prinzip ganz einfach nach der vollständigen Installation - wobei alle Ports in der gleichen Reihenfolge installiert werden müsssen - eineKopie des David Verzeichnisses über die Neuinstallation kopieren.

Unter Novell NetWare gilt im Prinzip Ähnliches, hier muß jedoch sichergestellt werden, daß Volume und Servername gleich bleiben.

Ansonsten gibt es lediglich die Möglichkeit, Daten der Archive zu übernehmen, nicht aber Konfigurationsdaten.
Diese Datenübernahme kann auch nicht durch einfaches Kopieren stattfindensondern MUSS mit Hilfe eines Archivpackers oder mit ArcUtil vollzogen werden.

Sicherlich ist die Datenübernahme mit ArcUtil (per suchen und ersetzen) die schnellere Lösung, jedoch sollten Sie von dieser Vorhegensweise Abstand nehmen, da bei unsachgemäßem Gebrauch des Tools ArcUtil es zu irreparablen Schäden in der Archivestruktur kommen kann.

Aus diesem Grunde halten Sie sich an die unten aufgeführte Vorgehensweise, die eine Datenübernahme mit Hilfe des Tools "MKINST" beschreibt.Dieses Tool finden Sie auch auf Ihrem David-Server im Verzeichnis »DAVID\UTIL\WINDOWS\MKARC«.

Diese Anleitung ist für eine Datenübernahme am Beispiel Novell NetWare -> Windows NT geschrieben.Sofern Sie eine Datenübernahme Windows NT -> Windows NT oder Novell NetWare -> Novell NetWare planen müssen natürlich eventuelle Pfadangaben ersetzt werden.

Um diese nötigen Schritte durchzuführen gehen Sie bitte wie folgt vor:

  1. Zunächst sorgen Sie bitte dafür, dass Verzeichniseinträge, welche nicht mehr in der neuen Installation benötigt werden aus den Archiven entfernt werden. Dazu stellen Sie bitte im David Client am Arbeitsplatz des Administrators die Archivbereinigung alles Archive auf z.B. 30 Tage (oder gewünschter Zeitraum).

    Unter der Netware wechseln Sie nun in den Servicelayer und drücken die ALT-1 TASTE. Nun können Sie mit ALT-3 verfolgen, wie die Bereinigung durchgeführt wird.

    Unter Windows NT öffnen Sie bitte die Datei DAVID.INI im DAVID/CODE Verzeichnis. Dort ändern Sie bitte den Eintrag ";deletetime=1" auf folgenden Eintrag:
    deletetime= x (x steht für die nächste volle Stunde!)
    Nun starten Sie unter Windows NT dem Servicelayer neu. Warten Sie nun ab, bis die nächste Volle Stunde erreicht ist.

    David führt nun eine Bereinigung durch.
    Das Ende der Bereinigung können Sie im Archiv :
    SYSTEM/DAVID/FEHLERMELDUNGEN
    feststellen. Dort gibt es dann einen Eintrag: "Finish purging Archive". Erst wenn dieser Eintrag dort erscheint können Sie mit Schritt 2 fortsetzen!

  2. Starten Sie nun auf dem alten Server aus den Verzeichnis LW:\DAVID\UTIL\WINDOWS\MKARC die MKINST.EXE :
    • Aus Integritäts- und Sicherheitsgründen sollte ein mit MKINST32.EXE gepacktes Archiv nicht mehr als 250 MB umfassen. Sollten Sie ein größeres Volumen an Daten verfügen, so teilen Sie Inhalte Ihrer Archive entsprechend auf, z.B. das Adressarchiv in ein eigenes gepacktes File.
    • Hier haben Sie nun im Fenster OPTIONEN zunächst die Möglichkeit, die Bezeichnung des alten Archivs anzugeben.
      Normalerweise wäre dies : ARCHIVE
    • Sofern Sie jedoch nur bestimmte Archive packen möchten geben Sie bitte explizit den Archivnamen an. (Bsp.: FAXBOX)
    • Jetzt gibt es im nächsten Feld (Bezeichnet mit Quellverzeichnis) die Möglichkeit genau anzugeben, wo sich das gewünschte Archiv befindet. Mit Hilfe des links liegenden Buttons (...) können Sie nun den Pfad wählen. (Bsp.: "\\»SERVERNAME«\SYS:\DAVID\ARCHIVE")
    • Hier wechseln Sie bitte herunter in das Archiv, welches Sie übernehmen möchten. (Möglichkeit: "\\»SERVERNAME«\SYS:DAVID\ARCHIVE\FAXBOX\")
    • Jetzt geben Sie bitte im letzten Feld des Tools (Bezeichnet mit Zieldatei) einen Pfad und Dateinamen an, in dass das gepackte Archiv von MKINST gelegt werden soll.
      Anbieten würde sich ein Pfad außerhalb der DAVID Verzeichnis Struktur. (Bsp.: "C:\DATEN\OLDARCH.exe")
    • Nachdem Sie nun noch das Feld "Absender beibehalten" angeklickt haben klicken Sie bitte den PACKEN Button.
    Nun werden die Daten gepackt. Dieser Vorgang kann durchaus einige Zeit dauern. Da der Arbeitsplatz relativ stark von diesem Prozess belastet wird, arbeiten Sie bitte während des Packvorgangs nicht mit einem anderen Programm.

  3. Nachdem die Archive nun gepackt außerhalb der David Struktur liegen, führen Sie bitte eine Datensicherung des David Verzeichnisses durch. Sofern nämlich bei dem Packvorgang ein Fehler aufgetreten ist, wäre dies die einzige Möglichkeit die Daten zurück zu erhalten!

  4. Jetzt kann eine Deinstallation von David auf dem alten Server durchgeführt werden.

    Dazu starten Sie unter Windows NT die Uninstallroutine, unter Novell Netware löschen Sie das David Verzeichnis einfach. (Benennen Sie bitte auch die Dateien DVNDS.NLM und DAVID.*, sowie TLDxxx.nlm um)
    Jetzt löschen Sie auf den Arbeitsplätzen die Datei "TOBIT.INI".

  5. Jetzt können Sie wie gewohnt eine Neuinstallation auf dem neuen Server durchführen.
    Führen Sie bitte noch kein Client Setup durch!

  6. Jetzt starten Sie die in Schritt 2 erstellte Archivedatei mit der Endung ".exe". Hier brauchen Sie nur noch den Zielpfad angeben. Dazu können Sie wieder mit Hilfe des (...) Buttons in das Verzeichnis
    "\\»SERVERNAME«\DAVID\ARCHIVE\"
    wandern.
    Benutzen Sie jedoch bitte keine gemappten Laufwerke. Geben Sie anschließend per Hand hinter dem letzten "\" noch OLD ein.

    Was natürlich nun noch nötig ist, ist die Nennung des Servernamens.
    Bsp.: "\\»SERVERNAME«"
    Unter Novell hieße dieser : "\\»SERVERNAME«\VOL1:"
    Nun betätigen Sie den "INSTALLIEREN" Button. Jetzt werden die Archive neu eingespielt.

  7. Installieren Sie nun die Clients an den Arbeitsplätzen. Dazu loggen Sie sich bitte an allen Arbeitsplätzen als ADMINISTRATOR oder gleich gestellter Benutzer ein.

    Hier sehen Sie jetzt unterhalb der blauen Archivebox einen Archivordner "ARCHIVE" oder "FAXBOX" oder je nachdem, was Sie zuvor in Schritt 2 als Archivebezeichnung genannt haben.

  8. Nun brauchen Sie nur noch die gewünschten Archivinhalte via DRAG&DROP in das gewünschte Archiv verschieben.
    Wichtig ist nur, dass das Verschieben innerhalb des Clients geschieht.
Nun sind die alten Daten auf Ihren neuen Server übernommen.

Beachten Sie auch den Text zur Übernahme von Verteilregeln im
KnowledgeBase Article Q-102.382.

Hinweis:
Achten Sie grundlegend darauf, dass Sie nicht komplette Archive-Bäume packen, sondern z.B. nur kleinere Verzeichnisstrukturen wie einzelne User-Verzeichnisse. Diese Verzeichnisstrukturen sollte eine Größe von 1 GB nicht überschreiten.
