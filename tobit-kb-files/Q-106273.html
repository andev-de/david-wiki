Q-106.273
Einrichtung des David.InfoCenter Mobile


Frage
Problem Wie wird das David.InfoCenter Mobile installiert und eingerichtet?
Produkt David.InfoCenter Mobile
Priorität normal
Effekt normal
Datum 18.02.2008

Antwort
Das David.InfoCenter Mobile ist eine spezielle Version des David.InfoCenter PC,die auch auf mobilen Computern ohne dauerhafte Netzwerkverbindunggenutzt werden kann. Damit können z.B. Außendienstmitarbeiter, die mit ihremNotebook unterwegs sind, auch ohne Verbindung zum David Server und somitunabhängig von Ort und Zeit mit dem David InfoCenter arbeiten. Zu versendendeNachrichten werden im Offline-Modus – also wenn keine Verbindungzum David Server besteht – als Aufträge im Versandbuch abgelegt und erst dannverschickt, wenn wieder eine Verbindung besteht.

Auch die Inhalte der persönlichen Archives können offline wie gewohnt bearbeitet werden. Nach der Wiederherstellung der Server-Verbindung werden die Inhalte mit den zugehörigen Archives auf dem David Server abgeglichen und so auf den gleichen Stand gebracht.

Die Replizierung der Archive-Inhalte vom David Server auf den mobilen Computerund umgekehrt geschieht durch die David Replication Services. Diesewerden bereitgestellt durch das Software-Modul »David Replica«. Für denBetrieb des David.InfoCenter Mobile sind im Allgemeinen keine manuellenEinstellungen in diesem Modul erforderlich. Lediglich ein Passwort zur Authentifikation für den Archive-Abgleich muss definiert werden. Dieses Passwort wird vom Installationsprogramm automatisch abgefragt.

Voraussetzungen
Die zu verwendenden mobilen Computer müssen unter Windows 2000, WindowsXP Professional oder Windows Server 2003 betrieben werden. Auf ihnen darf sichkeine eigene David Installation befinden. Eventuell vorhandene David Modulemüssen vor der Installation des David.InfoCenter Mobile von diesen Computernentfernt werden.

Stellen Sie sicher, dass bei der Installation des David.InfoCenter Mobile unter Windows Vista die »Benutzerkontensteuerung« aktiviert ist.

Der komplette Funktionsumfang steht nur dann zur Verfügung, wenn das David.InfoCenter Mobile auf einer NTFS-formatierten Festplattenpartition installiert wird. Stellen Sie daher unbedingt sicher, dass alle Computer, auf denen das David.InfoCenter Mobile genutzt werden soll, über eine NTFS-Partition verfügen.

Einstellungen während der David Installation
Bei der David Installation auf den Server können Sie entscheiden, ob die Nutzung des David.InfoCenter Mobile für die David Benutzer generell möglich sein soll oder nicht. Wenn Sie die Nutzung des David.InfoCenter Mobile zunächst unterbunden haben, jedoch zu einem späteren Zeitpunkt dennoch ermöglichen wollen, führen Sie einfach das David Installationsprogramm erneut aus und aktivieren Sie die entsprechende Option.

Falls bisher noch kein System-Passwort für die David Replication Services vergeben wurde, werden Sie während der David-InfoCenter-Mobile-Installation zur Eingabe eines Passworts aufgefordert. Das nachfolgende Fenster bietet die Möglichkeit, dieses Passwort für die anschließende Installation des David.InfoCenter Mobile auf den Arbeitsstationen zu speichern. In diesem Fall wird das Passwort dann bei der Einrichtung der Clients automatisch in die Konfiguration des David.InfoCenter (»Optionen > Einstellungen > Mobile«) übernommen und muss später nicht manuell eingetragen werden.

Beachten Sie jedoch, dass bei einer Speicherung des Passworts jeder Benutzer, der auf seiner Arbeitsstation über Administratorrechte verfügt, das David.InfoCenter Mobile an dieser Arbeitsstation installieren und nutzen kann, da ja ein Passwort bei der Installation nicht mehr abgefragt wird. Legen Sie anschließend fest, welche globalen Archives in die Replizierung einbezogen werden sollen, und geben Sie schließlich den Domain-Namen bzw. die IP-Adresse des David Servers an.

Arbeitsweise
Die Dienste des David.InfoCenter Mobile werden unter Verwendung der DavidReplica Services bereitgestellt. Lesen Sie daher auch die vorhergehendenAbschnitte dieses Kapitels. Damit ein Betrieb des David.InfoCenter Mobile auch im Offline-Modus – also ohne direkte Verbindung zum David Server – möglich ist,werden die David Systemdienste »Service Layer« und »Replica« auch auf denArbeitstationen installiert und gestartet. Aus diesem Grund ist die Nutzung desDavid.InfoCenter Mobile ausschließlich auf Arbeitsstationen möglich, die unterWindows 2000, Windows XP oder Windows Vista betrieben werden.

Das David.InfoCenter Mobile kann ausschließlich mit genau einem David Server als Host betrieben werden. Eine abwechselnde Replizierung mit unterschiedlichen David Servern ist nicht möglich.

Auf den mobilen Computern wird bei der Installation des David.InfoCenter Mobileunterhalb des Programmordners des David.InfoCenters eine lokale DAVID Verzeichnisstrukturangelegt. Diese enthält Kopien aller Objekte des David Servers, die für denOffline-Betrieb benötigt werden. Der David.Administrator ist auf den mobilenArbeitsstationen jedoch nicht verfügbar. Für eine Verbindung zum David Server und die Archive-Replizierung ist keine lokale Anmeldung am Server notwendig, es genügt eine IP-Verbindung. Die Legitimation findet über ein sicheres Authentifizierungsprotokoll statt, bei dem keine Anmeldedaten übertragen werden.

Replizierung der persönlichen Archives
Bei der Archive-Replizierung werden generell die persönlichen Archives des aufdem mobilen Computer angemeldeten David Benutzers mit den zugehörigenDatenbeständen auf dem David Server abgeglichen. Dieser Abgleich erfolgt als sogenannte »Zwei-Weg-Replizierung«, was bedeutet, dass der David Server und dieArbeitsstation gleichberechtigt sind. Einträge oder Unterordner, die in den persönlichen Archives auf dem mobilen Computer neu erstellt wurden, werden auch in die entsprechenden Archives des David Servers übernommen und umgekehrt. Ebenso werden Einträge, Archives oder Dokumente, die auf einem der Computer gelöscht oder geändert wurden, auch auf dem anderen entfernt oder geändert.

Falls bestimmte Dokumente zwischen zwei Abgleichintervallen unabhängigvoneinander sowohl auf dem David Server als auch auf der Arbeitsstation bearbeitet wurden, so sind nach dem Abgleich auf jedem der Computer beideVersionen des Dokumentes vorhanden. Das Dokument des jeweils anderen Rechnerswird durch den Zusatz »REP:« in der Betreffzeile gekennzeichnet.

Replizierung globaler Archives
Außer den persönlichen Benutzer-Archives können auch bestimmte globale Archives in die Replizierung einbezogen werden. Hierbei sind ausschließlich folgende Archives wählbar:
  • Archives/Ressource/Feiertage
  • Archives/Ressource/Formulare
  • Archives/Ressource/Textbausteine
  • Archives/Ressource/Vorlagen
  • Archives/Adressen (globales Adress-Archive)
Welche dieser Archives in die Replizierung einbezogen werden sollen, wird direktwährend der Installation der Server-Komponenten festgelegt. Diese Auswahl giltdaraufhin für den Abgleich mit allen mobilen Clients. Der Abgleich dieser Archiveserfolgt grundsätzlich als Master-Slave-Replizierung, wobei in jedem Fall der DavidServer der Master ist. Die Arbeitsstationen fungieren dann zwangsläufig als Slaves.

Bei dieser Art der Replizierung verläuft der Abgleich ausschließlich in eine Richtung, nämlich vom Master auf den Slave. Nach Abschluss des Abgleichs verfügt der Slave also über ein Abbild der Datenstruktur des Masters. Befinden sich auf dem Slave Änderungen oder neu erstellte Archives oder Dokumente, so werden diese nicht zum Master übertragen, sondern statt dessen auf dem Slave gelöscht.

Einrichten der Arbeitsstationen
Um die Installation des David.InfoCenter Mobile auf den mobilen Arbeitsstationen vornehmen zu können, müssen diese korrekt im Netzwerk angemeldet sein und Zugriff auf den David Server besitzen. Weiterhin muss der Benutzer, der die Installation durchführt, über Administratorrechte auf der jeweiligen Arbeitsstation verfügen. Standardmäßig können bis zu 25 mobile Arbeitsstationen gleichzeitig ihre Daten mit demselben Host Server replizieren. Dieser Maximalwert kann bei Bedarf in der Systemkonfiguration der David Replication Services am David Server verändert werden (»Max. Verbindungen«).

Die Installation des David.InfoCenter Mobile sollte unbedingt auf eine NTFS-formatierten Festplattenpartitionerfolgen.

Wechseln Sie von der Arbeitsstation aus über das Netzwerk in das \DAVID\CLIENTSVerzeichnis auf dem David Server und führen Sie dort die Datei SETUP.EXE aus, um die Installation zu starten. Sie können die Installation des David.InfoCenter Mobile auch direkt von der David-Installations-DVD vornehmen. Legen Sie dazu die DVD in das DVD-Laufwerk der gewünschten Arbeitsstation ein. Sollte der Start des Installationsprogamms nicht automatisch erfolgen, rufen Sie die Datei SETUP.EXE aus dem Wurzelverzeichnis der DVD auf.

Wurde das System-Passwort für die Replizierung während der Server-Installation nicht gespeichert, erfolgt nun eine Aufforderung zur Eingabe dieses Passwortes. Ohne die Eingabe des korrekten System-Passworts wird die Installation zwar zu Ende geführt, eine Replizierung mit dem Host Server führt jedoch zu einer Fehlermeldung (Replikationsmonitor). Erst wenn im David.InfoCenter unter »Optionen -> Einstellungen« auf dem Register »Mobile« das korrekte System-Passwort eingetragen wurde, erfolgt eine fehlerfreie Replizierung.

Zum Abschluss wird abgefragt, ob beim Hoch- bzw. Herunterfahren des Betriebssystemsan die Durchführung eine Replizierung erinnert werden soll. Zu diesemZweck wird auf der mobilen Arbeitsstation ein Dienst installiert und gestartet.Setzen Sie den Schalter »Erinnerung einschalten«, wenn Sie an die Replizierungerinnert werden möchten. Sofern für den Remote Access auf die Archives einesBenutzers noch keine Zugriffsparameter vergeben wurden, erfolgt beim erstenStart des David InfoCenters die Abfrage dieser Parameter, die für die Durchführung der Replizierung benötigt werden.

Betrieb des David.InfoCenter Mobile
Das David.InfoCenter Mobile verfügt im Menü »Netzwerk« über die Option »Offline«, mit welcher der Offline-Modus aktiviert bzw. deaktiviert werden kann. Des Weiteren finden Sie dort als Server-Eintrag außer dem Namen des David Servers auch den der lokalen Arbeitsstation. Über diese beiden Einträge kann ohne Replizierung zwischen David Server und Arbeitsstation umgeschaltet werden. Der Wechsel zur Arbeitsstation entspricht dann dem Wechsel in den Offline-Modus.

Unter dem Hauptmenüpunkt »Optionen - > Einstellungen -> Mobile« können Sie Einstellungen für den Abgleich mit dem David Server konfigurieren. Hier stehen die folgenden Konfigurationspunkte zur Verfügung:

  • Host IP/Domäne
    Tragen Sie in dieses Feld die IP-Nummer des Host Servers (z.B. 192.168.168.168) oder Ihren Domain-Namen ein (z.B. yourcompany.com).

  • System Passwort
    Das hier eingetragene System-Passwort gilt für die Replizierung Ihrer Daten mit dem Host-Rechner. Das Passwort darf nur verändert werden, wenn auch der gleichnamige Eintrag auf dem Host Server (Menüpunkt »David Replica«) angepasst wird. Im Zweifelsfall belassen Sie den Eintrag unverändert, um Fehler bei der Replizierung zu vermeiden.

  • Beim Starten/Beenden des PCs replizieren
    Aktivieren Sie den Schalter, wenn beim Herauf- bzw. Herunterfahren des Betriebssystems eine Abfrage erfolgen soll, die Arbeitsstation mit dem Host Server zu replizieren.

  • Remote Access Parameter verwenden
    Setzen Sie diesen Schalter, wenn Sie wünschen, dass für die Archive-Replizierung dieselben Authentifizierungsparameter verwendet werden, wie für den Remote Access. Wenn Sie die Remote Access Parameter nicht verwenden möchten, tragen Sie Benutzername und Passwort in die zugehörigen Felder ein.

  • Intervall
    Wählen Sie aus der Liste ein Intervall für den automatischen Abgleich (Replizierung) mit dem David Server. 15 Minuten z.B. bedeutet, dass Ihr David.InfoCenter Mobile in 15-Minuten-Abständen Kontakt zum David Server aufnimmt und die eingestellten Archives repliziert (abgleicht). Ist nach Ablauf des Intervalls keine Verbindung zum David Server möglich, wird eine entsprechende Meldung im Archive »System -> David -> Ereignisse« erzeugt.

  • Software Updates automatisch beziehen
    Setzen Sie diesen Schalter, um Ihren Mobile Client (das David.InfoCenter Mobile) zusammen mit dem David.InfoCenter automatisch zu aktualisieren, sobald eine Verbindung zum Host-Rechner besteht und ein Update zur Verfügung steht.

Offline-Modus aktivieren bzw. deaktivieren
Solange sich die mobile Arbeitsstation im lokalen Netzwerk befindet, also direkte Verbindung zum David Server besitzt, greift das David.InfoCenter direkt auf diesen Server zu. Das bedeutet, dass sämtliche Aktionen im David.InfoCenter sofort über den David Server abgewickelt werden. Änderungen an Inhalten im Tobit Archive System werden ebenfalls sofort auf dem Server vorgenommen.

Wenn der mobile Computer vom Netzwerk getrennt wird, ist der direkte Zugriff auf den Server natürlich nicht mehr möglich. Daher muss vor dem Trennen der Netzwerkverbindung der Offline-Modus aktiviert werden.

Klicken Sie dazu auf »Netzwerk -> Offline« in der Menüleiste des David.InfoCenters. Dieses wird nun intern auf die lokale Kopie des Tobit Archive Systems umgeschaltet. Im Normalfall sollte anschließend eine Archive-Replizierung durchgeführt werden, um Ihre Arbeitsstation und den David Server auf den gleichen Stand zu bringen. Es erscheint ein Dialogfenster, das zur Durchführung der Replizierung auffordert.

Im Offline-Modus wird ausschließlich mit der lokalen Kopie des Tobit Archive Systems gearbeitet. Die während der Offline-Zeit durchgeführten Änderungen werden auf den David Server übertragen, sobald die Arbeitsstation wieder mit dem Netzwerk verbunden und das David.InfoCenter wieder in den Online-Modus geschaltet wird. Das hat zur Folge, dass z.B. ein Versandauftrag, der im Offline-Modus erstellt wurde, erst bei der nächsten Replizierung an den Host Server übertragen und von diesem anschießend versendet wird. Arbeitet man mit der Arbeitsstation zwischenzeitlich wieder im Offline-Modus, so wird dieser Versandauftrag mit dem entsprechenden Status-Eintrag wiederum erst mit der nächsten Replizierung im Ausgangsbuch der Arbeitsstation sichtbar.

Klicken Sie dazu wiederum den Menüpunkt »Netzwerk -> Offline« an. Das Häkchen vor »Offline« verschwindet und das David.InfoCenter wird nun wieder für den direkten Zugriff auf den David Server umgeschaltet. Auch hier erscheint das Dialogfenster, das zur Durchführung der Archive-Replizierung auffordert, um wiederum die Datenbestände beider Computer miteinander abzugleichen.

Deinstallation des David.InfoCenter Mobile
Klicken Sie am Arbeitsplatz-Rechner auf das Menü »Start« und wählen Sie den Punkt »Einstellungen -> Systemsteuerung -> Software«. Wählen Sie aus der Liste der installierten Programme den Eintrag »David.InfoCenter« aus und klicken Sie auf »Entfernen«. Nach Bestätigen einer Sicherheitsabfrage werden sämtliche Dateien des David.InfoCenters sowie alle zugehörigen Registry-Einträge entfernt.

Hinweis
Die Nutzung des David.InfoCenter Mobile auf einem Computer ist nur von einem Benutzer bzw. mit einem Benutzerkonto möglich.
