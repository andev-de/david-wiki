Q-101.193
Client-Server Verbindung unter Windows NT/2000


Frage
Problem In einem homogenen Netzwerk, d.h. es sind nur Windows NT/2000 Server sowie Windows Workstations vorhanden, können sich ein oder mehrere Benutzer mit dem Tobit InfoCenter nicht am DvISE Server anmelden.
Produkt David.InfoCenter
Priorität normal
Effekt normal
Datum 26.06.2002

Antwort
Wenn die Clients sich nicht anmelden können, sollten Sie folgende Punkte beachten:
  1. Führen Sie zuächst die Deinstallationsroutine über »Start -> Programme -> Tobit InfoCenter -> Uninstall ...« durch.
  2. Falls noch Startmenü Einträge nach der Deinstallation vorhanden sein sollten, so entfernen Sie diese manuell.
  3. Starten Sie über »DAVID\CLIENTS\WINDOWS\FAXIMG« die SETUP.EXE und selektieren Sie nur den Haken bei "Alten Fax-Image Treiber entfernen" und klicken Sie auf "Installieren".
  4. Wechseln Sie mittels Windows Explorer ins Verzeichnis WINNT und lassen Sie für dieses und alle untergeordneten Verzeichnisse nach den folgenden Dateien suchen:
    • DV*.*
    • DAVID*.*
    • TOBIT*.*
    • FW*.*
    • FAXWARE*.*
  5. Entfernen Sie jeweils die so gefunden Einträge von der Festplatte. Bei der Suche nach DV*.* könnten u.U. DVD*.* Dateien mitangezeigt werden, diese dürfen Sie nicht löschen, da es sich um Windows Systemdateien handelt.
  6. Starten Sie nun über »Start -> Ausführen« mittels dem Befehl "regedit" den Registry Editor von Windows.
  7. Löschen Sie die folgenden Einträge händisch aus der Registry falls vorhanden:
    • HKEY_CLASSES_ROOT/Applications/DVATT.EXE
    • HKEY_CLASSES_ROOT/Applications/DVEXTINF.EXE
    • HKEY_CLASSES_ROOT/Applications/DVWIN32.EXE
    • HKEY_CLASSES_ROOT/Applications/TobitFaxFile
    • HKEY_CLASSES_ROOT/Applications/TobitInfoCenter*
      (die jeweiligen Einträge unterhalb von Applications)
    • HKEY_CURRENT_USER/Software/Tobit
    • HKEY_LOCAL_MACHINE/Software/Classes/
      Applications/DVATT.EXE
    • HKEY_LOCAL_MACHINE/Software/Classes/
      Applications/DVEXTINF.EXE
    • HKEY_LOCAL_MACHINE/Software/Classes/
      Applications/DVWIN32.EXE
    • HKEY_LOCAL_MACHINE/Software/Classes/
      Applications/TobitFaxFile
    • HKEY_LOCAL_MACHINE/Software/Classes/
      Applications/TobitInfoCenter*
      (die jeweiligen Einträge unterhalb von Applications)
    • HKEY_LOCAL_MACHINE/Software/Clients/Mail/Tobit InfoCenter
    • HKEY_LOCAL_MACHINE/Software/Microsoft/Windows/
      CurrentVersion/App Managment/ARPCache/Tobit InfoCenter
    • HKEY_LOCAL_MACHINE/Software/Microsoft/Windows/
      CurrentVersion/Unindstall/Tobit InfoCenter
    • HKEY_LOCAL_MACHINE/Software/Microsoft/Windows NT/
      CurrentVersion/Terminal Server/Install/Software/Tobit
    • HKEY_USERS/.DEFAULT/Software/Tobit
Prüfen Sie weiterhin folgende Sachverhalte:
  1. Die Dateien NWCALLS.DLL und NWIPXSPX.DLL sollten nicht auf der Workstation vorhanden sein, wenn Sie TCP/IP konfiguriert haben. Löschen Sie die Dateien die sich im Windows- oder WINNT-Hauptverzeichnis befinden. Wenn Sie das IPX-Protokoll im Netz installiert haben, solten Sie sicherstellen, dass diese Dateien nicht davon verwendet werden.
  2. Die Benutzer sollten nicht manuell angelegt werden, da beim Starten des Clients die Benutzer automatisch angelegt werden.
  3. Ersetzen Sie in der TOBIT.INI in der Section [COMMON] den Servernamen durch die IP-Adresse. Der Eintrag sollte lauten "SERVER=IP-Adresse Ihres Servers".
  4. In der Bezeichnung des Servers und der Domäne sollten keine Sonderzeichen verwendet werden. Benutzen Sie auf einem NT-Server in der Bezeichnung keinesfalls einen Unterstrich. Bei einem Windows 2000 Server ist dies bedingt möglich, sollte aber vermieden werden. Eine Alternative stellt der Bindestrich dar.
  5. Tragen Sie das Standardgateway einmal aus und melden Sie den Client erneut an. Wenn der Client sich angemeldet hat kann das Gateway in den Netzwerkeigenschaften (Einstellungen des TCP/IP-Protokolls) wieder eingetragen werden.
  6. Es empfiehlt sich, nur das TCP/IP-Protokoll zu verwenden, da dieses ausreicht um sich am DvISE Server anzumelden und im Netzwerk zu arbeiten.
  7. Überprüfen Sie auch, ob die IP-Adressen und Subnetmask am Server mit denen der Workstations übereinstimmen.
  8. Spielen Sie das aktuelle Service Pack am Server ein.
  9. Gegebenenfalls ist eine hohe Serverauslastung dafür verantwortlich, dass sich die Clients nicht am DvISE Server anmelden können. Auslastung Server insbesondere Netzwerkkarte.
  10. Testen Sie außerdem, ob der Benutzer über die Netzwerkumgebung auf die Verzeichnisse des David zugreifen kann.
Falls es sich um Clients mit den Betriebssystemen Windows 95, 98 oder ME handelt beachten Sie auch die folgenden Punkte:
  1. Die Datei UNIDRV.DLL darf nur einmal im Windows-Hauptverzeichnis vorhanden sein und muss eine Größe von 197.840 Bytes haben.
  2. Deaktivieren Sie die Daten- und Druckerfreigabe in den Netzwerkeigenschaften der Workstation.
Falls es sich um NT-Workstations handelt beachten Sie auch die folgenden Punkte:
  1. Tragen Sie die Clients mit der IP-Adresse und dem Namen der Workstation in die HOSTS-Datei am Server ein. Diese Datei finden Sie unter »WINNT\SYSTEM32\DRIVERS\ETC«. Der Eintrag sollte z.B. wie folgt aussehen:
    DvISE-Server(3 Leerzeichen)192.168.0.1
  2. Binden Sie in den Netzwerkeigenschaften das TCP/IP-Protokoll am Server und am Client an die erste Stelle.
Hinweis:
Gegebenenfalls muss die WINSOCK.DLL erstezt werden. Informationen darüber erhalten Sie im KnowledgeBase Artikel Q-100.935.
