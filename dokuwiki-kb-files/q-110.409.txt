====== Q-110.409 - Überprüfung UTF-8 Content ======
==== Problem ====
PostMan und Grabbing Server unterstützen nun die Auswertung von UTF-8 Content. Voraussetzung ist, das MIS aktiviert ist. Dabei werden die Zeichen der Nachricht überprüft und der Anteil von Kyrillischen und Asiatischen Zeichen ermittelt. Wird ein gewisser Schwellenwert überschritten, so bekommt die Nachricht die entsprechende Spam Kennzeichnung. Die Überprüfung lässt sich per INI Schalter aktivieren (entweder jeweils über die INI des Moduls oder global in der David.ini)CheckUTF8Charset = TRUEDie Schwellenwerte (in %) lassen sich über folgende Einstellungen in der INI anpassen (hier angegeben sind die Default Werte)PlainTextThreshold = 70HTMLThreshold  = 10
==== Produkt ====
David Grabbing Server
==== Datum ====
31.01.2011
